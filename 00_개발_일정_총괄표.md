## 6. 의존성 체인 (Critical Path)

```
W0 5/8 금~5/11 월:  Gradle 멀티모듈 ✅ → OpenAPI 5종 ✅ → CI yml ✅ → K8s 네임스페이스 생성
     └─▶ W1 5/12(월): 강태오 Flyway V1 + Gateway AuthFilter 선행 → 나머지 5명 blocked 해소
           └─▶ W1 5/13~14: 서비스별 도메인 Entity + Repository 병렬 작성
                 └─▶ W1 5/15: K8s 스켈레톤 6 pod Running + Kafka 토픽 8종 생성
                       └─▶ W1 5/19~22: Loki·Prometheus·Jaeger 기동, Foundation 5항목 완성
                             └─▶ W2 5/26~5/29: 5 서비스 핵심 도메인 병렬 구현
                                   └─▶ W3 6/1~6/5: Kafka 이벤트 통합 + SSE E2E
                                         └─▶ W3 6/3(수) Feature Freeze
                                               └─▶ W4 6/8~6/12: 완성도·시연 빌드
                                                     └─▶ W5 6/15~6/17: 발표
```

**Critical Path 단축 전략**
- W0에 Gradle 골격·OpenAPI 완성 → W1 첫날부터 코드 작성 가능 (이미 완료 ✅)
- 강태오가 W1 5/12 오전 중 `gradlew build -x test` + K8s namespace 완료 → 5명 blocked 즉시 해소
- ChromaDB 시드 데이터(강상민)를 W0~W1 사이에 5개 이상 준비 → W2 AI 1턴 테스트 가능
- Kafka는 W3 집중 통합 — W2까지는 이벤트 없이 REST만으로 서비스 동작 확인

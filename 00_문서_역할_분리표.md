# 문서 역할 분리표 — QT-AI v0.1

> **문서 버전:** v0.1
> **작성일:** 2026-05-14
> **기준 문서:** `05_요구사항_정의서.md` v2.3

---

## 1. 목적

이 문서는 QT-AI 프로젝트 문서가 서로 같은 내용을 반복하거나, 서로 다른 기준을 말해서 혼란이 생기는 것을 막기 위한 문서 책임 기준표다.

현재 기준은 `05_요구사항_정의서.md` v2.3이다. 이후 문서를 분리하더라도 `05_요구사항_정의서.md`의 요구사항 기준은 흔들지 않고, 다른 문서는 자기 역할에 맞는 내용만 책임진다.

---

## 2. 문서 분리 원칙

| 원칙 | 설명 |
| --- | --- |
| 단일 책임 | 한 문서는 하나의 목적만 가진다. 요구사항, 계획, 설계, API, 품질 기준을 한 문서에 섞지 않는다. |
| 중복 최소화 | 같은 내용을 여러 문서에 반복하지 않는다. 필요한 경우 원문을 복사하지 않고 관련 문서를 참조한다. |
| 기준 문서 우선 | 요구사항 충돌 시 `05_요구사항_정의서.md` v2.3을 우선한다. |
| 상세 위치 고정 | API 상세는 API 명세서, 아키텍처 상세는 아키텍처 정의서, 일정과 범위는 계획서에 둔다. |
| 변경 이력 유지 | 각 문서는 자기 변경 이력만 가진다. 다른 문서의 변경 이력을 대신 관리하지 않는다. |
| 템플릿은 참고용 | 외부 GitHub 템플릿은 문서 형식 참고용이며, QT-AI 기준은 `05_요구사항_정의서.md`를 따른다. |

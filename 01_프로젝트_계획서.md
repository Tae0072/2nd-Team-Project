## 2. 프로젝트 개요

### 2.1 한 줄 정의

QT와 성경을 잘 모르는 사용자도 오늘의 본문을 쉽게 읽고, 한/영 성경과 AI가 미리 생성한 해설·장면 시뮬레이터를 함께 보며, 개인 묵상 기록을 남길 수 있도록 돕는 심플한 QT 보조 앱.

### 2.2 프로젝트 목표

| 목표 | 설명 |
| --- | --- |
| 쉬운 QT 진입 | QT를 처음 접하는 사용자도 오늘의 본문, 해설, 묵상 기록 흐름을 따라갈 수 있게 한다. |
| 개인 묵상 중심 | 피드 중심 커뮤니티보다 개인 기록과 선택적 익명 공유를 우선한다. |
| 사전 생성형 AI | 사용자가 AI에게 직접 질문하지 않고, 배치로 미리 생성·검증된 해설과 시뮬레이터 콘텐츠를 조회한다. |
| 저작권 리스크 완화 | 성경 본문, 주석, 해설 데이터는 출처·라이선스·재배포 가능 여부를 확인해 저작권 리스크를 낮춘다. |
| 구현 가능한 MVP | RAG, Kafka, K8s, 실시간 AI Q&A 등 범위가 큰 기능을 제외하고 v1 완성도를 우선한다. |

### 2.3 해결하는 문제

| # | 대상 | 페인 포인트 | 해결 방식 |
| --- | --- | --- | --- |
| 1 | 신앙 입문자 | 어디서부터 읽어야 할지 막막 | 오늘의 QT 본문 → 해설 → 시뮬레이터 → 묵상 기록 순서의 단계형 흐름 |
| 2 | 시간 부족 직장인 | 두꺼운 주석책 펼칠 시간 없음 | 필요할 때만 본문 요약·해설을 선택 열람 |
| 3 | 묵상 방법론 부재 신앙인 | 자기중심 해석에 빠지는 위험 | 객관적 큐티 A~D 가이드 + 사전 작성된 묵상 가이드 질문 |
| 4 | 기존 성경 AI 앱 사용자 | 답변 출처 확인이 어려움, 이단성 검증 부재 | AI 사전 생성 해설 + 편집자 에이전트 검증 + 응답 하단 출처 표시 |
| 5 | 본문 장면이 잘 안 그려지는 사용자 | 텍스트만으로 상황을 상상하기 어려움 | 본문 옆 아이콘 클릭 시 해당 장면 애니메이션 팝업 |
| 6 | 꾸준히 묵상하고 싶은 사용자 | 내 묵상 흐름을 한눈에 보기 어려움 | 마이페이지 묵상 달력 + 날짜별 기록 조회 |

### 2.4 서비스 방향성

- 초심자도 사용할 수 있는 쉬운 QT 앱을 지향한다.
- 예쁜 디자인보다 과하지 않은 심플한 디자인을 우선한다.
- 커뮤니티 중심이 아니라 개인 공간 중심으로 설계한다.
- 공유는 사용자가 필요할 때만 선택적으로 수행한다.
- 성경 본문, 영어 본문, 해설, 시뮬레이터, 묵상 기록을 하나의 흐름으로 제공한다.
- AI는 사용자가 직접 질문하는 챗봇이 아니라 본문에 딸리는 콘텐츠를 미리 만들어 두는 백엔드 도구로 사용한다.

### 2.5 대상 사용자

| 주요 사용자 | 사용자 문제 |
| --- | --- |
| QT를 처음 접하는 사용자 | QT가 무엇인지 모르고 앱을 켰을 때 무엇부터 해야 할지 모름 |
| 성경 지식이 많지 않은 사용자 | 성경 본문과 해설·묵상·적용 같은 용어가 어렵게 느껴짐 |
| 개인적으로 묵상하고 기록하고 싶은 사용자 | 개인적인 생각을 공개적으로 공유하기 부담스러움 |
| 커뮤니티 공유에 부담을 느끼는 사용자 | 활발한 피드·댓글·반응 중심 구조가 부담스러움 |
| 한/영 성경을 함께 보고 싶은 사용자 | 본문이 덩어리 단위로 제공되면 절별 비교가 불편함 |
| 본문 장면 상상이 어려운 사용자 | 텍스트만으로는 인물·배경·사건을 떠올리기 어려움 |
| 꾸준히 묵상하고 싶은 사용자 | 내가 며칠째 묵상하고 있는지, 빠진 날이 언제인지 한눈에 보고 싶음 |

## 2. Today QT 조회

```mermaid
sequenceDiagram
    actor User as 사용자
    participant App as Flutter App
    participant BFF as bff
    participant Bible as bible
    participant Sim as simulator
    participant Cache as Today QT Cache

    User->>App: 앱 진입
    App->>BFF: GET /api/v1/qt/today
    BFF->>Cache: 오늘 QT 캐시 조회
    alt HIT
        Cache-->>BFF: 본문·해설 진입점·묵상 진입점·시뮬레이터 상태 + cacheStatus=HIT
    else STALE_FALLBACK
        Cache-->>BFF: 이전 QT 캐시 + servedQtDate + staleReason
    else EMPTY
        BFF->>Bible: BibleQueryPort로 QT 본문 조회
        BFF->>Bible: ExplanationQueryPort로 해설 진입점 확인
        BFF->>Sim: SimulatorQueryPort로 상태 조회
        alt DB fallback 가능
            BFF->>Cache: 응답 캐시 저장
        else fallback 불가
            BFF-->>App: 준비 중 응답 + cacheStatus=EMPTY
        end
    end
    BFF-->>App: Today QT 응답 + cache 객체
    App-->>User: 본문과 진입점·캐시 상태 표시
```

| 기준 | 내용 |
| --- | --- |
| 00:00~04:00 | 새 범위 수집 전이므로 `STALE_FALLBACK`으로 이전에 준비된 캐시를 제공할 수 있다. |
| Today QT 100% | 본문, 해설 진입점, 묵상 진입점, 시뮬레이터 상태값을 반환한다. |
| 캐시 상태 | `HIT`, `MISS`, `STALE_FALLBACK`, `EMPTY` 중 하나를 `cache.cacheStatus`로 반환한다. |
| 캐시 없음 | 기존 캐시도 없으면 준비 중 상태를 표시한다. |
| 시뮬레이터 | `READY`가 아니면 버튼을 비활성화한다. |
| 금지 | 사용자 요청 처리 중 LLM을 호출하지 않는다. |

## 3. 해설 보기

```mermaid
sequenceDiagram
    actor User as 사용자
    participant App as Flutter App
    participant Auth as gatewayauth
    participant BFF as bff
    participant Bible as bible

    User->>App: 해설 보기 클릭
    App->>Auth: Access Token 확인
    alt 비로그인
        Auth-->>App: 로그인 필요
        App-->>User: 로그인 유도
    else 로그인
        App->>BFF: GET /api/v1/explanations/{bookCode}/{ch}/{v}
        BFF->>Bible: ExplanationQueryPort
        Bible-->>BFF: C 테이블 해설 + sources
        BFF-->>App: 해설 응답
        App-->>User: 해설·출처 표시
    end
```

| 기준 | 내용 |
| --- | --- |
| 사용자 노출 | C 테이블만 노출한다. |
| 비공개 | A 테이블 원문과 B 테이블 원문은 사용자 화면에 직접 노출하지 않는다. |
| AI 호출 | 해설 보기 시점에 LLM을 호출하지 않는다. |

## 4. 묵상 DRAFT 생성·자동 저장

```mermaid
sequenceDiagram
    actor User as 사용자
    participant App as Flutter App
    participant Auth as gatewayauth
    participant BFF as bff
    participant Bible as bible/journal
    participant Event as ApplicationEventPublisher

    User->>App: 묵상 작성 진입
    App->>Auth: 로그인 확인
    Auth-->>App: USER 확인
    App->>BFF: POST /api/v1/journals/today
    BFF->>Bible: JournalCommandPort.createOrGetTodayDraft
    Bible-->>BFF: journalId
    BFF-->>App: DRAFT 응답
    User->>App: 4개 섹션 중 일부 입력
    App->>BFF: PATCH /api/v1/journals/{id}
    BFF->>Bible: JournalCommandPort.update
    Bible->>Event: JournalUpdatedEvent 발행
    Event-->>Bible: 달력 캐시 무효화 핸들러 실행
    BFF-->>App: 저장 완료
```

| 기준 | 내용 |
| --- | --- |
| 생성 기준 | 오늘 QT 기준으로만 DRAFT를 생성한다. |
| 저장 기준 | 4개 섹션 중 최소 1개 이상 입력하면 저장 대상이다. |
| 이벤트 실패 | 핸들러 실패 로그와 재처리 근거를 남긴다. |
| 금지 | AI가 묵상 내용을 대신 생성·수정·요약하지 않는다. |

## 5. 익명 나눔 공개·반응·신고

```mermaid
sequenceDiagram
    actor User as 사용자
    participant App as Flutter App
    participant Auth as gatewayauth
    participant BFF as bff
    participant Journal as bible/journal
    participant Event as ApplicationEventPublisher

    User->>App: 묵상 상세에서 익명 공유 켜기
    App->>Auth: 로그인 확인
    Auth-->>App: USER 확인
    App->>BFF: POST /api/v1/journals/{id}/share
    BFF->>Journal: ShareCommandPort.createShare
    Journal-->>BFF: shareId
    BFF-->>App: 공유 생성 결과
    App-->>User: 익명 공유 상태 표시

    User->>App: 익명 나눔 목록 진입
    App->>BFF: GET /api/v1/shares
    BFF->>Journal: ShareQueryPort.findPublicShares
    Journal-->>BFF: 작성자 식별 정보 없는 공유 목록
    BFF-->>App: 공유 목록

    User->>App: 좋아요·댓글·신고
    App->>Auth: 로그인 확인
    Auth-->>App: USER 확인
    alt 좋아요
        App->>BFF: POST /api/v1/shares/{shareId}/likes
        BFF->>Journal: ShareCommandPort.like
    else 댓글
        App->>BFF: POST /api/v1/shares/{shareId}/comments
        BFF->>Journal: ShareCommandPort.comment
    else 신고
        App->>BFF: POST /api/v1/shares/{shareId}/reports
        BFF->>Journal: ShareCommandPort.report
        Journal->>Event: NotificationRequestedEvent 발행
    end
    BFF-->>App: 처리 결과
```

| 기준 | 내용 |
| --- | --- |
| 공유 방식 | 사용자가 명시적으로 선택한 묵상만 익명 공유한다. |
| 작성자 보호 | 공유 글 응답에는 `userId`, 이메일, 닉네임 등 작성자 식별 정보를 싣지 않는다. |
| 조회 | 공유 목록과 상세 조회는 PUBLIC 가능하다. |
| 반응 | 좋아요, 댓글 작성, 신고는 USER 권한이 필요하다. |
| 관리자 연계 | 신고된 글은 관리자 신고 관리 대상으로 전달한다. |
| 제외 | 팔로우, 랭킹, 실시간 댓글 피드, 세분화된 공개 범위는 v1에서 제외한다. |

## 6. 04:00 KST AI 배치

```mermaid
sequenceDiagram
    participant Scheduler as 04:00 KST Scheduler
    participant Bible as bible
    participant AI as ai
    participant Editor as editor verification
    participant Sim as simulator
    participant Cache as Today QT Cache
    participant Event as ApplicationEventPublisher

    Scheduler->>Bible: QT range 수집
    Bible-->>Scheduler: qtRange 저장
    Scheduler->>AI: 해설 생성 요청
    AI->>Bible: B 테이블 입력 조회
    AI->>AI: DeepSeek 호출
    AI->>Editor: A 테이블 기준 검증
    alt 검증 통과
        Editor-->>AI: PASS
        AI->>Bible: ExplanationSavePort로 C 테이블 저장
    else 검증 실패
        Editor-->>AI: REJECT
        AI->>AI: ai_runs 실패/반려 기록
    end
    Scheduler->>Sim: 시뮬레이터 생성 또는 상태 갱신
    Sim-->>Scheduler: READY/MISSING/FAILED/DISABLED
    Scheduler->>Cache: Today QT 캐시 교체
    Scheduler->>Event: AiContentGeneratedEvent 발행
```

| 기준 | 내용 |
| --- | --- |
| 실행 시점 | 매일 04:00 KST 수집 배치 이후 실행한다. |
| 실행 주체 | 사용자 계정이 아니라 `SYSTEM` 작업으로 기록한다. |
| 실패 | 실패 로그를 남기고 가능한 상태값을 반환한다. |
| 금지 | RAG, ChromaDB, vector DB, Elasticsearch를 사용하지 않는다. |

## 7. 관리자 AI 재생성

```mermaid
sequenceDiagram
    actor Admin as 관리자
    participant AdminWeb as Admin UI
    participant Auth as gatewayauth
    participant BFF as bff
    participant AI as ai
    participant Audit as admin_audit_logs

    Admin->>AdminWeb: 재생성 요청
    AdminWeb->>Auth: ADMIN 권한 확인
    Auth-->>AdminWeb: ADMIN 확인
    AdminWeb->>BFF: POST /api/v1/admin/ai/regenerate
    BFF->>Audit: 관리자 요청 기록
    BFF->>AI: 관리자 트리거 Command Port
    AI->>AI: SYSTEM 작업으로 재생성 실행
    AI-->>BFF: 접수/실행 결과
    BFF-->>AdminWeb: 결과 반환
```

| 기준 | 내용 |
| --- | --- |
| 접수 권한 | `ADMIN`만 요청 가능하다. |
| 실행 기록 | 실제 배치·AI 작업은 `SYSTEM`으로 기록한다. |
| 감사 로그 | 요청과 결과를 감사 로그에 남긴다. |
| 금지 | 사용자 요청 경로에서 LLM을 호출하지 않는다. |

## 8. 관리자 운영

```mermaid
sequenceDiagram
    actor Admin as 관리자
    participant AdminWeb as Admin UI
    participant Auth as gatewayauth
    participant BFF as bff
    participant Bible as bible
    participant AI as ai
    participant Sim as simulator
    participant Songs as bible/songs
    participant Audit as admin_audit_logs

    Admin->>AdminWeb: 관리자 화면 진입
    AdminWeb->>Auth: ADMIN 권한 확인
    Auth-->>AdminWeb: ADMIN 확인
    AdminWeb->>BFF: GET /api/v1/admin/dashboard
    BFF->>Bible: 총 묵상·QT 캐시 상태 조회
    BFF->>AI: AI 배치 처리량 조회
    BFF->>Bible: 신고 대기 수 조회
    BFF-->>AdminWeb: 대시보드 요약

    alt QT 범위 등록·수정·삭제
        AdminWeb->>BFF: POST/PATCH/DELETE /api/v1/admin/qt-ranges/**
        BFF->>Bible: QT 범위 변경 Command Port
        BFF->>Audit: 변경 감사 로그 기록
    else 해설 C 수정
        AdminWeb->>BFF: PATCH /api/v1/admin/explanations/{explanationId}
        BFF->>Bible: Explanation 관리 Command Port
        BFF->>Audit: 해설 수정 감사 로그 기록
    else 시뮬레이터 상태 조정
        AdminWeb->>BFF: PATCH /api/v1/admin/simulator/clips/{clipId}
        BFF->>Sim: Simulator 상태 변경 Command Port
        BFF->>Audit: 상태 변경 감사 로그 기록
    else 신고 처리
        AdminWeb->>BFF: PATCH /api/v1/admin/reports/{reportId}
        BFF->>Bible: 신고 처리·공유 글 비공개
        BFF->>Audit: 신고 처리 감사 로그 기록
    else 찬양 큐레이션 관리
        AdminWeb->>BFF: POST/PATCH/DELETE /api/v1/admin/songs/**
        BFF->>Songs: Song 관리 Command Port
        BFF->>Audit: 찬양 큐레이션 감사 로그 기록
    end
    BFF-->>AdminWeb: 처리 결과
```

| 기준 | 내용 |
| --- | --- |
| 권한 | `/api/v1/admin/**`는 `ADMIN`만 접근 가능하다. |
| 대시보드 | 총 묵상 횟수, AI 배치 처리량, 신고 대기 수, Today QT 캐시 상태를 표시한다. |
| 감사 로그 | QT 범위, C 해설, 시뮬레이터 상태, 신고, 찬양 변경은 감사 로그 대상이다. |
| A 테이블 | 사용자 화면과 사용자 API에는 A 테이블 원문을 노출하지 않는다. |
| 제외 | 교회 인증, AI 찬양 추천, 사용자 AI Q&A 기능은 관리자 화면에도 만들지 않는다. |

## 9. 찬양 큐레이션 저장

```mermaid
sequenceDiagram
    actor User as 사용자
    participant App as Flutter App
    participant Auth as gatewayauth
    participant BFF as bff
    participant Songs as bible/songs

    User->>App: 큐레이션 찬양 조회
    App->>BFF: GET /api/v1/songs/curated
    BFF->>Songs: SongQueryPort.findCurated
    Songs-->>BFF: 곡 메타데이터 목록
    BFF-->>App: 큐레이션 목록
    User->>App: 내 찬양 저장
    App->>Auth: 로그인 확인
    Auth-->>App: USER 확인
    App->>BFF: POST /api/v1/me/songs/{songId}
    BFF->>Songs: SongCommandPort.saveUserSong
    Songs-->>BFF: 저장 결과
    BFF-->>App: 저장 완료
```

| 기준 | 내용 |
| --- | --- |
| 조회 | 큐레이션 목록 조회는 PUBLIC이다. |
| 저장 | 내 찬양 저장·제거는 USER 권한이다. |
| 금지 | AI 찬양 추천, 가사 저장, 음원 저장, 사용자 직접 URL 입력은 MVP 제외다. |

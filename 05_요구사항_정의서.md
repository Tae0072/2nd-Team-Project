## 1. 프로젝트 개요

### 1.1 한 줄 정의

> QT와 성경을 잘 모르는 사용자도 오늘의 본문을 쉽게 읽고, 한/영 성경과 주석을 함께 보며, AI의 도움으로 이해하고, 개인 묵상 기록을 남길 수 있도록 돕는 심플한 QT 보조 앱.

### 1.2 해결하는 문제

| # | 대상 | 페인 포인트 | 해결 방식 |
| --- | --- | --- | --- |
| 1 | 신앙 입문자 | 어디서부터 읽어야 할지 막막 | 오늘의 QT 본문 → 설명 → 질문 → 기록 순서의 단계형 흐름 |
| 2 | 시간 부족 직장인 | 두꺼운 주석책 펼칠 시간 없음 | 필요할 때만 본문 설명·주석을 선택 열람 |
| 3 | 묵상 방법론 부재 신앙인 | 자기중심 해석에 빠지는 위험 | 객관적 큐티 A~D 가이드 + 적용 질문 |
| 4 | 기존 성경 AI 앱 사용자 | 답변 출처를 확인하기 어려움 | 본문·허용 주석 기반 답변 + 응답 하단 출처 표시 |

### 1.3 서비스 구성 (2026-05-12 확정)

| 서비스 | 담당자 | 주요 역할 |
| --- | --- | --- |
| API Gateway | 강태오 | JWT 검증, 라우팅, Rate Limit |
| BFF Aggregator | 강태오 | 병렬 호출 집계, UseCase 패턴 |
| Bible Service | 이지윤·이승욱 | 성경·주석·QT·묵상일지(Journal) 통합 |
| AI Service | 강태오(팀장)·김태혁·강상민 | DeepSeek 출처 기반 1회성 Q&A, ChromaDB RAG, SSE |
| Flutter 앱 | 김지민 | 모바일 UI, 관리자 웹 |

> **Auth Service 제거:** JWT 발급·검증은 Gateway에서 처리.
> **Journal Service 제거:** 묵상일지 기능 Bible Service로 통합.

### 1.4 서비스 방향성

- 초심자도 사용할 수 있는 쉬운 QT 앱
- 예쁜 디자인보다 과하지 않은 심플한 디자인
- 커뮤니티 중심이 아닌 개인 공간 중심
- 필요할 때만 선택적으로 공유하는 구조
- 성경 본문, 영어 본문, 주석, AI 질문, 묵상 기록을 한 흐름으로 제공

### 1.5 대상 사용자와 문제

| 주요 사용자 | 사용자 문제 |
| --- | --- |
| QT를 처음 접하는 사용자 | QT가 무엇인지 모르고 앱을 켰을 때 무엇부터 해야 할지 모름 |
| 성경 지식이 많지 않은 사용자 | 성경 본문과 주석·묵상·적용 같은 용어가 어렵게 느껴짐 |
| 개인적으로 묵상하고 기록하고 싶은 사용자 | 개인적인 생각을 공개적으로 공유하기 부담스러움 |
| 커뮤니티 공유에 부담을 느끼는 사용자 | 활발한 피드·댓글·반응 중심 구조가 부담스러움 |
| 한/영 성경을 함께 보고 싶은 사용자 | 본문이 덩어리 단위로 제공되면 절별 비교가 불편함 |

### 1.6 핵심 사용 흐름

1. 사용자가 앱을 처음 실행한다.
2. 튜토리얼에서 QT가 무엇인지 안내받는다.
3. 홈 화면에서 "오늘의 QT 시작하기"를 선택한다.
4. 오늘의 성경 본문을 절별 한글/영어 형태로 읽는다.
5. 어려운 단어, 본문 요약, 주석을 확인한다.
6. AI에게 궁금한 점을 질문한다.
7. 적용 질문에 답하며 개인 묵상 기록을 작성한다.
8. 원하면 찬양을 추가하거나 재생목록에 저장한다.
9. 기록은 기본적으로 비공개 저장된다.
10. 사용자가 선택한 경우에만 익명 형태로 공유한다.

### 2.2 F-01: 성경 묵상 화면 (입체 뷰)

**설명:** 사용자가 성경 구절을 선택하면 한글 성경·영어 성경을 절별로 비교하고, 필요할 때 본문 설명과 주석을 열람할 수 있는 화면.

**사용자 흐름:**
1. 앱 실행 → 오늘의 QT 또는 성경 검색에서 책·장·절 선택
2. 입체 묵상 화면 진입
3. 한/영 성경이 절별로 표시되고 긴 본문은 스크롤 동기화 (Sync Scroll)
4. 사용자가 필요할 때 본문 설명·주석 패널을 선택 열람
5. 로그인한 사용자는 구절 선택 후 AI 질문 또는 묵상 기록으로 이동 가능

**상세 요구사항:**

| ID | 요구사항 | 비고 |
| --- | --- | --- |
| F-01-01 | 한글 성경(개역한글) + 영어 성경(KJV)을 절별로 함께 표시한다 | 예: 1절 한글 → 1절 영어 |
| F-01-02 | 주석(Matthew Henry 영문, 더미 한글)은 사용자가 선택했을 때 표시한다 | 주석 열람은 로그인 필요 |
| F-01-03 | BFF가 Bible Service에 한글·영어 본문을 병렬 호출하고, 로그인 사용자의 선택 시 주석까지 단일 응답으로 반환한다 | CompletableFuture 사용 |
| F-01-04 | 자주 조회되는 구절은 Redis에 캐시한다 | TTL 24시간, 인기 구절 Top 100은 7일 |
| F-01-05 | 구절 조회 이력은 Kafka(`user.activity.tracked`)로 비동기 기록한다 | |
| F-01-06 | 각 서비스 호출 타임아웃은 800ms, 실패 시 부분 응답을 반환한다 | 폴백 처리 |
| F-01-07 | 성경 목록 조회(`GET /bible/books`)를 제공한다 | |
| F-01-08 | 성경 본문 보기와 오늘의 QT 미리보기는 비로그인 사용자도 접근할 수 있다 | 소프트 로그인 |

**저작권 제약:**

| 데이터 | 사용 가능 여부 |
| --- | --- |
| KJV (영어 성경) | ✅ Public Domain |
| 개역한글 | ⚠️ 비상업·교육 목적 한정, 출처 표기 필수 |
| Matthew Henry 주석 (영문) | ✅ Public Domain |
| 개역개정 / ESV / NIV | ❌ **사용 금지** |
| 한글 주석 / 신학 논문 | ❌ 더미 데이터로 대체 |

### 2.3 F-02: AI 질문 답변 (출처 기반 1회성 Q&A)

**설명:** 사용자가 오늘의 QT 본문이나 선택한 성경 구절에 대해 궁금한 점을 질문하면, AI가 성경 본문·허용 주석·승인된 RAG 출처를 기반으로 한 번의 질문에 답한다. 이어서 대화하는 자유형 챗봇이 아니라, QT 이해와 묵상을 돕는 질의응답 기능이다.

**큐티 단계 정의:**

| 단계 | 이름 | 설명 |
| --- | --- | --- |
| A | 관찰 | "본문에서 무엇이 보이나요?" — 사실 파악 |
| B | 해석 | "이 구절이 왜 쓰였을까요?" — 맥락 이해 |
| C | 적용 | "내 삶에 어떻게 적용할 수 있을까요?" |
| D | 결단 | "오늘 한 가지 실천하겠습니다" (MVP 시간 가능 시 구현) |

**사용자 흐름:**
1. 구절 선택 후 "AI에게 질문하기" 버튼 클릭
2. 로그인이 필요한 기능이므로 미로그인 사용자는 로그인 화면으로 유도
3. AI 세션 생성 (`POST /ai/sessions`)
4. 사용자가 질문 입력 (`POST /ai/sessions/{id}/turns`)
5. AI 응답을 SSE 스트리밍으로 실시간 표시
6. 응답 하단에 참고 출처를 표시
7. 사용자가 "묵상 완료"를 선택하면 세션을 완료하고 묵상 노트 생성 흐름으로 연결

**상세 요구사항:**

| ID | 요구사항 | 비고 |
| --- | --- | --- |
| F-02-01 | AI 응답을 SSE(Server-Sent Events)로 스트리밍한다 | Spring SseEmitter |
| F-02-02 | SSE 이벤트 순서는 `turn_started → token → rag_sources → turn_completed → [DONE]`이다 | |
| F-02-03 | ChromaDB에서 관련 주석·허용 자료 임베딩을 RAG로 조회한다 | 무단 수집 논문 제외 |
| F-02-04 | RAG 컨텍스트 외 추론을 금지하고 출처 인용을 강제한다 | 신학적 오류 방지 |
| F-02-05 | LLM은 DeepSeek API (OpenAI 호환)를 사용한다 | Spring RestClient |
| F-02-06 | 첫 토큰 응답 타임아웃은 5초, 초과 시 안내 메시지를 반환한다 | |
| F-02-07 | AI 세션·질문·응답 전체는 MySQL(`AI_SESSIONS`, `AI_TURNS`)에 저장한다 | 관리자 이상 응답 모니터링 가능 |
| F-02-08 | 사용자가 "묵상 완료"를 누를 때 Kafka(`ai.session.completed`)를 발행한다 | Bible Service가 구독 → 묵상 노트 DRAFT 생성 |
| F-02-09 | QT A~D 단계는 가이드/프롬프트 구조로 제공하되, AI가 자동으로 이어지는 다중 턴 챗봇처럼 동작하지 않는다 | 1회성 Q&A 원칙 |
| F-02-10 | AI API Key는 K8s Secret으로만 관리하며 코드·YAML에 평문 저장을 금지한다 | |
| F-02-11 | QT 가이드 답변은 쉬운 말투, 본문 해설은 차분하고 상세한 말투로 구분한다 | 회의 결정 |

### 2.4 F-03: 묵상 노트 관리

**설명:** 사용자가 오늘의 QT 본문을 읽고 느낀 점, 기억할 구절, 적용할 점, 기도 제목을 분리해 저장한다. 기록은 기본 비공개이며, 사용자가 원할 때만 나눔으로 공개한다.

> **구현 기준:** 별도 Journal API는 사용하지 않는다. 묵상 노트 API 계약은 Bible Service의 `apis/bible/openapi.yaml`에 포함한다.

**사용자 흐름:**
1. 오늘의 QT 또는 AI 질문 흐름에서 "묵상 완료" 선택
2. Bible Service가 DRAFT 상태의 묵상 노트 생성 또는 기존 DRAFT 갱신
3. 사용자가 4개 섹션(느낀 점, 기억할 구절, 적용할 점, 기도)을 작성
4. 마이페이지 → 묵상 기록 리스트 확인
5. 노트 클릭 → 상세 보기 (구절·날짜·AI 응답 요약·개인 기록)
6. 기록 수정, 삭제, 나눔 공개 여부 선택

**상세 요구사항:**

| ID | 요구사항 | 비고 |
| --- | --- | --- |
| F-03-01 | 묵상 노트는 `ai.session.completed` Kafka 이벤트를 Bible Service가 구독하여 자동 생성한다 | 수동 생성 API(`POST /api/v1/journals`) 없음 |
| F-03-02 | 묵상 노트 목록을 조회한다 (`GET /api/v1/journals`) | |
| F-03-03 | 묵상 노트 단건을 조회한다 (`GET /api/v1/journals/{id}`) | |
| F-03-04 | 묵상 노트를 수정한다 (`PATCH /api/v1/journals/{id}`) | 개인 메모 추가·수정 |
| F-03-05 | 묵상 노트를 삭제한다 (`DELETE /api/v1/journals/{id}`) | |
| F-03-06 | 이벤트 변경 이력을 조회한다 (`GET /api/v1/journals/{id}/events`) | 이벤트 소싱 |
| F-03-07 | 묵상 노트 생성·수정·삭제 이벤트는 Kafka로 발행한다 | producer: bible-service |
| F-03-08 | Kafka 컨슈머는 멱등성 키를 검증하여 중복 처리를 방지한다 | |
| F-03-09 | 기록 필드는 `느낀 점`, `기억할 구절`, `적용할 점`, `기도` 4개 섹션으로 분리한다 | 추후 필요 시 통합 가능 |
| F-03-10 | 공개 범위는 MVP에서 `PRIVATE`, `ANONYMOUS_SHARED` 중심으로 시작한다 | 세분화 공개 범위는 v1.1 이후 |

### 2.5 F-04: 회원 인증

**설명:** JWT 기반 인증으로 앱에 로그인한다. 인증 처리는 Gateway에서 담당하며, 첫 앱 진입 시 강제 로그인하지 않는 소프트 로그인 정책을 사용한다.

**상세 요구사항:**

| ID | 요구사항 | 비고 |
| --- | --- | --- |
| F-04-01 | Google 소셜 로그인을 지원한다 | OAuth 2.0 |
| F-04-02 | Access Token 유효기간은 30분(1800s)이다 | RS256 알고리즘 |
| F-04-03 | Refresh Token 유효기간은 14일이다 | |
| F-04-04 | Refresh Token 블랙리스트는 Redis에 저장한다 | `auth:refresh:revoked:{jti}` |
| F-04-05 | JWT 검증은 Gateway 필터에서 처리한다 | Auth Service 없음 |
| F-04-06 | 계정 탈퇴 기능은 MVP 범위에서 제외한다 | v1.1에서 구현 |
| F-04-07 | 튜토리얼, 성경 본문 보기, 오늘의 QT 미리보기는 비로그인 접근을 허용한다 | 첫 진입 강제 로그인 없음 |
| F-04-08 | 묵상 기록 저장, 주석 열람, AI 질문, 찬양 저장·공유는 로그인 필수다 | 기능 진입 시 로그인 유도 |

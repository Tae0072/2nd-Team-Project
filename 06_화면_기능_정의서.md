## 1. 문서 목적과 경계

이 문서는 사용자와 관리자가 보는 화면 단위의 기능을 정의한다. API 상세와 DB 구조는 반복하지 않는다.

| 구분 | 이 문서에서 관리 | 다른 문서에서 관리 |
| --- | --- | --- |
| 화면별 기능 | 포함 | 기능 원본은 `07` |
| 버튼·상태 | 포함 | API 계약은 `04` |
| 권한·로그인 유도 | 포함 | 권한 용어는 `23` |
| 화면 흐름 | 요약만 포함 | 상세 흐름은 `08_스토리보드.md`에서 관리 |

## 2. 화면 공통 규칙

| 항목 | 기준 |
| --- | --- |
| 첫 진입 | 별도 홈 화면 없이 Today QT 화면으로 진입한다. |
| 로그인 정책 | 첫 진입은 강제 로그인하지 않는다. 저장·개인화 기능 진입 시 로그인 유도한다. |
| AI 질문 | AI 질문하기 버튼, 챗봇, SSE 스트리밍 UI를 두지 않는다. |
| 찬양 | AI 추천 영역을 두지 않는다. 운영자 큐레이션 목록만 제공한다. |
| 교회 인증 | 회원가입·마이페이지 어디에도 교회 인증 버튼을 두지 않는다. |
| 시뮬레이터 | `READY`일 때만 보기 버튼을 활성화한다. |

## 3. 화면 목록

| 화면 ID | 화면명 | 접근 | 주요 기능 |
| --- | --- | --- | --- |
| S-01 | 온보딩 | PUBLIC | QT 안내, 오늘 QT 시작 |
| S-02 | Today QT | PUBLIC | 본문, 해설 진입점, 묵상 진입점, 시뮬레이터 상태 |
| S-03 | 해설 보기 | USER | C 테이블 해설, 요약, 단어, 출처 |
| S-04 | 묵상 작성 | USER | 4개 섹션 작성, 자동 저장 |
| S-05 | 마이페이지 | USER | 묵상 달력, 묵상 목록, 내 찬양 목록 |
| S-06 | 묵상 상세 | USER | 기록 조회, 수정, 삭제, 공개 토글 |
| S-07 | 익명 나눔 | PUBLIC/USER | 공유 글 조회, 좋아요, 댓글, 신고 |
| S-08 | 시뮬레이터 팝업 | PUBLIC | 클립 재생, 정지, 닫기 |
| S-09 | 찬양 큐레이션 | PUBLIC/USER | 큐레이션 조회, 내 목록 저장/제거 |
| S-10 | 관리자 | ADMIN | QT 범위, 해설, AI 로그, 신고, 찬양 큐레이션 관리 |

## 4. 사용자 화면 상세

### 4.1 S-01 온보딩

| 항목 | 내용 |
| --- | --- |
| 목적 | QT가 처음인 사용자에게 앱 사용 순서를 알려준다. |
| 주요 요소 | QT 설명, 앱 사용 순서, 오늘 QT 시작 버튼 |
| 권한 | PUBLIC |
| 다음 화면 | Today QT |
| 제외 | 회원가입 강제, 교회 인증 안내 |

### 4.2 S-02 Today QT

| 항목 | 내용 |
| --- | --- |
| 목적 | 당일 QT 본문과 주요 진입점을 제공한다. |
| 주요 요소 | 한글 본문, 영어 본문 접기/펼치기, 해설 보기, 묵상 작성, 시뮬레이터 보기, 찬양 큐레이션 진입 |
| 권한 | PUBLIC. 단, 해설 본문 열람·묵상 저장·찬양 저장은 USER 필요 |
| 빈 상태 | 04:00 배치 전에는 이전 캐시를 보여줄 수 있다. 캐시도 없으면 "준비 중" 상태를 표시한다. |
| 오류 상태 | 부분 실패 시 가능한 본문과 진입점 상태를 보여주고 실패 영역만 비활성화한다. |
| 제외 | AI 질문하기, AI 찬양 추천, 자유 본문 묵상 진입 |

### 4.3 S-03 해설 보기

| 항목 | 내용 |
| --- | --- |
| 목적 | C 테이블의 사용자 노출 해설을 제공한다. |
| 주요 요소 | 한 줄 요약, 배경 설명, 어려운 단어, 해설 본문, 출처 |
| 권한 | USER |
| 로그인 전 | 로그인 유도 후 원래 본문으로 복귀한다. |
| 제외 | A 테이블 원문, B 테이블 원문 직접 노출, 실시간 AI 생성 |

### 4.4 S-04 묵상 작성

| 항목 | 내용 |
| --- | --- |
| 목적 | 오늘 QT 기준 개인 묵상을 작성한다. |
| 주요 요소 | 느낀 점, 기억할 구절, 적용할 점, 기도, 자동 저장 상태 |
| 권한 | USER |
| 저장 기준 | 4개 필드 중 최소 1개 이상 입력 시 저장 가능 |
| 빈 상태 | 오늘 QT에 아직 작성한 묵상이 없으면 빈 DRAFT를 보여준다. |
| 제외 | AI가 묵상 내용을 대신 생성·수정·요약하는 기능 |

### 4.5 S-05 마이페이지

| 항목 | 내용 |
| --- | --- |
| 목적 | 개인 묵상 이력과 내 찬양 목록을 확인한다. |
| 주요 요소 | 월별 묵상 달력, 묵상 기록 리스트, 내 찬양 목록 |
| 권한 | USER |
| 빈 상태 | 묵상 기록이 없으면 시작 안내를 표시한다. |
| 제외 | 팔로우, 랭킹, 공개 피드 중심 UI |

### 4.6 S-06 묵상 상세

| 항목 | 내용 |
| --- | --- |
| 목적 | 특정 날짜의 묵상 기록을 조회하고 수정한다. |
| 주요 요소 | 본문 정보, 4개 섹션, 수정, 삭제, 공개 토글 |
| 권한 | USER |
| 오류 상태 | 본인 기록이 아니면 접근 불가 |
| 공유 | 기본값은 비공개이며 사용자가 선택할 때만 익명 공유 |

### 4.7 S-07 익명 나눔

| 항목 | 내용 |
| --- | --- |
| 목적 | 사용자가 선택한 묵상을 조용히 공유한다. |
| 주요 요소 | 공유 글, 좋아요, 댓글, 신고 |
| 조회 권한 | PUBLIC 가능 |
| 작성·반응 권한 | USER |
| 제외 | 작성자 정보 노출, 팔로우, 실시간 피드, 랭킹 |

### 4.8 S-08 시뮬레이터 팝업

| 항목 | 내용 |
| --- | --- |
| 목적 | 본문 장면을 사전 생성 클립으로 보여준다. |
| 주요 요소 | 재생, 정지, 닫기 |
| 권한 | PUBLIC |
| 상태 | `READY`만 활성화. `MISSING`, `FAILED`, `DISABLED`는 비활성화 |
| 오류 상태 | 클립 로딩 실패 시 닫기와 재시도만 제공 |

### 4.9 S-09 찬양 큐레이션

| 항목 | 내용 |
| --- | --- |
| 목적 | 운영자 큐레이션 찬양을 조회하고 내 목록에 저장한다. |
| 주요 요소 | 곡명, 아티스트, 메타데이터, 저장/제거 버튼 |
| 조회 권한 | PUBLIC |
| 저장 권한 | USER |
| 제외 | AI 추천, 가사·음원 저장, 사용자 직접 URL 입력 |

## 5. 관리자 화면 상세

| 화면 | 주요 기능 | 권한 | 감사 로그 |
| --- | --- | --- | --- |
| QT 범위 관리 | 오늘 QT 범위 등록·수정·삭제 | ADMIN | 필수 |
| 성경 데이터 상태 | JSON 적재 누락과 소스 검토 상태 조회 | ADMIN | 조회는 선택 |
| 해설 C 관리 | 사용자 노출 해설 조회·수정 | ADMIN | 필수 |
| AI 로그 | 생성·검증·반려 로그 조회, 재생성 트리거 | ADMIN | 재생성 필수 |
| 신고 관리 | 신고 조회, 삭제 또는 비공개 처리 | ADMIN | 필수 |
| 찬양 큐레이션 | 곡 메타데이터 등록·수정·삭제 | ADMIN | 필수 |
| 감사 로그 | 관리자 작업 이력 조회 | ADMIN | 조회는 선택 |

관리자 화면에서도 A 테이블 원문 사용자 노출, 교회 인증, AI 찬양 추천, 사용자 AI Q&A 기능을 만들지 않는다.

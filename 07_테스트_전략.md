## 1. 테스트 정책 개요

### 1.1 테스트 피라미드 (역피라미드 아닌 정상 피라미드 강제)

```
                           ┌──────────┐
                           │   E2E    │   ~10%  (느림, 비싸고, 깨지기 쉬움)
                           │ (REST    │
                           │ Assured) │
                       ┌───┴──────────┴───┐
                       │   Integration    │   ~30% (Testcontainers + Kafka + DB)
                       │   (Spring Boot)  │
                   ┌───┴──────────────────┴───┐
                   │       Unit Test          │   ~60% (빠름, 격리, 많이)
                   │     (JUnit + Mockito)    │
                   └──────────────────────────┘
```

| 계층 | 비율 | 도구 | 목적 |
| --- | --- | --- | --- |
| Unit | 60% | JUnit 5 + Mockito + AssertJ | 도메인 로직 격리 검증 |
| Integration | 30% | Spring Boot Test + Testcontainers (mysql, kafka, redis) | 실제 인프라와의 통합 |
| E2E | 10% | REST Assured + Flutter integration_test | 사용자 시나리오 |

> **금지:** 단위 테스트로 검증 가능한 것을 통합 테스트로 작성 X. 통합으로 검증 가능한 것을 E2E로 X. 위 피라미드 비율을 W4 회고 시 점검.

### 1.2 테스트 원칙

| 원칙 | 적용 |
| --- | --- |
| **Given-When-Then** | 모든 테스트는 3단계 명시 (BDD 스타일 prose 또는 주석) |
| **Test Isolation** | 각 테스트는 독립 — `@DirtiesContext` 남용 X. 테스트 간 데이터 누수 X |
| **Fail Loud** | 실패 시 명확한 메시지 (`assertThat().as("이 케이스는 ...")`) |
| **Determinism** | 무작위·시간 의존 테스트 X. `Clock` 주입, seed 고정 |
| **Repeatable** | 같은 input → 같은 output. CI에서 ~~한번~~ 100번 돌려도 같음 |
| **Fast Feedback** | 단위 테스트 전체 < 30s, 통합 < 5min, E2E < 10min |
| **Test What Could Break** | 100% coverage 목표 X. 비즈니스 critical + 1차(HMS) 사고 영역 우선 |
| **Test as Documentation** | 테스트 이름은 한글로 사양 묘사 (예: `이미_탈퇴된_사용자가_같은_email로_재가입_시_새_USERS_생성된다`) |

### 1.3 RACI

| 영역 | Responsible | Accountable |
| --- | --- | --- |
| 단위 테스트 (각 service) | service owner | service owner |
| 통합 테스트 (각 service) | service owner | service owner |
| Service 간 통합 (Kafka, Saga) | 영향 owner들 페어 | 강태오 |
| E2E 테스트 (사용자 시나리오) | 강태오 (BFF) + 김지민 (Flutter) | 강태오 |
| 부하 테스트 (k6 시나리오) | 강태오 | 강태오 |
| 보안 테스트 (IDOR · 인증) | 강태오 (Gateway Owner) | 강태오 (Lead 단독 — Gateway 내부 인증 모듈) |
| AI 회귀 (환각·신학) | 강상민 | 강상민 |
| Flutter 단위·위젯 | 김지민 | 김지민 |
| Test fixture · Factory | service owner | 강태오 |
| ArchUnit 아키텍처 룰 | 강태오 | 강태오 |
| CI 테스트 시간 모니터링 | 강태오 | 강태오 |

### 2.2 Testcontainers 표준 설정

> **Testcontainers**: 테스트 실행 시 Docker 컨테이너로 실제 인프라(MySQL, Kafka 등)를 띄우는 라이브러리. H2 in-memory DB와 달리 운영과 동일한 환경.

```kts
// build.gradle.kts (각 service)
dependencies {
    testImplementation("org.springframework.boot:spring-boot-starter-test")
    testImplementation("org.testcontainers:testcontainers:1.19.7")
    testImplementation("org.testcontainers:junit-jupiter:1.19.7")
    testImplementation("org.testcontainers:mysql:1.19.7")
    testImplementation("org.testcontainers:kafka:1.19.7")
    testImplementation("com.redis:testcontainers-redis:2.2.2")
    testImplementation("com.github.tomakehurst:wiremock-jre8-standalone:3.0.1")
    testImplementation("org.awaitility:awaitility:4.2.0")
    testImplementation("io.rest-assured:rest-assured:5.4.0")
    testImplementation("com.tngtech.archunit:archunit-junit5:1.2.1")
}

tasks.test {
    useJUnitPlatform()
    systemProperty("testcontainers.reuse.enable", "true")     // 컨테이너 재사용 (속도 ↑)
}
```

> **속도 팁 — `~/.testcontainers.properties`**:
> ```
> testcontainers.reuse.enable=true
> ```
> 한번 띄운 mysql 컨테이너를 다음 테스트 실행 때 재사용 → 30s → 5s 단축.

```java
@SpringBootTest
@Testcontainers
@ActiveProfiles("test")
public abstract class IntegrationTestBase {

    @Container
    static final MySQLContainer<?> MYSQL = new MySQLContainer<>("mysql:8.0")
        .withDatabaseName("testdb")
        .withUsername("test")
        .withPassword("test")
        .withReuse(true);

    @Container
    static final ConfluentKafkaContainer KAFKA = new ConfluentKafkaContainer("confluentinc/cp-kafka:7.6.0")
        .withReuse(true);

    @Container
    static final RedisContainer REDIS = new RedisContainer(DockerImageName.parse("redis:7-alpine"))
        .withReuse(true);

    @DynamicPropertySource
    static void registerProperties(DynamicPropertyRegistry registry) {
        registry.add("spring.datasource.url",      MYSQL::getJdbcUrl);
        registry.add("spring.datasource.username", MYSQL::getUsername);
        registry.add("spring.datasource.password", MYSQL::getPassword);
        registry.add("spring.kafka.bootstrap-servers", KAFKA::getBootstrapServers);
        registry.add("spring.data.redis.host", REDIS::getHost);
        registry.add("spring.data.redis.port", () -> REDIS.getMappedPort(6379).toString());
    }

    @BeforeEach
    void cleanDatabase(@Autowired DataSource dataSource) {
        // 각 테스트마다 truncate (Flyway 시드는 유지)
        try (var conn = dataSource.getConnection(); var stmt = conn.createStatement()) {
            stmt.execute("SET FOREIGN_KEY_CHECKS=0");
            // ... 각 테이블 truncate
            stmt.execute("SET FOREIGN_KEY_CHECKS=1");
        } catch (SQLException e) { throw new RuntimeException(e); }
    }
}
```

## 3. 단위 테스트 (Unit Test)

### 3.1 표준 패턴 — Given-When-Then

```java
// AuthServiceTest.java
@ExtendWith(MockitoExtension.class)
class AuthServiceTest {

    @Mock UserRepository userRepository;
    @Mock RefreshTokenRepository refreshTokenRepository;
    @Mock BCryptPasswordEncoder encoder;
    @Mock JwtTokenProvider tokenProvider;

    @InjectMocks AuthService authService;

    @Test
    @DisplayName("유효한_이메일과_비밀번호로_로그인_시_JWT_토큰_2종_반환")
    void login_with_valid_credentials_returns_token_pair() {
        // given
        String email = "user@example.com";
        String rawPassword = "P@ssw0rd123";
        User user = UserBuilder.builder().email(email).passwordHash("$2a$12$...").build();
        given(userRepository.findActiveByEmail(email)).willReturn(Optional.of(user));
        given(encoder.matches(rawPassword, user.getPasswordHash())).willReturn(true);
        given(tokenProvider.createAccessToken(user)).willReturn("access-jwt");
        given(tokenProvider.createRefreshToken(user)).willReturn("refresh-jwt");

        // when
        LoginResponse result = authService.login(email, rawPassword);

        // then
        assertThat(result.accessToken()).isEqualTo("access-jwt");
        assertThat(result.refreshToken()).isEqualTo("refresh-jwt");
        verify(refreshTokenRepository).save(any(RefreshToken.class));
    }

    @Test
    @DisplayName("탈퇴된_사용자가_로그인_시도_시_INVALID_CREDENTIALS_예외")
    void login_with_deactivated_user_throws_exception() {
        // given
        given(userRepository.findActiveByEmail("user@example.com"))
            .willReturn(Optional.empty());                       // SQLRestriction으로 탈퇴자 제외

        // when + then
        assertThatThrownBy(() -> authService.login("user@example.com", "any"))
            .isInstanceOf(InvalidCredentialsException.class)
            .hasMessageContaining("INVALID_CREDENTIALS");
    }

    @Test
    @DisplayName("타이밍_공격_방어_사용자_없어도_bcrypt_matches_호출됨")
    void timing_attack_defense_encoder_matches_called_even_for_missing_user() {
        // given
        given(userRepository.findActiveByEmail(anyString())).willReturn(Optional.empty());

        // when
        try { authService.login("ghost@example.com", "any"); } catch (Exception ignored) {}

        // then — encoder.matches는 dummy hash로라도 호출되어 응답 시간 일정 (05번 § 3.2)
        verify(encoder, atLeastOnce()).matches(anyString(), anyString());
    }
}
```

### 3.2 IDOR 방어 단위 테스트 표준 (05번 § 4.2)

> **반드시 작성** — 자원의 owner 검증을 빼먹지 않도록 강제.

```java
@ExtendWith(MockitoExtension.class)
class JournalServiceTest {

    @Mock JournalRepository journalRepository;
    @InjectMocks JournalService journalService;

    @Test
    @DisplayName("IDOR_다른_사용자의_journal_수정_시도_시_JOURNAL_NOT_OWNED_403")
    void IDOR_other_users_journal_throws_forbidden() {
        // given
        Long journalId = 12345L;
        Long ownerId = 5678L;
        Long attackerId = 9999L;
        Journal journal = JournalBuilder.builder().id(journalId).userId(ownerId).build();
        given(journalRepository.findById(journalId)).willReturn(Optional.of(journal));

        // when + then
        assertThatThrownBy(() -> journalService.updateJournal(journalId, attackerId, new JournalUpdateDto(...)))
            .isInstanceOf(ForbiddenException.class)
            .hasMessageContaining("JOURNAL_NOT_OWNED");

        // 자원이 변경되지 않았는지 검증
        verify(journalRepository, never()).save(any());
    }

    @Test
    @DisplayName("본인_journal_수정_시_정상_저장")
    void own_journal_update_succeeds() {
        Long journalId = 12345L, userId = 5678L;
        Journal journal = JournalBuilder.builder().id(journalId).userId(userId).build();
        given(journalRepository.findById(journalId)).willReturn(Optional.of(journal));

        journalService.updateJournal(journalId, userId, new JournalUpdateDto(...));

        verify(journalRepository).save(journal);
    }
}
```

> **W1 PR 머지 룰:** owner 받는 service 메서드는 IDOR 단위 테스트 1개 이상 필수.

### 4.2 트랜잭션 경계 테스트 (1차 핵심 가드레일)

> **1차 실패:** 트랜잭션 누락으로 인해 DB write A 성공, B 실패 시 부분 커밋. **본 § 4.2가 본 문서 핵심.**

```java

@DisplayName("동일_journal에_동시_PATCH_10회_sequence_충돌_없음")

    ExecutorService pool = Executors.newFixedThreadPool(threadCount);
    CountDownLatch latch = new CountDownLatch(threadCount);
    List<Throwable> errors = Collections.synchronizedList(new ArrayList<>());

    // when — 10 thread 동시 PATCH

        pool.submit(() -> {
            try {
                journalService.updateJournal(journalId, ownerId,
                    new JournalUpdateDto("title-" + Thread.currentThread().getId(), null, null));
            } catch (Throwable t) {
                errors.add(t);
            } finally {
                latch.countDown();
            }
        });
    }
    latch.await(30, SECONDS);
    pool.shutdown();

    List<JournalEvent> events = journalEventRepository.findByJournalIdOrderBySequence(journalId);
    assertThat(events).hasSize(10);
```

```java
@Tag("ai-regression")
class AIRegressionTest {

    @Autowired AISessionService sessionService;

        return YamlLoader.load("/ai-regression/golden-set.yaml", RegressionCase.class);

    @MethodSource("loadGoldenSet")
    @DisplayName("AI_응답이_신학_가드레일을_통과")
    void ai_response_meets_theological_guardrail(RegressionCase tc) {

            tc.passage(), tc.userMessage(), tc.step());

                .containsIgnoringCase(pattern);

        for (String pattern : tc.mustNotContainPatterns()) {

                .doesNotContainIgnoringCase(pattern);

                .isNotEmpty();
        }
```

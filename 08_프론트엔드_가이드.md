### 2.2 Clean Architecture 3-layer

| Layer | 의존 방향 | 책임 |
| --- | --- | --- |
| **presentation** | → domain | UI·상태 (RiverPod Notifier)·라우팅 |
| **domain** | (의존 없음) | 순수 모델·비즈니스 룰 (Flutter·dio 등 외부 의존 X) |
| **data** | → domain | API 호출·DTO ↔ domain 모델 변환·repository 인터페이스 구현 |

### 3.2 토큰 저장 구현

```dart
// core/auth/token_storage.dart
import 'package:flutter_secure_storage/flutter_secure_storage.dart';

class TokenStorage {
  static const _kRefreshKey = 'qtai.refresh_token';
  
  final FlutterSecureStorage _storage;
  
  TokenStorage()
      : _storage = const FlutterSecureStorage(
          aOptions: AndroidOptions(
            encryptedSharedPreferences: true,  // Android Keystore + EncryptedSharedPreferences
          ),
          iOptions: IOSOptions(
            accessibility: KeychainAccessibility.first_unlock,
          ),
        );

  Future<void> saveRefreshToken(String token) async {
    await _storage.write(key: _kRefreshKey, value: token);
  }

  Future<String?> readRefreshToken() async {
    return await _storage.read(key: _kRefreshKey);
  }

  Future<void> clearRefreshToken() async {
    await _storage.delete(key: _kRefreshKey);
  }
}
```

### 3.3 자동 갱신 인터셉터 (가장 중요)

**문제:** Access Token 30분 만료 → API 호출 시 401 → 사용자가 매번 로그인 X. 자동으로 Refresh로 새 Access 발급.

**해결:** DIO 인터셉터의 `onError`에서 401 감지 → `/auth/refresh` 호출 → 원래 요청 재시도. 단, **race condition 방어 필수** (여러 요청 동시 401 → refresh 5번 호출되면 안 됨).

```dart
// core/network/auth_interceptor.dart
import 'dart:async';
import 'package:dio/dio.dart';
import '../auth/token_storage.dart';

class AuthInterceptor extends QueuedInterceptor {
  final Dio _dio;
  final TokenStorage _storage;
  String? _accessToken;  // 메모리에만

  // race condition 방어 — refresh 진행 중이면 다른 요청은 대기
  Completer<String?>? _refreshCompleter;

  AuthInterceptor(this._dio, this._storage);

  void setAccessToken(String? token) {
    _accessToken = token;
  }

  @override
  void onRequest(RequestOptions options, RequestInterceptorHandler handler) {
    // 인증 면제 endpoint는 헤더 추가 X
    if (_isAuthExempt(options.path)) {
      return handler.next(options);
    }

    if (_accessToken != null) {
      options.headers['Authorization'] = 'Bearer $_accessToken';
    }
    handler.next(options);
  }

  @override
  void onError(DioException err, ErrorInterceptorHandler handler) async {
    if (err.response?.statusCode != 401 ||
        _isAuthExempt(err.requestOptions.path) ||
        err.requestOptions.extra['_retried'] == true) {
      return handler.next(err);
    }

    try {
      final newAccessToken = await _refreshAccessToken();
      if (newAccessToken == null) {
        return handler.next(err);
      }

      // 원래 요청 재시도
      final clone = await _retry(err.requestOptions, newAccessToken);
      handler.resolve(clone);
    } catch (e) {
      // refresh 실패 → 로그인 화면 유도 (외부 콜백)
      handler.next(err);
    }
  }

  Future<String?> _refreshAccessToken() async {
    // 이미 다른 요청이 refresh 진행 중이면 대기
    if (_refreshCompleter != null) {
      return _refreshCompleter!.future;
    }

    final completer = Completer<String?>();
    _refreshCompleter = completer;

    try {
      final refreshToken = await _storage.readRefreshToken();
      if (refreshToken == null) {
        completer.complete(null);
        return null;
      }

      final response = await _dio.post(
        '/auth/refresh',
        data: {'refreshToken': refreshToken},
        options: Options(
          extra: {'_retried': true},  // 재귀 방지
        ),
      );

      final newAccess = response.data['accessToken'] as String;
      final newRefresh = response.data['refreshToken'] as String;

      _accessToken = newAccess;
      await _storage.saveRefreshToken(newRefresh);

      completer.complete(newAccess);
      return newAccess;
    } catch (e) {
      completer.complete(null);
      // refresh 실패 → 토큰 폐기
      _accessToken = null;
      await _storage.clearRefreshToken();
      rethrow;
    } finally {
      _refreshCompleter = null;
    }
  }

  Future<Response<dynamic>> _retry(RequestOptions options, String newToken) {
    final newOptions = Options(
      method: options.method,
      headers: {...options.headers, 'Authorization': 'Bearer $newToken'},
      extra: {...options.extra, '_retried': true},
    );

    return _dio.request<dynamic>(
      options.path,
      data: options.data,
      queryParameters: options.queryParameters,
      options: newOptions,
    );
  }

  bool _isAuthExempt(String path) {
    return path.startsWith('/auth/login') ||
        path.startsWith('/auth/register') ||
        path.startsWith('/auth/refresh') ||
        path.startsWith('/auth/oauth/');
  }
}
```

| 로그아웃 시 메모리 토큰만 폐기 | secure storage도 clear + 백엔드 `/auth/logout` 호출 (Refresh blacklist — 04번 § 4.5) |

```dart
// features/auth/presentation/auth_providers.dart
@riverpod
class AuthState extends _$AuthState {
  @override
  Future<AuthStatus> build() async {
    // 앱 시작 시 secure storage의 Refresh 시도
    final storage = ref.read(tokenStorageProvider);
    final refresh = await storage.readRefreshToken();
    
    if (refresh == null) return AuthStatus.unauthenticated();

    try {
      // refresh로 access 발급 시도
      final api = ref.read(authApiProvider);
      final tokens = await api.refresh(refresh);
      
      ref.read(authInterceptorProvider).setAccessToken(tokens.accessToken);
      await storage.saveRefreshToken(tokens.refreshToken);

      // 본인 정보 조회
      final user = await api.me();
      return AuthStatus.authenticated(user);
    } catch (e) {
      await storage.clearRefreshToken();
      return AuthStatus.unauthenticated();
    }
  }

  Future<void> login(String email, String password) async {
    state = const AsyncLoading();
    try {
      final api = ref.read(authApiProvider);
      final tokens = await api.login(email, password);
      final storage = ref.read(tokenStorageProvider);

      ref.read(authInterceptorProvider).setAccessToken(tokens.accessToken);
      await storage.saveRefreshToken(tokens.refreshToken);

      final user = await api.me();
      state = AsyncData(AuthStatus.authenticated(user));
    } on DioException catch (e) {
      state = AsyncError(_mapAuthError(e), StackTrace.current);
    }
  }

  Future<void> logout() async {
    final storage = ref.read(tokenStorageProvider);
    final refresh = await storage.readRefreshToken();
    final api = ref.read(authApiProvider);

    // 백엔드 blacklist 등록 (best-effort — 실패해도 로컬은 폐기)
    if (refresh != null) {
      try {
        await api.logout(refresh);
      } catch (_) {}
    }

    await storage.clearRefreshToken();
    ref.read(authInterceptorProvider).setAccessToken(null);
    state = const AsyncData(AuthStatus.unauthenticated());
  }
}
```

## § 5. API 클라이언트 (DIO)

### 5.1 DIO 최상위 설정

| 항목 | 값 | 이유 |
| --- | --- | --- |
| `connectTimeout` | 5s | 서버 연결 대기 최대 |
| `receiveTimeout` | 30s (기본) / SSE·Long-poll 제외 | LLM 답변은 의외로 길 수 있음 |
| `sendTimeout` | 10s | 큰 요청 본문 (Journal `content` 등) |
| `headers.Content-Type` | `application/json` | 전역 기본 |
| `validateStatus` | 기본 (200~399 OK) | 4xx·5xx는 onError로 |

### 5.2 인터셉터 체이닝 (중요!)

```
Request 흐름:
  TraceInterceptor.onRequest    → traceparent 헤더 생성
  AuthInterceptor.onRequest     → Authorization 헤더 첨부
  ErrorMappingInterceptor.onRequest (no-op)
  PrettyDioLogger.onRequest     → 로그 출력

Response 흐름 (역순):
  PrettyDioLogger.onResponse
  ErrorMappingInterceptor.onResponse (no-op)
  AuthInterceptor.onResponse (no-op)
  TraceInterceptor.onResponse

Error 흐름 (역순):
  PrettyDioLogger.onError
  ErrorMappingInterceptor.onError  → ProblemDetail → AppException
  AuthInterceptor.onError          → 401 때 refresh + retry
  TraceInterceptor.onError
```

> **⚠️ 입력 순서 주의:** AuthInterceptor가 ErrorMappingInterceptor보다 먼저 `addInterceptor` 되어야 **onError 역순에서** ErrorMapping이 먼저 실행됨. 401 retry 이전에 ProblemDetail 변환이 끝난 상태.

### 5.3 TraceInterceptor (W3C Trace Context)

```dart
// core/network/trace_interceptor.dart
import 'dart:math';
import 'package:dio/dio.dart';

class TraceInterceptor extends Interceptor {
  static final _random = Random.secure();

  @override
  void onRequest(RequestOptions options, RequestInterceptorHandler handler) {
    // W3C traceparent: version-trace_id-span_id-flags
    final traceId = _hex(16);  // 32 hex chars
    final spanId = _hex(8);    // 16 hex chars
    options.headers['traceparent'] = '00-$traceId-$spanId-01';
    options.extra['traceId'] = traceId;
    handler.next(options);
  }

  String _hex(int bytes) {
    final buf = StringBuffer();
    for (var i = 0; i < bytes; i++) {
      buf.write(_random.nextInt(256).toRadixString(16).padLeft(2, '0'));
    }
    return buf.toString();
  }
}
```

> 이 traceId는 백엔드 Tempo에서 Flutter → Gateway → BFF → 4 service 전체 연결된 span tree로 보임. 사용자가 버그 신고 시 traceId만 안면 운영자가 전 구간 로그 조회 가능 (06번 § 12).

### 5.4 ErrorMappingInterceptor (ProblemDetail → AppException)

```dart
// core/network/error_mapper.dart
class ErrorMappingInterceptor extends Interceptor {
  @override
  void onError(DioException err, ErrorInterceptorHandler handler) {
    final response = err.response;
    if (response == null) {
      // 네트워크 자체 실패
      return handler.next(err.copyWith(
        error: NetworkException(message: err.message ?? '네트워크 오류'),
      ));
    }

    final contentType = response.headers.value('content-type') ?? '';
    if (!contentType.contains('application/problem+json')) {
      return handler.next(err);
    }

    try {
      final data = response.data as Map<String, dynamic>;
      final problem = ProblemDetail.fromJson(data);
      handler.next(err.copyWith(error: AppException.fromProblem(problem)));
    } catch (_) {
      handler.next(err);
    }
  }
}
```

### 6.1 SSE 자체 설명

**Server-Sent Events** (HTTP 단방향 스트림). 04번 § 6.3 표준에 따라 AI Service가 4 종류 이벤트 발행:

```

data: {"text":"창세"}

event: token
data: {"text":"기의 첫 구절은 "}
```

### 6.2 SSE 구현 (dio ResponseType.stream)

```dart
// features/ai_session/data/sse_turn_stream.dart
import 'dart:async';
import 'dart:convert';
import 'package:dio/dio.dart';

class SseEvent {
  final String event;
  final Map<String, dynamic> data;
  SseEvent({required this.event, required this.data});
}

class SseTurnStream {
  final Dio _dio;
  SseTurnStream(this._dio);

  Stream<SseEvent> open({
    required int sessionId,
    required String step,
    required String content,
  }) async* {
    final response = await _dio.post<ResponseBody>(
      '/ai/sessions/$sessionId/turns',
      data: {'step': step, 'content': content},
      options: Options(
        responseType: ResponseType.stream,
        receiveTimeout: const Duration(minutes: 5), // SSE는 일반 30s 초과
        headers: {'Accept': 'text/event-stream'},
      ),
    );

    final stream = response.data!.stream
        .transform(utf8.decoder)
        .transform(const LineSplitter());

    String? currentEvent;
    final buf = StringBuffer();

    await for (final line in stream) {
      if (line.isEmpty) {
        // 이벤트 경계 (빈 줄)
        if (currentEvent != null && buf.isNotEmpty) {
          final raw = buf.toString();
          if (raw == '[DONE]') {
            yield SseEvent(event: currentEvent, data: {'done': true});
            return;
          }
          try {
            final data = jsonDecode(raw) as Map<String, dynamic>;
            yield SseEvent(event: currentEvent, data: data);
          } catch (_) {
            // 잘못된 JSON — 무시
          }
        }
        currentEvent = null;
        buf.clear();
      } else if (line.startsWith('event:')) {
        currentEvent = line.substring(6).trim();
      } else if (line.startsWith('data:')) {
        if (buf.isNotEmpty) buf.write('\n');
        buf.write(line.substring(5).trim());
      }
      // 그 외 필드(예: id:, retry:)는 v1.0에 무시
    }
  }
}
```

```dart
@riverpod
class AiSessionTurns extends _$AiSessionTurns {
  @override
  AsyncValue<List<Turn>> build(int sessionId) {
    return const AsyncValue.data([]);
  }

  Future<void> sendUserTurn(String step, String content) async {
    // 1. 사용자 턴 즉시 추가 (optimistic)
    final userTurn = Turn.user(step: step, content: content);
    state = AsyncData([...state.value ?? [], userTurn]);

    // 2. AI 턴 placeholder
    final aiTurn = Turn.assistant(step: step, content: '', isStreaming: true);
    state = AsyncData([...state.value!, aiTurn]);

    final stream = ref.read(sseTurnStreamProvider).open(
          sessionId: sessionId,
          step: step,
          content: content,
        );

    final aiTurnIndex = state.value!.length - 1;
    final tokenBuf = StringBuffer();

    try {
      await for (final event in stream) {
        switch (event.event) {
          case 'token':
            tokenBuf.write(event.data['text'] as String);
            _updateAiTurn(aiTurnIndex, tokenBuf.toString(), null);
            break;
          case 'turn_completed':
            final ragSources = (event.data['ragSources'] as List?)
                ?.map((e) => RagSource.fromJson(e))
                .toList();
            _updateAiTurn(aiTurnIndex, tokenBuf.toString(), ragSources);
            break;
          case 'error':
            state = AsyncError(
              AppException(code: event.data['code'], message: event.data['message']),
              StackTrace.current,
            );
            return;
          case 'end':
            _finalizeAiTurn(aiTurnIndex);
            return;
        }
      }
    } catch (e, st) {
      state = AsyncError(e, st);
    }
  }

  void _updateAiTurn(int index, String content, List<RagSource>? sources) {
    final list = [...state.value!];
    list[index] = list[index].copyWith(content: content, ragSources: sources);
    state = AsyncData(list);
  }

  void _finalizeAiTurn(int index) {
    final list = [...state.value!];
    list[index] = list[index].copyWith(isStreaming: false);
    state = AsyncData(list);
  }
}
```

## § 7. STOMP WebSocket 클라이언트 (알림)

### 7.1 STOMP 연결 흐름

04번 § 10.2 + 05번 § 3.7 + ADR-0012에 정의된 표준:

```
Flutter → wss://qtai.local/api/v1/ws/notifications (Upgrade)
  Gateway 에서 패스스루 (검증 안 함)
Flutter → BFF (STOMP CONNECT 프레임)
  Headers:
    Authorization: Bearer <accessToken>
    accept-version: 1.2
BFF → STOMP CONNECTED + sub-protocol
Flutter → SUBSCRIBE /user/queue/notifications
  destination은 user-specific — BFF가 sub-protocol로 user_id 격리
BFF → MESSAGE (notification.requested Kafka 소비 시 해당 user만)
Flutter → (적절 시) DISCONNECT 또는 연결 끊김
```

### 7.2 stomp_dart_client 구현

```dart
// core/streaming/stomp_client.dart
import 'dart:async';
import 'dart:convert';
import 'package:stomp_dart_client/stomp_dart_client.dart';

class NotificationStompClient {
  StompClient? _client;
  String? _accessToken;
  final _notificationController = StreamController<AppNotification>.broadcast();
  Timer? _reconnectTimer;
  int _reconnectAttempt = 0;

  Stream<AppNotification> get notifications => _notificationController.stream;

  void setAccessToken(String? token) {
    _accessToken = token;
  }

  void connect() {
    if (_accessToken == null) return;

    _client = StompClient(
      config: StompConfig(
        url: '${Env.current.baseUrl.replaceFirst("http", "ws")}/ws/notifications',
        onConnect: _onConnect,
        onWebSocketError: (error) => _scheduleReconnect(),
        onStompError: (frame) => _handleStompError(frame),
        onDisconnect: (_) => _scheduleReconnect(),
        stompConnectHeaders: {
          'Authorization': 'Bearer $_accessToken',
          'accept-version': '1.2',
        },
        webSocketConnectHeaders: {
          'Authorization': 'Bearer $_accessToken',
        },
        heartbeatOutgoing: const Duration(seconds: 10),
        heartbeatIncoming: const Duration(seconds: 10),
      ),
    );

    _client!.activate();
  }

  void _onConnect(StompFrame frame) {
    _reconnectAttempt = 0;

    // user-specific destination — BFF가 user_id로 자동 매핑
    _client!.subscribe(
      destination: '/user/queue/notifications',
      callback: (frame) {
        if (frame.body == null) return;
        try {
          final data = jsonDecode(frame.body!) as Map<String, dynamic>;
          final notification = AppNotification.fromJson(data);
          _notificationController.add(notification);
        } catch (e) {
          // 잘못된 메시지 무시
        }
      },
    );
  }

  void _handleStompError(StompFrame frame) {
    if (frame.headers['message']?.contains('UNAUTHORIZED') ?? false) {
      // JWT 만료 → 다른 계층이 refresh 상태 처리
      disconnect();
      // RiverPod AuthState가 401 재로그인 유도
    } else {
      _scheduleReconnect();
    }
  }

  void _scheduleReconnect() {
    _reconnectTimer?.cancel();
    _reconnectAttempt++;
    // exponential backoff (max 30s)
    final delay = Duration(
      seconds: (1 << (_reconnectAttempt > 5 ? 5 : _reconnectAttempt)).clamp(1, 30),
    );
    _reconnectTimer = Timer(delay, () {
      if (_accessToken != null) connect();
    });
  }

  void disconnect() {
    _reconnectTimer?.cancel();
    _client?.deactivate();
    _client = null;
  }

  void dispose() {
    disconnect();
    _notificationController.close();
  }
}
```

### 7.5 1차 사고 패턴 매핑 (STOMP)

| 1차 또는 일반 사고 | 본 가이드 가드레일 |
| --- | --- |
| WS 연결 끊긴 건 모르고 알림 놓침 | exponential backoff 재연결 (§ 7.2) |
| 다른 user의 알림 받아보임 (보안 사고) | BFF가 user_id 격리 (ADR-0012) + Flutter는 destination으로 자기 것만 수신 |
| WS 이용해 서버로 명령 전송 | 본 v1.0에는 수신 전용. SEND 프레임 사용 X (BFF도 STOMP SEND 핸들러 안 설정) |
| WS 인증 토큰 만료 시 조용히 재접속 | onStompError → disconnect + AuthState가 재로그인 유도 (§ 7.2) |
| 서버 재시작으로 일괄 재연결 폭주 | exponential backoff (max 30s) 으로 서버 보호 |

### 7.6 W1 Lock-in 검증

| 검증 항목 | 방법 |
| --- | --- |
| STOMP CONNECT에 Bearer 토큰 전달 | wireshark 또는 BFF 로그에서 확인 |
| user-specific destination 구독 | 다른 user_id 계정으로 구독 시도 → 메시지 수신 안 됨 테스트 |
| 재연결 backoff 조정 동작 | BFF 강제 종료 후 Flutter 재연결 접속 로그 확인 |
| 모든 알림 type에 한국어 메시지 | switch 문에 4 케이스 모두 처리 (§ 7.4) |

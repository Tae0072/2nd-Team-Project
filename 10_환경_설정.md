## 1. 문서 목적과 경계

이 문서는 구현 저장소를 처음 받은 개발자가 로컬에서 백엔드, DB, 앱을 실행하기 위한 기준을 정리한다. 실제 명령은 구현 저장소 구조가 확정된 뒤 보강한다.

| 구분 | 이 문서에서 관리 | 다른 문서에서 관리 |
| --- | --- | --- |
| 필수 도구 | 포함 | 버전 업그레이드 결정은 `03` |
| 환경 변수 | 포함 | Secret 값 자체는 저장하지 않음 |
| 로컬 실행 순서 | 포함 | 배포 절차는 `14` |
| CI 실행 | 요약만 포함 | 상세 게이트는 `18` |

---

## 2. 필수 도구

| 도구 | 기준 | 용도 |
| --- | --- | --- |
| Java | 21 | Spring Boot 백엔드 |
| Gradle | Wrapper 사용 | 백엔드 빌드 |
| Spring Boot | 3.3.x | `qtai-server` |
| MySQL | 8.0 | 운영 기준 RDB |
| Docker / Docker Compose | Docker Compose 기준 | 로컬 DB와 배포 |
| Flutter | 구현 저장소 기준 | 모바일·관리자 앱 |
| Git | 최신 안정 버전 | 브랜치·PR 작업 |

Kubernetes, Helm, Kafka, Elasticsearch, ChromaDB, vector DB는 v1 로컬 환경에 포함하지 않는다.

---

## 3. 환경 변수 기준

| 변수 | 설명 | 로컬 예시 |
| --- | --- | --- |
| `SPRING_PROFILES_ACTIVE` | 실행 프로필 | `local` |
| `MYSQL_HOST` | MySQL host | `localhost` |
| `MYSQL_PORT` | MySQL port | `3306` |
| `MYSQL_DATABASE` | DB 이름 | `qtai` |
| `MYSQL_USER` | DB 사용자 | `qtai` |
| `MYSQL_PASSWORD` | DB 비밀번호 | `.env`에서 관리 |
| `JWT_SECRET` | JWT 서명 키 | 로컬 전용 더미 |
| `GOOGLE_CLIENT_ID` | Google OAuth client id | 개발용 |
| `GOOGLE_CLIENT_SECRET` | Google OAuth secret | `.env`에서 관리 |
| `AI_PROVIDER_API_KEY` | 배치 AI 호출 키 | 관리자·배치 전용 |

Secret 값은 문서, Git, 테스트 로그에 남기지 않는다.

## 4. 로컬 실행 순서

| 순서 | 작업 | 완료 기준 |
| --- | --- | --- |
| 1 | 저장소 clone | `dev` 기준 최신 코드 확보 |
| 2 | `.env.example` 복사 | `.env` 생성, Secret은 개인별 입력 |
| 3 | Docker Compose 실행 | MySQL 8.0 준비 |
| 4 | 백엔드 빌드 | `./gradlew build` 통과 |
| 5 | 백엔드 실행 | `qtai-server` local profile 실행 |
| 6 | Flutter 의존성 설치 | `flutter pub get` 통과 |
| 7 | Flutter 실행 | Today QT 화면 접근 가능 |

## 5. 로컬 확인 체크리스트

| 체크 | 기준 |

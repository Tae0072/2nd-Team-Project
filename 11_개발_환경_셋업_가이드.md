## 7. 데이터베이스 로컬 기동 (MySQL·Redis·Kafka·ChromaDB)

**방법 A (권장): Helm으로 Minikube 내부에 기동**

```bash
# Helm repo 추가
helm repo add bitnami https://charts.bitnami.com/bitnami
helm repo update

# MySQL (각 서비스별 별도 DB — 개발 편의상 1개 인스턴스 공유 가능)
helm install mysql bitnami/mysql -n qtai \
  --set auth.rootPassword=qtai_dev_root \
  --set auth.database=qtai_dev \
  --set auth.username=qtai_user \
  --set auth.password=qtai_dev_pass

# Redis
helm install redis bitnami/redis -n qtai \
  --set auth.enabled=false

# Kafka
helm install kafka bitnami/kafka -n qtai \
  --set replicaCount=1

# ChromaDB (강상민 담당)
helm install chromadb chromadb/chromadb -n qtai \
  --set chromadb.apiImpl=chromadb.api.segment.SegmentAPI
```

**방법 B: Docker Compose로 로컬 기동 (빠른 개발 시)**

```yaml
# docker-compose.dev.yml (프로젝트 루트)
version: '3.8'
services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: qtai_dev_root
      MYSQL_DATABASE: qtai_dev
      MYSQL_USER: qtai_user
      MYSQL_PASSWORD: qtai_dev_pass
    ports:
      - "3306:3306"

  redis:
    image: redis:7
    ports:
      - "6379:6379"

  kafka:
    image: confluentinc/cp-kafka:7.6.0
    environment:
      KAFKA_BROKER_ID: 1
      # ZooKeeper 불필요 — KRaft 모드 사용 (03번 기술 스택)
      KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://localhost:9092
    ports:
      - "9092:9092"
    depends_on:


  chromadb:
    image: chromadb/chroma:latest
    ports:
      - "8000:8000"
```

```bash
docker-compose -f docker-compose.dev.yml up -d
```

**.env.example 구조:**
```bash
# Database

MYSQL_PASSWORD=

# Redis

# Kafka

# Anthropic (강상민 발급 후 팀 공유)
ANTHROPIC_API_KEY=

# Google OAuth
GOOGLE_OAUTH_CLIENT_ID=
GOOGLE_OAUTH_CLIENT_SECRET=

# JWT
JWT_PRIVATE_KEY_PATH=./keys/private.pem
JWT_PUBLIC_KEY_PATH=./keys/public.pem

# ChromaDB
CHROMADB_HOST=localhost
CHROMADB_PORT=8000
```

### 10.2 JWT RS256 키 생성 (이지윤 담당)

```bash
# RSA 키 쌍 생성 (1회)
mkdir -p keys/
openssl genrsa -out keys/private.pem 2048
openssl rsa -in keys/private.pem -pubout -out keys/public.pem

# keys/ 디렉토리도 .gitignore에 추가
echo "keys/" >> .gitignore
```

생성 후 팀원에게 안전 채널(카카오톡 단톡 또는 직접 USB)로 공유. 이메일·Slack 공유 금지.

### 10.3 K8s Secret 설정

```bash
# Anthropic API key
kubectl create secret generic anthropic-secret \
  --from-literal=ANTHROPIC_API_KEY=sk-ant-xxxx \
  -n qtai

# MySQL 비밀번호
kubectl create secret generic mysql-secret \
  --from-literal=MYSQL_PASSWORD=qtai_dev_pass \
  -n qtai

# JWT 키
kubectl create secret generic jwt-keys \
  --from-file=private.pem=keys/private.pem \
  --from-file=public.pem=keys/public.pem \
  -n qtai

# 등록된 시크릿 확인
kubectl get secrets -n qtai
```

역할 핵심 선언
김태혁(DevB) — 시뮬레이터 단독 담당 (com.qtai.simulator 도메인):
① 본문 장면 시각화 시뮬레이터 — 본문 좌표(book·chapter·verse) 기반으로 장면을 애니메이션으로 시각화. 스프라이트 기반 검토 중(Flame / pixi.js / Lottie / 자체 sprite sheet 중 본인 선택).
② 데이터 모델·저장소·생성 이미지 라이선스 일임 — 본인이 정의 후 22_기능_명세서 FS-12에 박제.
③ 저장 정책 — 그날그날 생성 후 저장 → 재사용. 5년 단위 QT 본문 범위 갱신 시 폐기/재생성 (회의록 §3-3).
④ 난이도 — 회의록 §6에서 "가장 난이도 높음 — 일정 양보 허용"으로 명시. 시연 6/17에 안 되면 정적 일러스트로 대체 또는 후속 기능으로 발표 자료에 표기.
⚠️ 이전 AI/RAG 보조 범위는 강상민 단독 주도로 이관됨. RAG·ChromaDB·벡터 DB·엘라스틱서치는 ADR-0013으로 전부 폐기. 본인은 시뮬레이터 단독에 집중.
⚠️ 생성 이미지 라이선스 본인 확인 필수(DECISIONS §10) — (1) 생성 모델의 학습 데이터 출처(StabilityAI · OpenAI 등) (2) 결과물 상업/재배포 권리 (3) 종교 인물 묘사(예수·하나님·성령) 정책. W2 첫째 주 안 확인.

📝 무엇을 (WHAT)
SimulatorFacade.findScene() 구현 — 캐시 hit 시 simulator_scenes 행 반환, miss 시 lazy 생성 트리거. AssetStorage(Docker volume mount)에 자산 저장.
🛠 어떻게 (HOW)
@Service
public class SimulatorFacadeImpl implements SimulatorFacade {
    private final SceneRepository repo;
    private final SceneGenerator generator;
    private final AssetStorage storage;

    @Override
    public SceneResponse findScene(String book, int ch, int vs, int ve) {
        return repo.findActiveByPassage(book, ch, vs, ve)
            .map(SceneResponse::from)
            .orElseGet(() -> {
                var asset = generator.generate(book, ch, vs, ve);
                var path = storage.save(asset);
                var saved = repo.save(Scene.create(book, ch, vs, ve, path, ...));
                return SceneResponse.from(saved);
            });
    }
}

# SimulatorFacadeImpl — lazy 생성
@Service
public class SimulatorFacadeImpl implements SimulatorFacade {
    @Override
    public SceneResponse findScene(String book, int ch, int vs, int ve) {
        return repo.findActiveByPassage(book, ch, vs, ve)
            .map(SceneResponse::from)
            .orElseGet(() -> {
                var asset = generator.generate(book, ch, vs, ve);
                var path = storage.save(asset);
                return SceneResponse.from(repo.save(Scene.create(book, ch, vs, ve, path, ...)));
            });
    }
}

5.3글로벌 익셉션 핸들러 + ProblemDetail (04번 §2.4) 
// journal-service/.../common/exception/GlobalExceptionHandler.java
@RestControllerAdvice
@Slf4j
public class GlobalExceptionHandler {

    @ExceptionHandler(BusinessException.class)
    public ProblemDetail handleBusiness(BusinessException e, HttpServletRequest req) {
        ErrorCode err = e.getError();
        ProblemDetail pd = ProblemDetail.forStatus(err.getHttpStatus());
        pd.setType(URI.create("https://api.qtai.app/errors/" + err.getCode().toLowerCase().replace('_','-')));
        pd.setTitle(err.getMessage());
        pd.setInstance(URI.create(req.getRequestURI()));
        pd.setProperty("code",      err.getCode());
        pd.setProperty("traceId",   MDC.get("traceId"));
        pd.setProperty("timestamp", Instant.now().toString());
        return pd;
    }

    // PESSIMISTIC_WRITE Lock timeout → JOURNAL_LOCKED
    @ExceptionHandler(PessimisticLockingFailureException.class)
    public ProblemDetail handleLockTimeout(PessimisticLockingFailureException e) {
        ProblemDetail pd = ProblemDetail.forStatus(HttpStatus.CONFLICT);
        pd.setProperty("code",     "JOURNAL_LOCKED");
        pd.setProperty("traceId",  MDC.get("traceId"));
        return pd;
    }
}

5.5Kafka 컨슈머 — ai.session.completed (Saga + 멱등성)
⚠️ 멱등성 필수: PROCESSED_EVENTS 테이블에 idempotencyKey 기록. @DltHandler에서 journal.creation.failed 발행 → AI Service가 컨슈밍해 세션 롤백.
// journal-service/.../infrastructure/kafka/consumer/AiSessionCompletedConsumer.java
@Component
@RequiredArgsConstructor
@Slf4j
public class AiSessionCompletedConsumer {

    private final CreateDraftFromSessionUseCase useCase;
    private final ProcessedEventRepository       processedRepo;
    private final KafkaTemplate  kafkaTemplate;

    @KafkaListener(
        topics            = "ai.session.completed",
        groupId           = "journal-service-ai-completion-consumer",
        containerFactory  = "kafkaListenerContainerFactory"
    )
    @Transactional
    public void consume(AiSessionCompletedEvent event) {
        String idempotencyKey = event.idempotencyKey();   // ai.session.completed:{sessionId}

        // 멱등성 체크
        if (processedRepo.exists(idempotencyKey)) {
            log.info("Already processed: {}", idempotencyKey);
            return;
        }

        // DRAFT Journal + JOURNAL_EVENTS (CREATED) 적재
        useCase.execute(event);

        // 멱등성 기록
        processedRepo.save(idempotencyKey);
    }

    // ── DLT (Dead Letter Topic) 핸들러 — Saga 보상 ──
    @DltHandler
    public void handleDlt(
        AiSessionCompletedEvent event,
        @Header(KafkaHeaders.EXCEPTION_MESSAGE) String error
    ) {
        log.error("DLT: AI session {} — {}", event.data().sessionId(), error);

        // journal.creation.failed 발행 → AI Service가 세션 롤백
        JournalCreationFailedEvent failed = new JournalCreationFailedEvent(
            UUID.randomUUID().toString(),
            "journal.creation.failed", 1,
            "journal.creation.failed-value",
            Instant.now().toString(),
            MDC.get("traceId"),
            "journal-service",
            "journal.creation.failed:" + event.data().sessionId(),
            new JournalCreationFailedEvent.Data(
                event.data().sessionId(),
                event.data().userId(),
                error,
                3   // 재시도 횟수
            )
        );
        kafkaTemplate.send("journal.creation.failed", failed.idempotencyKey(), failed);
    }
}

5.6Outbox 패턴 (Transactional Outbox)
⚠️ 이벤트 소싱에서 TX 내 KafkaTemplate 직접 호출 금지. JOURNAL_EVENTS가 outbox 역할 — AFTER_COMMIT 스케줄러가 발송.
// journal-service/.../infrastructure/outbox/OutboxPublisher.java
@Component
@RequiredArgsConstructor
@Slf4j
public class OutboxPublisher {

    private final OutboxRepository              outboxRepository;
    private final KafkaTemplate kafkaTemplate;

    /** AFTER_COMMIT 직후 호출 (TransactionPhase.AFTER_COMMIT) */
    @TransactionalEventListener(phase = TransactionPhase.AFTER_COMMIT)
    public void onJournalEvent(JournalDomainEvent event) {
        Object payload = event.toKafkaPayload();
        kafkaTemplate.send(event.getTopic(), event.getIdempotencyKey(), payload);
    }

    /** 스케줄러 — 높은 신뢰성이 필요한 경우 outbox 재발송 (5초마다) */
    @Scheduled(fixedDelay = 5000)
    @Transactional
    public void publishPendingEvents() {
        outboxRepository.findUnpublished(50).forEach(event ->
            kafkaTemplate.send(event.getTopic(), event.getIdempotencyKey(), event.getPayload())
        );
    }
}

🚫 폴더 격리 규칙 (AGENTS.md §13)
• 본인 폴더(workspaces/DevD_이승욱/)에서만 읽기·쓰기 허용
• 다른 팀원의 개인 폴더는 읽기·수정·삭제 모두 금지 (AI 에이전트 포함)
• workspaces/ 내 파일은 빌드·테스트·CI에 영향 없음 (import 금지)
• 다른 팀원 폴더 변경이 PR에 포함되면 자동 reject

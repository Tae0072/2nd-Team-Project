② PR 올리는 7단계 (반드시 이 순서로)상세 내용 열기/닫기
⌄
1
develop 최신 pull
git checkout develop && git pull origin develop
2
feature 브랜치 생성 (develop 기준)
git checkout -b feature/auth-jwt-filter
3
작업 후 커밋 — type(scope): 설명 형식 필수
git add .
git commit -m "feat(auth): add JWT RS256 filter with X-User-Id injection"
# type: feat | fix | refactor | test | docs | chore | ci
# scope: auth | bible | ai | journal | bff | gateway
4
develop rebase — 이 단계 빠뜨리면 머지 거부
git fetch origin
git rebase origin/develop
# 충돌 시: 해결 후 git rebase --continue
5
로컬 CI 확인 — push 전 반드시 통과
./gradlew :auth-service:test          # 자기 서비스 테스트
gitleaks detect --source . --verbose  # 시크릿 스캔
npx spectral lint apis/auth/openapi.yaml  # OpenAPI 수정 시
6
Push + GitHub PR 생성
git push origin feature/auth-jwt-filter
# GitHub → Compare & pull request 클릭
# base: develop ← compare: feature/auth-jwt-filter
# → PR 템플릿 자동 로드 (아래 ③ 참고)
7

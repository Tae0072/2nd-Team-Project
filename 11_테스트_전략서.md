## 2. 테스트 레벨

| 레벨 | 대상 | 필수 기준 |
| --- | --- | --- |
| Unit | 도메인 로직, DTO 변환, 상태 판단 | 핵심 분기 테스트 |
| Integration | Controller, DB, 도메인 Port | 권한, 트랜잭션, 실패 처리 |
| Contract | OpenAPI, 응답 DTO | `/api/v1` prefix, 권한, 상태값 |
| Architecture | 도메인 import 규칙 | Entity/Service/Repository 직접 import 차단 |
| E2E | 핵심 사용자 흐름 | Today QT, 묵상, 관리자 재생성 |
| Flutter | 화면 상태와 위젯 | PUBLIC/USER 상태, 버튼 활성화 |

## 3. 핵심 테스트 시나리오

| ID | 시나리오 | 통과 기준 |
| --- | --- | --- |
| T-01 | 비로그인 사용자가 Today QT 진입 | 본문과 진입점 상태 조회 가능 |
| T-02 | 해설 보기 진입 | 비로그인은 로그인 유도, 로그인 후 C 해설 표시 |
| T-03 | 묵상 작성·수정 | 오늘 QT 기준 기록 저장, 이벤트 기록 |
| T-04 | 묵상 달력 조회 | 묵상한 날짜가 정확히 표시 |
| T-05 | 시뮬레이터 상태 | `READY`만 버튼 활성화 |
| T-06 | 찬양 큐레이션 | PUBLIC 조회, USER 저장 |
| T-07 | 관리자 AI 재생성 | `ADMIN`만 가능, 감사 로그 기록 |
| T-08 | 금지 기능 검사 | AI Q&A, SSE, 교회 인증, AI 찬양 추천 없음 |
| T-09 | 도메인 경계 검사 | 직접 import 위반 시 실패 |

## 1. 클린 아키텍처 계층 원칙

### 1.1 4 계층 규칙

```
Presentation (Controller, DTO)
    ↓ (의존)
Application (UseCase, Service)
    ↓
Domain (Entity, Repository interface, Domain Exception)
    ↑ (구현)
Infrastructure (JPA, Redis, Kafka, REST Client)
```

**핵심 규칙:**

| 계층 | 허용 의존 | 금지 의존 |
| --- | --- | --- |
| Domain | 없음 (순수 Kotlin) | Spring, JPA, Kafka 등 모든 프레임워크 |
| Application | Domain | Spring Web, JPA 직접 |
| Presentation | Application | Domain Entity 직접 노출 |
| Infrastructure | Domain (interface 구현) | Application 직접 |

**1차 사고 재발 방지:**
```kotlin
// ❌ 1차 사고 패턴 — Controller에 비즈니스 로직
@PostMapping("/login")
fun login(@RequestBody req: LoginRequest): ResponseEntity<TokenResponse> {
    val user = userRepository.findByEmail(req.email) ?: throw Exception("not found")
    if (!passwordEncoder.matches(req.password, user.passwordHash)) throw Exception("wrong")
    val token = jwtService.generateToken(user.id)
    return ResponseEntity.ok(TokenResponse(token))
}

// ✅ 2차 표준 — UseCase로 위임
@PostMapping("/login")
fun login(@RequestBody req: LoginRequest): ResponseEntity<TokenResponse> =
    loginUseCase.execute(LoginCommand(req.email, req.password))
        .let { ResponseEntity.ok(it.toResponse()) }
```

### 2.2 RefreshToken 도메인 엔티티

```kotlin
// domain/entity/RefreshToken.java
data class RefreshToken(
    val id: Long,
    val userId: Long,
    val tokenHash: String,        // 평문 저장 X — SHA-256 hash
    val expiresAt: LocalDateTime, // 14일
    val revokedAt: LocalDateTime?,
    val revokeReason: RevokeReason?,
    val isUsed: Boolean,          // Rotation: 사용 시 true + 새 token 발급
    val createdAt: LocalDateTime,
) {
    fun isValid(): Boolean =
        revokedAt == null && !isUsed && expiresAt.isAfter(LocalDateTime.now())

    fun revoke(reason: RevokeReason): RefreshToken = copy(
        revokedAt = LocalDateTime.now(),
        revokeReason = reason,
    )

    fun markUsed(): RefreshToken = copy(isUsed = true)
}

enum class RevokeReason { ROTATION, LOGOUT, REUSE_DETECTED, ADMIN }
```

### 2.4 로그인 비즈니스 규칙

```kotlin
// application/usecase/LoginUseCase.java
class LoginUseCase(
    private val userRepository: UserRepository,
    private val refreshTokenRepository: RefreshTokenRepository,
    private val passwordEncoder: PasswordEncoder,
    private val jwtService: JwtService,
    private val redisLockService: RedisLockService,
) {
    fun execute(command: LoginCommand): TokenPair {
        val user = userRepository.findByEmail(command.email)
            ?: throw UserNotFoundException()

        if (!user.isActive()) throw UserDeactivatedException()

        if (!passwordEncoder.matches(command.password, user.passwordHash))
            throw InvalidCredentialsException()

        // Login rate limit: 1회/5초 (05번 § 3.1)
        val lockKey = "login:rate:${user.id}"
        redisLockService.acquireOrThrow(lockKey, ttl = Duration.ofSeconds(5)) {
            throw LoginRateLimitException()
        }

        val accessToken = jwtService.generateAccessToken(user)
        val refreshToken = jwtService.generateRefreshToken(user)
        refreshTokenRepository.save(refreshToken)

        return TokenPair(accessToken, refreshToken.plainToken)
    }
}
```

## 4. Bible Service 묵상 도메인 모델 (이벤트 소싱)

> **⚠️ v1.2 핵심 변경:** D형 QT 방법론(23_도메인_용어사전 참조) 반영으로 단일 `content` 필드를 7개 분리 필드로 재구조화. `JournalStep`(혼재 enum) → `JournalType`(QT 유형)으로 명확화.

### 4.1 JournalType — QT 유형 (A/B/C/D)

```kotlin
// domain/entity/JournalType.java
enum class JournalType(val displayName: String, val description: String) {
    A("느낌만",       "QT 입문자. 본문을 읽고 마음에 와닿은 것만 기록"),
    B("내용관찰+느낌", "본문 3회+ 읽고 자기 말로 풀어 쓴 후 느낌 기록"),
    C("B+적용",       "B형에 구체적 행동 결단(적용) 추가"),
    D("C+해석",       "C형에 연구와 묵상(해석 단계) 추가. 귀납적 성경공부 수준");

    fun requiresObservation(): Boolean = this in setOf(B, C, D)
    fun requiresInterpretation(): Boolean = this == D
    fun requiresApplication(): Boolean = this in setOf(C, D)
    fun requiresFelt(): Boolean = true  // 모든 유형이 느낌은 필수
}
```

> **DB 매핑:** `JOURNALS.qt_type VARCHAR(2)` — 값 `'A'`, `'B'`, `'C'`, `'D'`

### 4.2 Journal 도메인 엔티티 (이벤트 소싱)

```kotlin
// domain/entity/Journal.java
// ⚠️ Journal은 읽기 모델 (Projection) — 직접 수정 X, 이벤트로만 변경
// ⚠️ v1.2: 단일 content 필드 → D형 QT 4단계에 맞춘 7개 필드로 분리
data class Journal(
    val id: Long,
    val userId: Long,             // 외부 ID (FK X — 03번 ADR-0005)
    val title: String,            // 자동 생성: '{name_kr} {chapter}:{verse} 묵상'
    val bookId: Long,
    val chapter: Int,
    val verse: Int,
    val sessionId: Long?,         // AI 세션 연결 (nullable — 수동 생성 시 null)

    // === D형 QT 4단계 필드 (23번 § 3 참조) ===
    val qtType: JournalType,      // ⭐ v1.2 신규 — A/B/C/D 유형
    val observation: String?,     // ⭐ v1.2 신규 — 1단계 관찰 (단락나누기·재진술·5W1H)
    val interpretation: String?,  // ⭐ v1.2 신규 — 2단계 해석/연구와묵상 (하나님은 어떤 분?)
    val felt: String?,            // 3단계 느낌 (도전·반성·회개·감사)
    val memorableVerse: String?,  // 기억할 구절
    val application: String?,     // 4단계 적용 (구체적·측정 가능한 행동 결단)
    val prayer: String?,          // 마무리 기도

    val status: JournalStatus,
    val visibility: Visibility,
    val isAutoCreated: Boolean,
    val lastEventSequence: Long,  // 이벤트 소싱 시퀀스 카운터 (02번 § 5.4)
    val createdAt: LocalDateTime,
    val updatedAt: LocalDateTime,
    val deletedAt: LocalDateTime?,  // 소프트 딜리트 (ADR-0006)
) {
    fun isOwnedBy(userId: Long): Boolean = this.userId == userId
    fun isDeleted(): Boolean = deletedAt != null
    fun canEdit(): Boolean = !isDeleted() && status != JournalStatus.DELETED

    /** D형 QT 유형별 필수 필드 검증 (PUBLISHED 전환 시) */
    fun validateForPublish() {
        require(felt?.isNotBlank() == true) { "느낌은 모든 QT 유형에 필수입니다" }
        if (qtType.requiresObservation()) {
            require(observation?.isNotBlank() == true) { "${qtType.displayName} 유형은 관찰이 필수입니다" }
        }
        if (qtType.requiresInterpretation()) {
            require(interpretation?.isNotBlank() == true) { "D형은 해석/연구와묵상이 필수입니다" }
        }
        if (qtType.requiresApplication()) {
            require(application?.isNotBlank() == true) { "${qtType.displayName} 유형은 적용이 필수입니다" }
        }
    }
}

enum class JournalStatus { DRAFT, PUBLISHED, DELETED }
enum class Visibility { PRIVATE, ANONYMOUS_SHARED }
```

> **레거시 호환:** v1.1까지의 단일 `content: String` 필드는 v1.2에서 폐기. Flyway V4 마이그레이션 시 기존 `content` 컬럼 데이터는 `felt`로 이관(QT 입문자 가정 = A형 default).

```kotlin
// application/service/JournalAutoCreateService.java
@Transactional
fun createFromAiSession(event: AiSessionCompletedEvent) {
    // 멱등성 — 같은 sessionId로 중복 생성 방지 (ADR-0007)
    val idempotencyKey = "ai.session.completed:${event.sessionId}"
    if (journalEventRepository.existsByIdempotencyKey(idempotencyKey)) {
        log.info("Duplicate event skipped: $idempotencyKey")
        return

    val journal = Journal(

        title = generateTitle(event),  // '{name_kr} {chapter}:{verse} 묵상'
        bookId = event.bookId,
        chapter = event.chapter,
        verse = event.verse,
        sessionId = event.sessionId,
        qtType = qtType,
        observation = null,    // AI 대화 요약에서 추출 가능 (v1.1)
        interpretation = null, // AI 대화 요약에서 추출 가능 (v1.1)
        felt = event.summary,  // v1.0은 summary를 felt에 적재 (입문자 친화)
        memorableVerse = null,
        application = null,
        prayer = null,
        status = JournalStatus.DRAFT,
        visibility = Visibility.PRIVATE,
        isAutoCreated = true,
        lastEventSequence = 0,
        // ... 기타 필드
    )

    journalRepository.save(journal)
    publishEvent(JournalCreatedEvent(journal.id, journal.userId), idempotencyKey)
}
```

> **v1.1 개선 후보:** AI Service가 `ai.session.completed` 이벤트에 `observation`/`interpretation`/`felt`/`application` 4분할 요약을 별도 필드로 담아 발행하면, Bible Service가 각 필드에 직접 적재 가능. 현 v1.0은 단순 summary만 사용.

## 5. AI Service 도메인 모델

### 5.1 SessionStep vs QtStage — 의미 구분 (⚠️ 혼동 주의)

> **23번 § 8.2 정합:** A~D는 **QT 유형(Type)**, 관찰→해석→느낌→적용은 **QT 진행 단계(Stage)**. 둘은 다른 차원.

```kotlin
// domain/entity/SessionStep.java
// QT 유형: 사용자가 선택한 묵상 깊이 수준 (A/B/C/D)
// DB 매핑: AI_SESSIONS.guide_step, AI_TURNS.step
enum class SessionStep { A, B, C, D }

// domain/entity/QtStage.java
// QT 내부 진행 단계: AI가 현재 안내 중인 4단계
// DB 매핑 없음 — AI 세션 컨텍스트(메모리/Redis)로만 관리
enum class QtStage(val displayName: String, val korean: String) {
    OBSERVATION("Observation", "관찰"),       // 1단계: 본문 사실 파악
    INTERPRETATION("Interpretation", "해석"), // 2단계: 연구와 묵상 (D형만)
    FEELING("Feeling", "느낌"),               // 3단계: 마음의 반응
    APPLICATION("Application", "적용");        // 4단계: 행동 결단

    fun next(): QtStage? = when (this) {
        OBSERVATION -> INTERPRETATION
        INTERPRETATION -> FEELING
        FEELING -> APPLICATION
        APPLICATION -> null  // 마지막 단계
    }

    /** QT 유형별 해당 단계가 포함되는지 */
    fun isIncludedIn(type: JournalType): Boolean = when (this) {
        OBSERVATION -> type in setOf(JournalType.B, JournalType.C, JournalType.D)
        INTERPRETATION -> type == JournalType.D
        FEELING -> true  // 모든 유형 포함
        APPLICATION -> type in setOf(JournalType.C, JournalType.D)
    }
}
```

### 5.2 AiSession 도메인 엔티티

```kotlin
// domain/entity/AiSession.java
data class AiSession(
    val id: Long,
    val userId: Long,
    val bookId: Long,
    val chapter: Int,
    val verse: Int,
    val guideStep: SessionStep,        // ⭐ A/B/C/D — 사용자가 선택한 QT 유형
    val currentStage: QtStage,         // ⭐ v1.2 신규 — 관찰/해석/느낌/적용 (세션 컨텍스트)
    val status: SessionStatus,
    val summary: String?,              // 완료 시 생성 요약 (02번 v1.1)
    val startedAt: LocalDateTime,
    val endedAt: LocalDateTime?,
    val createdAt: LocalDateTime,
    val abandonReason: AbandonReason?,
) {
    fun canAdvanceTo(nextStage: QtStage): Boolean {
        // D형이 아니면 INTERPRETATION 단계 건너뜀
        if (nextStage == QtStage.INTERPRETATION && guideStep != SessionStep.D) {
            return false
        }
        return currentStage.next() == nextStage && nextStage.isIncludedIn(toJournalType())
    }

    fun isActive(): Boolean = status == SessionStatus.IN_PROGRESS

    fun advance(nextStage: QtStage): AiSession {
        require(canAdvanceTo(nextStage)) {
            "Cannot advance from $currentStage to $nextStage (qtType=$guideStep)"
        }
        return copy(currentStage = nextStage)
    }

    fun complete(summary: String): AiSession = copy(
        status = SessionStatus.COMPLETED,
        summary = summary,
        endedAt = LocalDateTime.now(),
    )

    fun abandon(reason: AbandonReason): AiSession = copy(
        status = SessionStatus.ABANDONED,
        abandonReason = reason,
        endedAt = LocalDateTime.now(),
    )

    private fun toJournalType(): JournalType = JournalType.valueOf(guideStep.name)
}

enum class SessionStatus { IN_PROGRESS, COMPLETED, COMPLETED_NO_JOURNAL, ABANDONED }
enum class AbandonReason { USER_REQUEST, SELF_HARM_DETECTED, TIMEOUT, ERROR }
```

```kotlin
// application/usecase/CreateJournalUseCase.java
@Component
@Transactional  // ← 모든 쓰기 UseCase에 필수 (1차 사고 재발 방지)
class CreateJournalUseCase(
    private val journalRepository: JournalRepository,
    private val journalEventStore: JournalEventStore,
    private val eventPublisher: DomainEventPublisher,
) : UseCase<CreateJournalCommand, Journal> {

    override fun execute(input: CreateJournalCommand): Journal {
        // 1. 도메인 유효성 검증
        require(input.title.isNotBlank()) { "Title must not be blank" }

        // 2. 도메인 생성 (D형 QT 필드는 초기 모두 null — 사용자가 단계별 작성)
        val journal = Journal(
            userId = input.userId,
            title = input.title,
            bookId = input.bookId,
            chapter = input.chapter,
            verse = input.verse,
            sessionId = input.sessionId,
            qtType = input.qtType,
            observation = null,
            interpretation = null,
            felt = null,
            memorableVerse = null,
            application = null,
            prayer = null,
            status = JournalStatus.DRAFT,
            visibility = Visibility.PRIVATE,
            isAutoCreated = false,
            lastEventSequence = 0,
            // ... 시간 필드
        )

        // 3. 저장 + 이벤트 적재 (SELECT FOR UPDATE — 02번 § 5.4)
        val saved = journalRepository.save(journal)
        journalEventStore.append(
            saved.id,
            JournalEventType.CREATED,
            createdEventPayload(saved),
            idempotencyKey = "manual:${saved.id}:${System.currentTimeMillis()}",
        )

        // 4. 이벤트 발행 (AFTER_COMMIT — 03번 § 2.3)
        eventPublisher.publish(JournalCreatedEvent(saved.id, saved.userId))

        return saved
    }
}
```

```kotlin
@Service

class JournalQueryService(
    private val journalRepository: JournalRepository,
) {
    fun findById(id: Long, userId: Long): Journal {
        val journal = journalRepository.findById(id)
            ?: throw JournalNotFoundException(id)
        if (!journal.isOwnedBy(userId)) throw ForbiddenException()
        return journal
    }

    fun listByUser(userId: Long, qtType: JournalType?, page: Int, size: Int): Page<Journal> {
        return if (qtType != null) {
            journalRepository.findByUserIdAndQtType(userId, qtType, page, size)
        } else {
            journalRepository.findByUserId(userId, page, size)
        }
    }
}
```

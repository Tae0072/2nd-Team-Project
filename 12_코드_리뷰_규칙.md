## 1. 문서 목적과 경계

이 문서는 리뷰어가 PR에서 확인할 기준을 별도로 정리한다. 브랜치, 커밋, PR 생성 방식은 `09_Git_규칙.md`가 책임진다.

---

## 2. 리뷰 필수 관점

| 관점 | 확인 항목 |
| --- | --- |
| 요구사항 | `07_요구사항_정의서.md`의 F-ID와 충돌하지 않는가 |
| 아키텍처 | 도메인 간 Entity/Service/Repository 직접 import가 없는가 |
| API | 외부 공개 API가 `/api/v1/**`와 권한 기준을 따르는가 |
| 데이터 | 금지 번역본, A 테이블 노출, 저작권 위험 표현이 없는가 |
| AI | 사용자 요청 경로 LLM 호출, AI Q&A, SSE가 없는가 |
| 화면 | 비로그인·로그인·관리자 상태가 `06` 기준과 맞는가 |
| 테스트 | 변경 범위에 맞는 단위·통합·E2E 테스트가 있는가 |

---

## 3. 머지 차단 기준

| 차단 항목 | 예시 |
| --- | --- |
| 요구사항 역행 | 교회 인증 버튼 추가, AI 찬양 추천 추가 |
| 금지 기술 도입 | RAG, ChromaDB, vector DB, Elasticsearch, Kafka, Kubernetes, Helm |
| 사용자 AI API | `/ai/sessions/**`, 챗봇, SSE |
| 도메인 경계 위반 | 다른 도메인의 내부 Entity 직접 import |
| 보안 위험 | Secret 커밋, 관리자 API 권한 누락 |
| 저작권 위험 | 개역개정, ESV, NIV 적재 또는 원문 주석 노출 |

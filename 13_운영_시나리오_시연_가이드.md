
## 3. 시연 시나리오 — 3분 스크립트

> **[조작자]** 표시는 김지민이 에뮬레이터에서 수행. **[강태오]** 는 설명 멘트.

### 3.1 파트 1 — 앱 시작 + 로그인 (30초)

```
[강태오] "지금부터 QT-AI 앱 시연을 시작하겠습니다.
  QT-AI는 AI가 코칭하는 모바일 성경 묵상 앱입니다."

[조작자] 에뮬레이터에서 앱 아이콘 탭
[화면] 스플래시 → 로그인 화면

[강태오] "Google 계정으로 간편하게 로그인합니다."

[조작자] Google 로그인 버튼 탭 → 계정 선택
[화면] 대시보드 진입 (2~3초)

[강태오] "대시보드에 오늘의 성경 구절과 최근 묵상 노트가 표시됩니다."
```

### 3.2 파트 2 — 입체 묵상 화면 (20초)

```
[조작자] 오늘의 구절 카드(창세기 1:1) 탭
[화면] 입체 묵상 화면 — 본문 텍스트 + 주석 + AI 버튼

[강태오] "성경 본문과 주석을 함께 볼 수 있는 입체 묵상 화면입니다.
  여기서 AI 큐티를 시작합니다."

[조작자] "AI 큐티 시작" 버튼 탭
[화면] AI 세션 화면 진입 + A 단계 안내 메시지 표시
```

### 3.3 파트 3 — AI 큐티 4단계 (2분)

```
[강태오] "AI가 귀납적 성경 묵상의 4단계, 관찰·해석·적용·결단을 코칭합니다.
  지금은 관찰 단계입니다."

─── A 단계 (관찰) ───
[조작자] 입력창에 "이 구절에서 누가 등장하나요?" 타이핑 후 전송
[화면] AI 응답 스트리밍 (글자 단위로 나타남)
  → "본문에서 가장 먼저 등장하는 분은 누구신가요? 함께 찾아보시겠어요?"

[강태오] "보시다시피 AI가 답을 직접 주지 않고 사용자가 스스로 발견하도록
  유도합니다. 이것이 코칭형 AI의 특징입니다."

[조작자] "하나님이요" 입력 전송
[화면] AI 응답 + "해석 단계로 넘어갈까요?" 제안

─── B 단계 (해석) ───
[조작자] "네" 탭 → B 단계 진입
[조작자] "'창조하시니라'는 무슨 의미인가요?" 입력

[강태오] "해석 단계에서는 RAG 기술로 신학 주석 문서를 검색해서
  근거 있는 답변을 제공합니다."

[화면] AI 응답 스트리밍 + 주석 출처 표시 [doc-001]
  → "히브리어 'bara'는 무에서의 창조를 뜻합니다 [doc-001].
     당신은 이 창조의 의미가 오늘 삶에서 어떻게 연결될까요?"

─── C 단계 (적용) — 빠르게 ───
[조작자] C 단계 진입 후 간단 응답 입력

─── D 단계 (결단) ───
[조작자] "오늘 저녁 감사 기도 10분 하겠습니다" 입력
[화면] AI 응답: "구체적인 결단이네요. 묵상 노트에 기록할게요."
  → SSE end 이벤트 → 세션 완료
```

### 3.4 파트 4 — 자동 노트 생성 + 알림 (30초)

```
[화면] 3초 후 STOMP 알림 도착
  → SnackBar: "묵상 노트가 자동으로 생성되었습니다"

[강태오] "AI 큐티가 완료되면 Kafka 메시지 브로커를 통해
  Journal Service가 자동으로 묵상 노트를 생성합니다."

[조작자] SnackBar "보기" 탭
[화면] 자동 생성된 묵상 노트 상세 화면

[강태오] "이것이 QT-AI의 핵심 — AI 코칭부터 묵상 노트 자동 저장까지
  완전한 디지털 큐티 경험입니다. 감사합니다."
```

## 4. 안전 fallback 시나리오

### 4.1 fallback 트리

```
시연 중 오류 발생
  ├─ Anthropic API 지연 (5초+ 응답 없음)
  │    → 강태오: "네트워크 지연이 있네요. 미리 녹화한 시연 영상으로 대체합니다."
  │    → fallback 영상 재생 (바탕화면 demo_fallback_qtai.mp4)
  │
  ├─ 에뮬레이터 앱 크래시
  │    → 김지민: 앱 재실행 (5초 대기)
  │    → 재실행 실패 시 → 영상 fallback
  │
  ├─ Minikube pod 다운
  │    → 강태오: 설명 멘트로 시간 벌기 (30초)
  │    → kubectl rollout restart 시도
  │    → 실패 시 → 영상 fallback
  │
  └─ 프로젝터 연결 실패
       → 노트북 화면 직접 시연
       → 또는 슬라이드로 전환 + 핵심 흐름 설명
```

### 4.2 fallback 대사 (강태오 암기)

```
"현재 실시간 AI 서비스 응답에 약간의 지연이 있습니다.
 미리 준비한 시연 영상으로 동일한 기능을 보여드리겠습니다.
 실제 서비스와 100% 동일한 흐름입니다."
```

### 4.3 fallback 영상 제작 (W4 6/9~12)

```
담당: 김지민 (영상 녹화) + 강태오 (편집·자막)
도구: Windows 게임바(Win+G) 또는 OBS Studio
형식: MP4, 1920x1080, 3분 이내
저장: 발표 PC 바탕화면 + USB 백업
```

## 6. 예상 Q&A + 답변

| 질문 | 답변 담당 | 핵심 답변 |
| --- | --- | --- |
| AI가 잘못된 신학을 말하면? | 강상민 | 신학 가드레일 9종 + LLM-as-judge 실시간 검증. 회피 영역 감지 시 즉시 차단 + 정통 범위 내 응답으로 유도 |
| 개인정보(묵상 내용) 보안은? | 이지윤 | JWT RS256 + TLS + K8s NetworkPolicy default deny. 묵상 내용은 AI Service에서 처리 후 저장 안 됨 (turn은 턴 단위 저장, 세션 종료 시 요약만) |
| MSA가 너무 복잡하지 않나? | 이승욱 | 6주 일정에 MSA가 도전적임을 인정. 그러나 서비스별 독립 배포·장애 격리·팀 역할 분담 명확화가 실익. 1차 모놀리식 대비 배운 교훈 |
| Kafka를 왜 사용했나? | 이승욱 | AI 세션 완료 → Journal 자동 생성이 비동기 처리 최적. Saga 패턴으로 분산 트랜잭션 구현. 동기 REST 호출이면 Journal 실패가 AI 응답을 블로킹 |
| RAG 문서 저작권은? | 강상민 | W1.0 시연용은 public domain 자료 5~50개만 사용. 상용 출시 시 CC BY 라이선스 자료 + 출판사 계약 필요 (v1.1 계획) |
| iOS는? | 강태오 | v1.0은 Android 우선. iOS는 Apple Developer Program($99/yr) + App Store Review 기간(평균 2~7일) 때문에 6주 일정에 포함 어려움. v1.1에 계획 |
| 비용은 얼마나? | 강상민 | 시연 기간 6주 약 USD 222 (Anthropic Sonnet 4.5). 개인 프로젝트 기준 월 3~4만원 수준 |

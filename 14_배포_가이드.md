## 1. 배포 원칙

| 원칙 | 내용 |
| --- | --- |
| 실행 단위 | 단일 Spring Boot 애플리케이션 `qtai-server` |
| 인프라 | Docker Compose 기준 |
| DB | MySQL 8.0 |
| 제외 | Kubernetes, Helm, Kafka는 v1 배포에 포함하지 않음 |
| Secret | `.env` 또는 Docker Compose secrets로 관리 |

---

## 2. 배포 전 체크리스트

| 체크 | 기준 |
| --- | --- |
| CI | 필수 품질 게이트 통과 |
| DB | 마이그레이션 검토 완료 |
| Secret | 운영 Secret이 Git에 없고 배포 환경에만 존재 |
| 데이터 | 성경 소스 검토 메타데이터 확인 |
| AI | 사용자 요청 경로 LLM 호출 없음 |
| 관리자 | `/api/v1/admin/**` 권한 확인 |

---

## 3. 배포 순서

| 순서 | 작업 | 완료 기준 |
| --- | --- | --- |
| 1 | 배포 태그 확인 | 시연 또는 릴리스 대상 커밋 확정 |
| 2 | 환경 변수 확인 | 운영 `.env` 준비 |
| 3 | DB 백업 | 롤백 대비 백업 완료 |
| 4 | Docker image build | `qtai-server` 이미지 생성 |
| 5 | Docker Compose up | 서비스 기동 |
| 6 | Health check | API health 응답 정상 |
| 7 | 핵심 시나리오 확인 | Today QT, 로그인, 묵상, 관리자 조회 |

---

## 4. 롤백 기준

| 상황 | 조치 |
| --- | --- |
| 서버 기동 실패 | 직전 이미지로 재배포 |
| DB 마이그레이션 실패 | 배포 중단, 백업 복구 검토 |
| Today QT 진입 불가 | 즉시 롤백 후보 |
| 관리자 권한 오류 | 배포 중단, 권한 설정 확인 |
| Secret 노출 | 키 폐기, 재발급, 배포 중단 |

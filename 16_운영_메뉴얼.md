## 1. 운영 원칙

| 원칙 | 내용 |
| --- | --- |
| 권한 | 운영 기능은 `ADMIN`, 배치·AI 작업은 `SYSTEM` 기준으로 기록 |
| 배치 시각 | 성서 유니온 공개 00:00 KST, 시스템 수집 04:00 KST |
| 사용자 노출 | 해설은 C 테이블만 노출 |
| AI 호출 | 사용자 요청 경로에서 LLM 호출 금지 |
| 이벤트 실패 | 핸들러 실패 로그와 재처리 가능성을 관리 |

## 2. 일일 운영 체크

| 시각 | 확인 항목 | 조치 |
| --- | --- | --- |
| 00:00 이후 | 성서 유니온 공개 여부 | 직접 저장하지 않고 범위 수집 대상만 확인 |
| 04:00 이후 | QT 범위 수집 배치 | 실패 시 실패 로그 확인 |
| 05:00 전후 | AI 해설·시뮬레이터 상태 | 실패 항목 재생성 검토 |
| 상시 | 신고·댓글 상태 | 관리자 화면에서 처리 |
| 상시 | Secret·오류 로그 | 노출 의심 시 즉시 폐기·재발급 |

## 3. 관리자 작업

| 작업 | 기준 |
| --- | --- |
| QT 범위 수정 | 감사 로그 필수 |
| C 해설 수정 | 사용자 노출 문구만 수정, A/B 원문 노출 금지 |
| AI 재생성 | 관리자 트리거로만 실행 |
| 신고 처리 | 삭제 또는 비공개 처리 후 감사 로그 |
| 찬양 큐레이션 | 메타데이터만 관리, 가사·음원 저장 금지 |

---

## 4. 장애 대응

| 상황 | 대응 |
| --- | --- |
| Today QT 본문 없음 | 마지막 정상 캐시 확인, 배치 실패 로그 확인 |
| 해설 생성 실패 | C 테이블 미노출 또는 준비 중 처리, 관리자 재생성 |
| 시뮬레이터 실패 | `FAILED` 상태로 버튼 비활성화 |
| DB 오류 | 쓰기 중단 여부 확인, 백업과 로그 확보 |
| 관리자 API 권한 오류 | `ADMIN` 권한 설정과 토큰 확인 |

## 1. AI 역할

| 역할 | 허용 범위 | 금지 범위 |
| --- | --- | --- |
| 해설 생성기 | `commentary_materials` 영어 원문을 쉬운 한국어 승인 해설로 재해석 | 검증 없는 사용자 노출 |
| 편집자 에이전트 | 이단성, 페르소나 이탈, 과장 표현 검토 | 검증 없이 사용자 노출 |
| 시뮬레이터 생성기 | 오늘 QT 본문 장면 클립 사전 생성 | 모든 본문 클립 100% 보장 표현 |
| 사실 기반 Q&A 처리 | 단발 질문을 차단/검증/응답 상태로 처리 | 다중 턴 상담형 챗봇, SSE |

## 2. 실행 원칙

| 원칙 | 기준 |
| --- | --- |
| 실행 시점 | 04:00 KST 배치, 관리자 트리거, 사실 기반 Q&A 단발 요청 |

## 3. 금지 프롬프트·금지 기능

| 금지 항목 | 이유 |
| --- | --- |
| 다중 턴 사용자 상담 | 제품 범위 밖 |
| 신학적 확정 판정 | 검증 위험 |
| 출처 없는 단정 | 신뢰도·저작권 리스크 |
| 검증 참조 원문 재노출 | 사용자 노출 금지 |
| AI 찬양 추천 | 운영자 큐레이션 정책과 충돌 |
| 교회 인증 판단 | MVP 제외 |

## 4. 산출물 검증 기준

| 기준 | 확인 |
| --- | --- |
| 출처 | 하단 `sources` 또는 attribution 표시 |
| 톤 | 초심자 친화, 과장·단정 회피 |
| 내용 | 본문 범위와 무관한 내용 추가 금지 |
| 저작권 | 원문 직접 복제보다 재해석으로 저작권 리스크를 낮춤 |
| 노출 | 검증 참조 원문 직접 노출 금지 |

## 5. 재생성 기준

| 상황 | 조치 |
| --- | --- |

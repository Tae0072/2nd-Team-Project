## 1. Flyway 마이그레이션 전략

### 1.1 파일 명명 규칙

```
src/main/resources/db/migration/
├── V1__create_tables.sql         ← 테이블 스키마 (최초)
├── V2__seed_books.sql            ← BOOKS 시드 (66권 메타)
├── V3__seed_prompt_templates.sql ← PROMPT_TEMPLATES 시드
├── V4__add_index_bcv.sql         ← 인덱스 추가
└── V5__add_trace_id.sql          ← 컬럼 추가 (W2~W3)
```

**핵심 규칙:**
- **이미 적용된 V{N} 파일 절대 수정 금지** — checksum 검증 실패로 서비스 시작 불가
- 수정 필요 시 V{N+1} 파일 신규 작성
- 팀원 간 버전 번호 충돌 방지 — PR 전에 `git pull origin develop` + 번호 확인

### 1.2 서비스별 migration 디렉토리

### 2.2 KR_BIBLE 시드 (개역한글 — 01번 § 3.1 저작권 기준)

성경 전문 31,102절은 파일 크기로 SQL에 인라인 불가. **별도 CSV 파일 → SQL LOAD**:

```sql
-- bible-service/src/main/resources/db/migration/V3__seed_kr_bible.sql
-- 방법 A: LOAD DATA (MySQL)
LOAD DATA LOCAL INFILE '/docker-entrypoint-initdb.d/kr_bible.csv'
INTO TABLE KR_BIBLE
FIELDS TERMINATED BY ','
ENCLOSED BY '"'
LINES TERMINATED BY '\n'
IGNORE 1 ROWS
(book_code, chapter, verse, content);

-- 방법 B: Spring Boot DataLoader (권장 — 컨테이너 환경 이식성)
```

```kotlin
// bible-service/src/main/java/com/qtai/bible/BibleDataLoader.java
@Component
class BibleDataLoader(
    private val verseRepository: VerseJpaRepository,
) : CommandLineRunner {

    override fun run(vararg args: String?) {
        if (verseRepository.count() > 0) return  // 이미 시드됨 — skip

        val csv = this::class.java.getResourceAsStream("/data/kr_bible.csv")!!
        csv.bufferedReader().useLines { lines ->
            lines.drop(1)  // 헤더 skip
                .map { it.split(",") }
                .chunked(1000)  // 1000개 배치 INSERT
                .forEach { batch ->
                    verseRepository.saveAll(batch.map { cols ->
                        VerseJpaEntity(
                            bookCode = cols[0],
                            chapter = cols[1].toInt(),
                            verse = cols[2].toInt(),
                            content = cols[3].trim('"'),
                        )
                    })
                }
        }
        log.info("Bible seed complete: ${verseRepository.count()} verses")
    }
}
```

**CSV 파일 위치:** `bible-service/src/main/resources/data/kr_bible.csv`
**라이선스:** 개역한글은 비상업·교육 목적 한정(출처 표기 필수). 개역개정/ESV/NIV는 01번 § 3.1에서 사용 금지로 명시됨 — **공개 저장소에 전문 commit 금지.** 팀원에게 별도 채널로 공유 후 `.gitignore`에 추가.

```gitignore
# bible-service/.gitignore
src/main/resources/data/kr_bible.csv
```

### 3.3 시연용 해설 시드 (W1~W2 — bible_explanations row)

> **2026-05-14 갱신:** 기존 "RAG 시드"는 폐기(ADR-0013). 사전 적재 `bible_explanations` row 시드로 대체.

강상민이 W1~W2에 해설 생성 파이프라인으로 시드. 비교 데이터 MD 파일: `workspaces/DevC_강상민/seeds/source-md/` (Public Domain 영어 주석만):

```
source-md/                          # AI 비교 데이터 (그대로 노출 X)
├── gen1_matthew_henry.md           # 창세기 1장 Matthew Henry 주석 (PD)
├── jhn3_16_matthew_henry.md        # 요한복음 3:16 (PD)
├── psalm23_matthew_henry.md        # 시편 23편 (PD)
└── gen1_calvin.md                  # 창세기 1장 J. Calvin 주석 (PD)
```

```java
// AI 도메인 — 해설 생성 → 편집자 검증 → bible_explanations 적재
@Component
class ExplanationSeedRunner implements CommandLineRunner {
    public void run(String... args) {
        if (explanationRepo.count() > 0) return;  // 이미 시드됨

        var demoPassages = List.of("GEN:1:1", "JHN:3:16", "PSA:23:1");
        for (var p : demoPassages) {
            var draft = generator.generate(p);                  // LLM 자체 해설
            var verified = editor.verify(draft, p);             // 편집자 검증
            if (verified.isPassed()) {
                explanationRepo.save(Explanation.aiGenerated(p, draft, verified.sourceIds()));
            } else {
                log.warn("Seed failed verification: {}", p);    // 이지윤 큐로 이관
            }
        }
    }
}
```

> **편집자 검증 통과한 해설만** `bible_explanations` 영구 적재. 메커니즘 상세는 09_AI §6.4 (강상민 박제).

## 4. ~~RAG 시드 (ChromaDB)~~ — 폐기 (2026-05-14 ADR-0013)

> **2026-05-14 결정:** RAG/벡터 DB/엘라스틱서치 전부 폐기. 본 절은 v2 자료로 보존하지 않고 완전 폐기. AI 출처 컨텍스트는 사전 적재 `bible_explanations` row를 LLM system prompt에 텍스트로 주입하고, 사용한 row의 id 배열을 `ai_turns.sources`에 기록한다.

### 4.1 대체 시드 — `bible_explanations` 적재 (해설 생성 파이프라인)

| 주차 | 적재 본문 수 | 책임 |
| --- | --- | --- |
| W1~W2 | 시연 본문 좌표 5~10개 | 강상민 + 이지윤 (편집자 골든셋 큐레이션) |
| W3 | 30~50개 | 강상민 자동 파이프라인 |
| W4 | 시연 본문 + 인기 본문 100개 | 강상민 (편집자 검증 통과만) |

### 4.2 해설 생성 파이프라인 (강상민, 09_AI §6.4)

```
[Public Domain 영어 주석 PDF (Matthew Henry 등)]
      ↓ MD 변환 (workspaces/DevC_강상민/seeds/source-md/)
[LLM 비교 데이터 MD]
      ↓ LLM 자체 생성 (본문 + 비교 데이터 system prompt)
[AI 해설 초안]
      ↓ 편집자 에이전트 자동 검증 (이단성·페르소나·정확성·쉬움 4축)
[검증 통과] → bible_explanations (source='AI_GENERATED', editor_verified_at SET)
[검증 실패] → 사람 검수 큐 (이지윤 보조 큐레이션·골든셋)
```

> 메커니즘 상세(입출력·평가 지표·합격선·재시도·캐시)는 09_AI 가이드 §6.4에 강상민이 박제.

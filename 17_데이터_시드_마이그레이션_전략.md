## 1. Flyway 마이그레이션 전략

### 1.1 파일 명명 규칙

```
src/main/resources/db/migration/
├── V1__create_tables.sql         ← 테이블 스키마 (최초)
├── V2__seed_books.sql            ← BOOKS 시드 (66권 메타)
├── V3__seed_prompt_templates.sql ← PROMPT_TEMPLATES 시드
├── V4__add_index_bcv.sql         ← 인덱스 추가
└── V5__add_trace_id.sql          ← 컬럼 추가 (W2~W3)
```

**핵심 규칙:**
- **이미 적용된 V{N} 파일 절대 수정 금지** — checksum 검증 실패로 서비스 시작 불가
- 수정 필요 시 V{N+1} 파일 신규 작성
- 팀원 간 버전 번호 충돌 방지 — PR 전에 `git pull origin develop` + 번호 확인

### 1.2 서비스별 migration 디렉토리

| 서비스 | 경로 |
| --- | --- |
| auth-service | `auth-service/src/main/resources/db/migration/` |
| bible-service | `bible-service/src/main/resources/db/migration/` |
| journal-service | `journal-service/src/main/resources/db/migration/` |
| ai-service | `ai-service/src/main/resources/db/migration/` |

각 서비스는 독립 DB를 사용하므로 **migration 버전 번호가 서비스 간 겹쳐도 무관.**

### 2.2 KR_BIBLE 시드 (개역한글 — 01번 § 3.1 저작권 기준)

성경 전문 31,102절은 파일 크기로 SQL에 인라인 불가. **별도 CSV 파일 → SQL LOAD**:

```sql
-- bible-service/src/main/resources/db/migration/V3__seed_kr_bible.sql
-- 방법 A: LOAD DATA (MySQL)
LOAD DATA LOCAL INFILE '/docker-entrypoint-initdb.d/kr_bible.csv'
INTO TABLE KR_BIBLE
FIELDS TERMINATED BY ','
ENCLOSED BY '"'
LINES TERMINATED BY '\n'
IGNORE 1 ROWS
(book_code, chapter, verse, content);

-- 방법 B: Spring Boot DataLoader (권장 — 컨테이너 환경 이식성)
```

```kotlin
// bible-service/src/main/kotlin/com/qtai/bible/BibleDataLoader.kt
@Component
class BibleDataLoader(
    private val verseRepository: VerseJpaRepository,
) : CommandLineRunner {

    override fun run(vararg args: String?) {
        if (verseRepository.count() > 0) return  // 이미 시드됨 — skip

        val csv = this::class.java.getResourceAsStream("/data/kr_bible.csv")!!
        csv.bufferedReader().useLines { lines ->
            lines.drop(1)  // 헤더 skip
                .map { it.split(",") }
                .chunked(1000)  // 1000개 배치 INSERT
                .forEach { batch ->
                    verseRepository.saveAll(batch.map { cols ->
                        VerseJpaEntity(
                            bookCode = cols[0],
                            chapter = cols[1].toInt(),
                            verse = cols[2].toInt(),
                            content = cols[3].trim('"'),
                        )
                    })
                }
        }
        log.info("Bible seed complete: ${verseRepository.count()} verses")
    }
}
```

**CSV 파일 위치:** `bible-service/src/main/resources/data/kr_bible.csv`
**라이선스:** 개역한글은 비상업·교육 목적 한정(출처 표기 필수). 개역개정/ESV/NIV는 01번 § 3.1에서 사용 금지로 명시됨 — **공개 저장소에 전문 commit 금지.** 팀원에게 별도 채널로 공유 후 `.gitignore`에 추가.

```gitignore

```

### 4.2 자동 인덱싱 스크립트 (W3)

```python
# ai-service/scripts/rag_index.py
import argparse
import chromadb
from sentence_transformers import SentenceTransformer

def main():
    parser = argparse.ArgumentParser()
    parser.add_argument("--seed-only", action="store_true")
    parser.add_argument("--dir", default="src/main/resources/rag/corpus/")
    args = parser.parse_args()

    model = SentenceTransformer("paraphrase-multilingual-mpnet-base-v2")
    client = chromadb.HttpClient(host="localhost", port=8000)
    collection = client.get_or_create_collection("qtai_corpus")

    existing_ids = set(collection.get()["ids"])
    corpus_dir = Path(args.dir)

    for txt_file in corpus_dir.glob("**/*.txt"):
        doc_id = f"doc-{txt_file.stem}"
        if doc_id in existing_ids:
            print(f"Skip (already indexed): {doc_id}")
            continue

        content = txt_file.read_text(encoding="utf-8")
        metadata = parse_metadata_from_filename(txt_file.name)
        embedding = model.encode(content).tolist()

        collection.add(
            ids=[doc_id],
            documents=[content],
            embeddings=[embedding],
            metadatas=[metadata],
        )
        print(f"Indexed: {doc_id}")

    print(f"Total docs: {collection.count()}")
```

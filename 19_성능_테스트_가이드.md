## 1. 성능 목표 (SLA)

### 1.1 응답시간 목표

| 경로 | p50 | p95 | p99 | 비고 |
| --- | --- | --- | --- | --- |
| POST `/auth/login` | < 300ms | < 800ms | < 1.5s | bcrypt cost=12 포함 |
| GET `/bible/books/{code}/chapters/{ch}/verses/{v}` | < 50ms | < 100ms | < 200ms | Redis 캐시 hit 기준 |
| GET `/me/dashboard` (BFF 집계) | < 500ms | < 1.2s | < 2s | 3개 서비스 병렬 호출 |
| POST `/ai/sessions` (세션 생성) | < 200ms | < 500ms | < 1s | DB + ChromaDB 없음 |
| POST `/ai/sessions/{id}/turns` (첫 token) | < 2s | < 4s | < 8s | Anthropic 첫 token TTFT |

### 1.2 처리량 목표 (시연 환경)

| 목표 | 수치 |
| --- | --- |
| 동시 사용자 | 10명 (시연 평가자 수) |
| RPS (로그인 제외) | 50 RPS |
| AI 동시 세션 | 5개 (semaphore 10 중 절반 여유) |
| 에러율 | < 1% |

> **⚠️ Minikube 1 노드 환경:** 실제 운영 서버 대비 성능이 낮음. 10 VU (Virtual User) 기준으로 p95 목표 달성이면 시연 충분.

### 1.3 테스트 실행 주차

| 주차 | 목적 | 대상 |
| --- | --- | --- |
| W3 (6/1~5) | 첫 성능 측정 + 명확한 병목 발견 | Auth·Bible·BFF |
| W4 (6/8~12) | 개선 후 재측정 + SLA 통과 확인 | 전 서비스 |

### 2.2 기본 스크립트 구조

```javascript
// perf/smoke_auth.js
import http from 'k6/http';
import { check, sleep } from 'k6';
import { Rate } from 'k6/metrics';

// 커스텀 메트릭
const errorRate = new Rate('errors');

// 부하 설정 (여기서 VU/시간 설정)
export const options = {
  vus: 5,          // Virtual Users 동시 접속자 수
  duration: '30s', // 실행 시간
  thresholds: {
    http_req_duration: ['p(95)<800'],  // p95 800ms 이하
    errors: ['rate<0.01'],             // 에러율 1% 이하
  },
};

// 테스트 함수 (각 VU가 반복 실행)
export default function () {
  const res = http.post(
    'http://qtai-gateway.local/api/v1/auth/login',
    JSON.stringify({ email: 'tester@qtai.dev', password: 'Demo1234!' }),
    { headers: { 'Content-Type': 'application/json' } }
  );

  check(res, {
    'status 200': (r) => r.status === 200,
    'has accessToken': (r) => JSON.parse(r.body).accessToken !== undefined,
  });

  errorRate.add(res.status !== 200);
  sleep(1);  // 각 VU가 1초 대기 (실제 사용자 행동 모사)
}
```

```bash
# 실행
k6 run perf/smoke_auth.js

# 결과 예시:
# http_req_duration p(95)=234.5ms  ← SLA 목표 800ms 이하 ✅
# ✓ status 200        (100% pass)
# ✓ has accessToken   (100% pass)
```

### 2.3 결과 해석

```
scenarios: (100.00%) 1 scenario, 5 max VUs, 1m0s max duration

data_received         120 kB  2.0 kB/s
data_sent             15 kB   243 B/s

http_req_duration:    avg=185.4ms min=120.2ms med=175.3ms max=823.4ms
                      p(90)=230.1ms p(95)=270.5ms p(99)=823.4ms

http_reqs:            150     2.5/s
```

**핵심 지표:**
- `p(95)`: 전체 요청의 95%가 이 시간 이하 → SLA 주요 기준
- `p(99)`: 99%의 요청 시간 → 간혹 느린 요청 탐지
- `http_reqs`: 총 요청 수 / 초당 요청 수 (throughput)

## 3. 인증 경로 부하 테스트

```javascript
// perf/load_auth.js
import http from 'k6/http';
import { check, sleep } from 'k6';

const BASE_URL = __ENV.BASE_URL || 'http://qtai-gateway.local/api/v1';

export const options = {
  stages: [
    { duration: '30s', target: 5 },   // Ramp-up: 0→5 VU
    { duration: '1m',  target: 10 },  // Load: 10 VU 유지
    { duration: '20s', target: 0 },   // Ramp-down
  ],
  thresholds: {
    'http_req_duration{name:login}':   ['p(95)<800'],
    'http_req_duration{name:refresh}': ['p(95)<300'],
    'http_req_duration{name:me}':      ['p(95)<200'],
  },
};

// 공유 token 저장
let accessToken = '';
let refreshToken = '';

export function setup() {
  // 초기 로그인 (setup 단계 — 1회)
  const res = http.post(`${BASE_URL}/auth/login`, JSON.stringify({
    email: 'tester@qtai.dev', password: 'Demo1234!'
  }), { headers: { 'Content-Type': 'application/json' } });

  const body = JSON.parse(res.body);
  return { accessToken: body.accessToken, refreshToken: body.refreshToken };
}

export default function (data) {
  const headers = {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${data.accessToken}`,
  };

  // 1. /auth/me 호출 (가장 빈번)
  const meRes = http.get(`${BASE_URL}/auth/me`, { headers, tags: { name: 'me' } });
  check(meRes, { 'me status 200': (r) => r.status === 200 });

  sleep(0.5);

  // 2. 3번에 1번꼴로 refresh
  if (Math.random() < 0.33) {
    const refreshRes = http.post(`${BASE_URL}/auth/refresh`,
      JSON.stringify({ refreshToken: data.refreshToken }),
      { headers, tags: { name: 'refresh' } }
    );
    check(refreshRes, { 'refresh status 200': (r) => r.status === 200 });
    const body = JSON.parse(refreshRes.body);
    data.accessToken = body.accessToken;
    data.refreshToken = body.refreshToken;
  }

  sleep(1);
}
```

```bash
k6 run perf/load_auth.js -e BASE_URL=http://$(minikube ip):30080/api/v1
```

**주요 확인 포인트:**
- `p(95)` login > 800ms → bcrypt cost 낮추기 (cost=10으로 조정 검토)
- `p(95)` me > 200ms → Redis 캐시 히트 확인 (`kubectl exec redis -- redis-cli MONITOR`)
- 에러율 > 1% → Distributed Lock 경합 or DB connection pool 고갈

## 4. 성경 조회 경로 부하 테스트

```javascript
// perf/load_bible.js
import http from 'k6/http';
import { check, sleep } from 'k6';

const BASE_URL = __ENV.BASE_URL || 'http://qtai-gateway.local/api/v1';

// 자주 조회되는 구절 30개
const POPULAR_VERSES = [
  { book: 'GEN', ch: 1, v: 1 },
  { book: 'JHN', ch: 3, v: 16 },
  { book: 'PSA', ch: 23, v: 1 },
  { book: 'PRO', ch: 3, v: 5 },
  { book: 'ROM', ch: 8, v: 28 },
  // ... (나머지 25개)
];

export const options = {
  stages: [
    { duration: '30s', target: 20 },  // 20 VU — 성경 조회는 캐시 히트 시 가볍다
    { duration: '1m',  target: 50 },  // 50 VU peak
    { duration: '20s', target: 0 },
  ],
  thresholds: {
    'http_req_duration{name:verse}':  ['p(95)<100'],   // 캐시 hit 기준
    'http_req_duration{name:chapter}':['p(95)<200'],
    'http_req_duration{name:books}':  ['p(95)<50'],    // 66권 목록
  },
};

export default function (data) {
  const headers = { 'Authorization': `Bearer ${data.token}` };
  const v = POPULAR_VERSES[Math.floor(Math.random() * POPULAR_VERSES.length)];

  // 단일 구절 조회
  const verseRes = http.get(
    `${BASE_URL}/bible/books/${v.book}/chapters/${v.ch}/verses/${v.v}`,
    { headers, tags: { name: 'verse' } }
  );
  check(verseRes, { 'verse 200': (r) => r.status === 200 });

  sleep(0.3);

  // 장 전체 조회 (20% 확률)
  if (Math.random() < 0.2) {
    const chRes = http.get(
      `${BASE_URL}/bible/books/${v.book}/chapters/${v.ch}`,
      { headers, tags: { name: 'chapter' } }
    );
    check(chRes, { 'chapter 200': (r) => r.status === 200 });
  }

  sleep(0.5);
}
```

**병목 시나리오별 대응:**

| 증상 | 원인 | 해결 |
| --- | --- | --- |
| p95 > 100ms (cache miss 폭발) | Redis TTL 만료 + 동시 요청 | Cache warming 스크립트 실행 |
| p95 > 200ms (DB 직접 조회) | Redis 연결 실패 | 10번 § 12.1 확인 |
| p99 > 1s (특정 구절만 느림) | 인덱스 미적용 | `EXPLAIN SELECT` + `idx_kr_bible_bcv` 확인 |

## 5. AI 큐티 SSE 부하 테스트

SSE는 일반 HTTP와 달리 장시간 연결 유지 → k6의 `http.get` 대신 **별도 패턴** 필요.

### 5.1 SSE 응답시간 측정 (첫 token TTFT)

```javascript
// perf/load_ai_ttft.js
// TTFT: Time To First Token — AI UX에서 가장 중요한 지표
import http from 'k6/http';
import { check } from 'k6';
import { Trend } from 'k6/metrics';

const ttft = new Trend('ai_ttft_ms');  // Time to First Token 커스텀 메트릭

export const options = {
  vus: 3,         // AI 동시 호출 3 (semaphore 10 중 여유)
  duration: '2m',
  thresholds: {
    'ai_ttft_ms': ['p(95)<4000'],  // TTFT p95 4초 이하 (09번 § 3.4 정합)
  },
};

export default function (data) {
  const startTime = Date.now();

  const res = http.post(
    `${BASE_URL}/ai/sessions/${data.sessionId}/turns`,
    JSON.stringify({ step: 'A', content: '이 구절에서 누가 등장하나요?' }),
    {
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${data.token}`,
        'Accept': 'text/event-stream',
      },
      responseType: 'text',  // SSE 전체 스트림 수신
      timeout: '300s',        // SSE 5분 타임아웃
      tags: { name: 'ai_turn' },
    }
  );

  // 첫 token 이벤트까지 시간 추출 (응답 본문 파싱)
  const firstTokenIdx = res.body.indexOf('event: token');
  if (firstTokenIdx > -1) {
    // 실제 TTFT는 Tempo 분산 추적으로 정확히 측정 (§ 8.2)
    ttft.add(Date.now() - startTime);
  }

  check(res, {
    'ai turn 200': (r) => r.status === 200,
    'has token event': (r) => r.body.includes('event: token'),
    'has end event': (r) => r.body.includes('event: end'),
  });
}
```

### 5.2 AI 동시 세션 한도 테스트

## 6. 대시보드 집계 부하 테스트

BFF Aggregator가 3개 서비스 병렬 호출 — 가장 복잡한 경로.

```javascript
// perf/load_dashboard.js
import http from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  stages: [
    { duration: '30s', target: 10 },
    { duration: '2m',  target: 20 },
    { duration: '30s', target: 0 },
  ],
  thresholds: {
    'http_req_duration{name:dashboard}': ['p(95)<1200'],
    'http_req_duration{name:journals}':  ['p(95)<500'],
  },
};

export default function (data) {
  const headers = { 'Authorization': `Bearer ${data.token}` };

  // 대시보드 집계 (Bible + Journal + AI Session 병렬)
  const dashRes = http.get(`${BASE_URL}/bff/me/dashboard`,
    { headers, tags: { name: 'dashboard' } });

  check(dashRes, {
    'dashboard 200': (r) => r.status === 200,
    'has todayVerse': (r) => {
      try { return JSON.parse(r.body).todayVerse !== undefined; }
      catch { return false; }
    },
  });

  sleep(2);

  // 묵상 노트 목록
  const journalRes = http.get(`${BASE_URL}/journals?page=0&size=10`,
    { headers, tags: { name: 'journals' } });
  check(journalRes, { 'journals 200': (r) => r.status === 200 });

  sleep(1);
}
```

## 8. 병목 찾기 — Grafana·Tempo·Loki

### 8.1 Grafana 대시보드로 1차 병목 확인

k6 실행 중 또는 직후 Grafana에서 확인:

```bash
# Grafana port-forward
kubectl port-forward svc/grafana -n qtai 3000:3000
# http://localhost:3000
```

**"서비스 응답시간" 패널 확인:**
- `http_server_requests_seconds` (Spring Actuator) → 각 endpoint p95
- `ai_turn_duration_seconds` (AI Service) → turn 처리 시간 분해
- `db_query_duration_seconds` (JPA metric) → DB 병목

**1차 병목 판단:**

| p95 초과 서비스 | 가능성 높은 원인 | 2차 확인 |
| --- | --- | --- |
| Gateway (> 50ms 추가) | 라우팅·필터 오버헤드 | Gateway 자체 metric |
| Auth (> 800ms) | bcrypt 비용 / DB pool 고갈 | DB connection metric |
| Bible (> 100ms) | Redis cache miss / index 없음 | Redis MONITOR |
| AI Service (> 4s TTFT) | Anthropic 레이턴시 / semaphore | ai_concurrent_calls_active |
| BFF (> 1.2s) | 직렬 서비스 호출 (병렬 아님) | BFF span tree |

### 8.2 Tempo 분산 추적으로 2차 병목 정밀 측정

```bash
# Grafana → Explore → Tempo
# 쿼리: { .http.url =~ "/api/v1/bff/me/dashboard" }
# p95 느린 trace 클릭 → span tree 확인
```

**span tree 예시 (BFF 대시보드):**
```
BFF /me/dashboard  [1200ms]
├── Auth /auth/me           [150ms]  ← 정상
├── Bible /bible/today      [80ms]   ← 정상
└── Journal /journals?page=0 [950ms] ← 병목 발견!
    └── DB SELECT journals   [900ms] ← N+1 쿼리 의심
```

**N+1 쿼리 수정:**
```kotlin
// ❌ N+1
fun listJournals(userId: Long): List<Journal> =
    journalRepository.findByUserId(userId)  // 10개 journal + 10개 event 개별 조회

// ✅ fetch join
@Query("SELECT j FROM Journal j LEFT JOIN FETCH j.events WHERE j.userId = :userId")
fun findWithEvents(userId: Long): List<Journal>
```

## 10. CI 성능 게이트 (W4)

### 10.1 GitHub Actions 성능 게이트

```yaml
# .github/workflows/perf.yml
name: Performance Gate

on:
  push:
    branches: [develop]

jobs:
  smoke-perf:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Setup k6
        uses: grafana/setup-k6-action@v1

      - name: Start Minikube
        run: |
          minikube start --memory=6144 --cpus=4 --driver=docker
          kubectl apply -f k8s/
          kubectl wait --for=condition=ready pod --all -n qtai --timeout=300s

      - name: Run smoke test
        run: k6 run perf/smoke_all.js -e BASE_URL=http://$(minikube ip):30080/api/v1
        # thresholds 실패 시 exit code 99 → CI 실패

      - name: Archive results
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: k6-results-${{ github.sha }}
          path: k6-summary.json
```

### 10.2 W4 성능 게이트 통과 기준 요약

```
✅ Smoke test: 10 iterations, p99 < 3s, error rate < 1%
✅ Auth load: p95 login < 800ms, p95 me < 200ms

✅ Dashboard: p95 < 1200ms
✅ AI TTFT: p95 < 4s (Anthropic 네트워크 상태 따라 유동)
```

| k6 결과 | 대응 |

| DB query p95 > 목표 2배 | 인덱스 추가 · N+1 수정 · query 최적화 |

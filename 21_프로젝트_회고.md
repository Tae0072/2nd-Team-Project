## 1. 문서 목적과 경계

이 문서는 프로젝트가 끝난 뒤 실제 회고를 작성하기 위한 기준 틀이다. 현재 시점에서는 달성률, 테스트 커버리지, PR 수, 빌드 성공률 같은 실제 결과를 지어내지 않고 `TBD`로 둔다.

| 구분 | 이 문서에서 관리 | 다른 문서에서 관리 |
| --- | --- | --- |
| KPT 회고 | 포함 | 실제 개선 반영은 후속 프로젝트 계획 |
| 개인별 회고 | 포함 | 개인별 일정은 `개발자별_일정표/` |
| AI 협업 회고 | 포함 | AI 작업 기준은 `17_AI_에이전트_협업_가이드.md` |
| 일정·범위 회고 | 포함 | 계획 기준은 `00_개발_일정_총괄표.md`, `01_프로젝트_계획서.md` |
| 테스트/품질 결과 | 요약만 포함 | 원본 결과는 `13_테스트_보고서.md`, `18_코드_품질_게이트.md` |
| 포트폴리오 소재 | 포함 | README 기준은 `26_포트폴리오_README.md` |

## 2. 회고 작성 원칙

| 원칙 | 기준 |
| --- | --- |
| 정직성 | 실패, 지연, 제거된 범위를 숨기지 않는다. |
| 근거 우선 | 수치, PR, 테스트 로그, 발표 리허설 결과를 근거로 쓴다. |
| 요구사항 기준 | `07_요구사항_정의서.md` v2.3과 다른 내용을 성공으로 포장하지 않는다. |
| 가짜 성과 금지 | 실제 측정 전 커버리지, 성능, PR 수를 임의로 쓰지 않는다. |
| 다음 행동 연결 | Problem은 반드시 Try 또는 Action Item으로 연결한다. |
| 개인 비난 금지 | 사람보다 프로세스, 결정, 기준, 자동화의 문제를 분석한다. |

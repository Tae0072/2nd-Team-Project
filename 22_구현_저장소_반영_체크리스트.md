## 1. 문서 목적과 경계

이 문서는 구현 저장소에서 첫 PR을 만들기 전에 반영해야 할 기준을 작업 가능한 체크리스트로 정리한다.

요구사항 원본은 `07_요구사항_정의서.md`가 책임지고, 아키텍처 상세는 `03_아키텍처_정의서.md`, API 계약은 `04_API_명세서.md`, 협업 규칙은 `09_Git_규칙.md`, 품질 게이트는 `18_코드_품질_게이트.md`, 용어 기준은 `23_도메인_용어사전.md`가 책임진다.

이 문서 저장소에는 실제 Spring Boot/Flutter 골격, OpenAPI 파일, DB/데이터 디렉터리, infra 디렉터리, CI workflow 파일을 두지 않는다. 아래 항목은 별도 구현 GitHub에서 만들 산출물의 기준이다.

| 구분 | 이 문서에서 관리 | 다른 문서에서 관리 |
| --- | --- | --- |
| 구현 저장소 초기 세팅 항목 | 포함 | 세부 설계 원칙은 `03` |
| PR 템플릿 반영 항목 | 포함 | 원문 템플릿은 `14` |
| CI/Guard 반영 순서 | 포함 | 상세 게이트 정의는 `18` |
| 패키지·API·데이터 기준 요약 | 포함 | 상세 기준은 `03`, `04`, `07`, `23` |
| 기능 요구사항 본문 | 포함하지 않음 | `07` |

문서 간 충돌이 발생하면 기능 포함 여부와 사용자 노출 기준은 `07_요구사항_정의서.md` v2.3을 우선한다.

| 우선순위 | 반영 항목 | 구현 저장소 산출물 |

## 3. 저장소 구조 체크리스트

구현 저장소는 v1에서 단일 `qtai-server` Modular Monolith 기준을 따른다.

```text
repo-root/
    .github/
        pull_request_template.md
        workflows/
            qt-ai-ci.yml
    apis/
        bff/
            openapi.yaml
    data/
        bible-sources/
            README.md
    qtai-server/
        src/main/java/com/qtai/
            gatewayauth/
            bff/
            bible/
                journal/
                songs/
            ai/
            simulator/
    flutter-app/
```

| 체크 | 기준 |
| --- | --- |
| 단일 백엔드 | `qtai-server` 하나만 둔다. `auth-service`, `bible-service`, `journal-service` 같은 독립 서비스는 만들지 않는다. |
| 도메인 경계 | 다른 도메인의 Entity, Service, Repository 직접 import를 금지한다. |
| 공개 범위 | 다른 도메인이 import 가능한 타입은 `api/` 하위 Port/DTO로 제한한다. |
| `journal`/`songs` | 최상위 도메인이 아니라 `bible` 내부 하위 모듈로 둔다. |
| 외부 API | Flutter/Admin이 호출하는 HTTP API는 `/api/v1/**`로 통일한다. |
| 내부 호출 | 내부 도메인 호출은 Java Interface로 표현하고 OpenAPI에 노출하지 않는다. |

## 4. PR 템플릿 반영 체크리스트

`09_Git_규칙.md`의 PR 템플릿을 구현 저장소 `.github/pull_request_template.md`에 반영한다.

| 체크 | 필수 내용 |
| --- | --- |
| 관련 요구사항 | 기능 PR은 `07`의 F-ID 또는 관련 문서 링크를 포함한다. |
| 변경 유형 | `feat`, `fix`, `refactor`, `test`, `docs`, `chore` 중 선택한다. |
| 도메인 경계 | 직접 import 금지, DTO/Interface 통신 여부를 확인한다. |
| 금지 기술 | RAG, ChromaDB, vector DB, Elasticsearch, Kafka, Kubernetes, Helm v1 도입이 없음을 확인한다. |
| 사용자 AI | Q&A, 챗봇, SSE, `/ai/sessions/**`가 없음을 확인한다. |
| Today QT | 본문, 해설 진입점, 묵상 진입점, 시뮬레이터 상태값을 반환하는지 확인한다. |
| 성경 데이터 | repo URL, license, translationName, attribution, redistributionAllowed를 확인한다. |
| 권한 | 관리자 API는 `ADMIN`, 배치/AI 내부 작업은 `SYSTEM` 기준을 확인한다. |
| 테스트 | 단위 테스트, 통합 테스트, 미작성 사유를 명시한다. |

## 5. CI/Guard 반영 체크리스트

`18_코드_품질_게이트.md`의 GitHub Actions 예시는 구현 저장소 구조에 맞게 경로만 조정해서 반영한다.

| 게이트 | 구현 저장소 반영 기준 | 실패 시 |
| --- | --- | --- |
| Spectral lint | `apis/*/openapi.yaml` 검사 | 머지 차단 |
| Backend build/test | Java 21, Spring Boot 3.3 기준 `./gradlew build` | 머지 차단 |
| Coverage | Jacoco 70% 이상 | 머지 차단 |
| Modulith/ArchUnit | 도메인 경계와 금지 import 검사 | 머지 차단 |
| Requirements Guard | 금지 기술, 금지 API, 금지 데이터 검사 | 머지 차단 |
| Document/Terminology Guard | 문서·용어 금지 표현 검사 | 머지 차단 또는 문맥 확인 |
| Flutter analyze/test | `flutter-app` 존재 시 실행 | 머지 차단 |
| Secret scan | gitleaks | 머지 차단 + Lead 알림 |

### 5.1 Requirements Guard 필수 검사

| 검사 대상 | 금지 또는 차단 기준 |
| --- | --- |
| 사용자 AI API | `/ai/sessions/**`, SSE, 사용자 요청 경로 LLM 호출 |
| 검색·임베딩 | RAG, ChromaDB, vector DB, Elasticsearch |
| 메시징·인프라 | Kafka, Kubernetes, Helm v1 도입 |
| 성경 데이터 | 개역개정, ESV, NIV |
| 교회 인증 | 회원가입 화면 버튼, API, `churchAuth`, `church_cert` |
| 찬양 | AI 찬양 추천, 가사·음원 저장 |

Requirements Guard는 구현 코드, API 산출물, DB seed, CI workflow를 대상으로 한다. `.github/pull_request_template.md`는 금지 항목을 체크리스트로 설명하는 문서성 파일이므로 금지 패턴 스캔 대상에서 제외하고, `.github/workflows`만 검사한다.

### 5.2 Document/Terminology Guard 필수 검사

| 피해야 할 표현 | 권장 표현 |
| --- | --- |
| 저작권 문제 없음 | 저작권 리스크를 낮춘다 |
| 이벤트 유실률 0% 보장 | 핸들러 실패 로그 + 재처리 가능 |
| 내부 API 경로 | 내부 Java Interface |
| AI 찬양 추천 | 운영자 사전 큐레이션 |
| 오늘 QT 100% = 모든 클립 있음 | 시뮬레이터 상태값 100% 반환 |

| 7 | 기본 CI 연결 | PR에서 build/test/guard 실행 |

## 7. API 첫 PR 체크리스트

API 산출물은 외부 공개 HTTP API만 OpenAPI로 관리한다.

| 체크 | 기준 |
| --- | --- |
| Prefix | 모든 외부 공개 API는 `/api/v1`로 시작한다. |
| 소프트 로그인 | 오늘 QT 미리보기, 성경 목록/본문, 찬양 큐레이션 조회는 `PUBLIC`이다. |
| 로그인 필요 | 묵상 저장, 해설 본문 열람, 찬양 저장, 묵상 달력 조회는 `USER`이다. |
| 관리자 | `/api/v1/admin/**`은 `ADMIN`이다. |
| Today QT | 해설 본문을 직접 싣지 않고 해설 진입점과 묵상 진입점을 제공한다. |
| Simulator | 상태는 `READY`, `MISSING`, `FAILED`, `DISABLED` 중 하나다. |
| 내부 Interface | `JournalCommandPort`, `SongQueryPort` 같은 내부 Java Interface는 OpenAPI에 포함하지 않는다. |

## 1. 기능 명세 원칙

### 1.1 사용자 경험 원칙

- 첫 진입에서 로그인을 강제하지 않는다.
- 사용자는 앱 실행 후 오늘의 QT 본문으로 바로 진입하고, 필요할 때 설명·주석·AI·묵상 기록으로 확장한다.
- 일반 성경 보기는 책·장·절 조회 전용이며, 자유 본문 기반 AI 질문·묵상 기록은 MVP에서 제외한다.
- 개인 묵상 기록은 기본 비공개다.
- 공유는 사용자의 명시적 선택이 있을 때만 동작한다.
- AI는 사용자의 신앙 상태를 평가하지 않고, 본문 이해와 묵상을 돕는 방향으로만 답한다.
- 설명·주석·AI 답변은 출처를 함께 보여준다.

### 1.2 구현 기준

- API 상세 스키마는 `apis/{service}/openapi.yaml`을 우선한다.
- Gateway 외부 경로와 인증 정책은 `DECISIONS.md`를 우선한다.
- 묵상 기록 API는 별도 Journal Service가 아니라 Bible Service에 포함한다.
- Auth Service와 Journal Service는 MVP에서 독립 서비스로 만들지 않는다.
- 성경 데이터는 개역한글, KJV, Matthew Henry 주석만 허용한다.
- 개역개정, ESV, NIV, 찬양 가사·음원 저장은 금지한다.

### 1.3 공통 상태 및 권한

| 구분 | 정책 |
| --- | --- |
| 비로그인 허용 | 온보딩, QT 가이드, 오늘의 QT 미리보기, 한/영 성경 본문, 성경 목록, 쉬운 본문 설명 |
| 로그인 필요 | 주석 열람, AI 질문, 묵상 기록 저장·수정·삭제, 찬양 목록 저장, 익명 나눔 공개·취소, 좋아요, 댓글 작성, 신고 |
| 선택 인증 | `GET /api/v1/qt/today`, `GET /api/v1/passages/{bookCode}/{chapter}/{verse}` |
| 기본 공개 상태 | `PRIVATE` |
| 공유 공개 상태 | `ANONYMOUS_SHARED` |
| AI 세션 상태 | `IN_PROGRESS`, `COMPLETED`, `COMPLETED_NO_JOURNAL` |

## 3. 상세 기능 명세

## FS-01. 온보딩·QT 가이드

### 목적

QT를 처음 접하는 사용자가 앱을 켰을 때 무엇을 해야 하는지 바로 이해하도록 돕는다.

### 진입 조건

| 항목 | 내용 |
| --- | --- |
| Actor | 비로그인 사용자, 로그인 사용자 |
| 진입점 | 앱 최초 실행, 홈의 `QT 가이드` |
| 인증 | 불필요 |

### 화면 구성

| 영역 | 표시 내용 |
| --- | --- |
| 첫 문구 | `QT가 처음이신가요?` |
| 설명 | QT의 의미, 오늘의 본문 읽기, 질문하기, 묵상 기록하기 순서 |
| CTA | `오늘의 QT 시작하기` |
| 보조 진입 | `나중에 보기`, `QT 용어 보기` |

### 동작 흐름

1. 앱 최초 실행 시 온보딩 완료 여부를 로컬 저장소에서 확인한다.
2. 완료 이력이 없으면 온보딩을 표시한다.
3. 사용자가 `오늘의 QT 시작하기`를 누르면 오늘의 QT 화면으로 이동한다.
4. 사용자가 `나중에 보기`를 누르면 오늘의 QT 화면으로 이동하되, 설정/가이드 진입점에서 다시 QT 가이드로 접근할 수 있다.

### 데이터

| 필드 | 저장 위치 | 설명 |
| --- | --- | --- |
| `onboardingSeen` | Flutter local storage | 온보딩 확인 여부 |
| `guideTerms` | 앱 정적 데이터 또는 Bible Service | QT, 주석, 묵상, 적용, 기도 제목 등 용어 |

### 예외

| 상황 | 처리 |
| --- | --- |
| 로컬 저장소 읽기 실패 | 온보딩을 다시 표시한다 |
| 용어 데이터 로딩 실패 | 기본 내장 용어 목록을 표시한다 |

### 완료 기준

- 첫 실행 시 온보딩이 표시된다.
- 로그인하지 않아도 온보딩과 QT 가이드를 볼 수 있다.
- 온보딩 완료 후 재실행 시 오늘의 QT 화면으로 진입한다.

## FS-02. 오늘 QT 바로 진입·소프트 로그인

### 목적

로그인 전에도 핵심 경험을 미리 볼 수 있게 하고, 앱 실행 시 별도 홈 화면 없이 오늘 QT 화면으로 바로 진입한다. 기록·AI·공유처럼 개인 데이터가 필요한 시점에만 로그인을 유도한다.

### 진입 조건

| 항목 | 내용 |
| --- | --- |
| Actor | 비로그인 사용자, 로그인 사용자 |
| 진입점 | 앱 실행, 온보딩 완료 |
| 인증 | 선택 |

### 화면 구성

| 메뉴 | 비로그인 | 로그인 |
| --- | --- | --- |
| 오늘의 QT 본문 | 가능 | 가능 |
| 내 묵상 기록 | 로그인 유도 | 가능 |
| 성경 검색 | 가능 | 가능 |
| AI에게 질문하기 | 로그인 유도 | 가능 |
| 내 찬양 목록 | 로그인 유도 | 가능 |
| QT 가이드 | 가능 | 가능 |
| 익명 나눔 | 목록 읽기 가능, 작성·반응은 로그인 유도 | 가능 |

### API

| 목적 | Method | Path | 인증 |
| --- | --- | --- | --- |
| 대시보드 | GET | `/api/v1/me/dashboard` | 필수 |
| 오늘의 QT 미리보기 | GET | `/api/v1/qt/today` | 선택 |
| 로그인 | POST | `/auth/login` 또는 Gateway Auth 경로 | 불필요 |
| Google OAuth | GET/POST | Gateway Auth 경로 | 불필요 |

### 동작 흐름

1. 앱 실행 시 토큰 존재 여부를 확인한다.
2. 토큰이 있으면 사용자 대시보드와 오늘의 QT를 함께 로딩한다.
3. 토큰이 없으면 오늘의 QT 본문과 공개 메뉴만 표시한다.
4. 사용자가 로그인 필요 기능을 누르면 로그인 화면으로 이동한다.
5. 로그인 완료 후 원래 진입하려던 화면으로 복귀한다.
6. 오늘 QT 데이터가 먼저 표시된 뒤 나머지 성경 데이터는 백그라운드에서 로딩한다.

### 예외

| 상황 | 처리 |
| --- | --- |
| Access Token 만료 | Refresh Token으로 재발급 후 원 요청 재시도 |
| Refresh 실패 | 로그인 상태 초기화 후 로그인 화면 이동 |
| 대시보드 실패 | 홈 기본 메뉴는 유지하고 오류 메시지 표시 |

### 완료 기준

- 비로그인 첫 진입에서 오늘의 QT 본문이 먼저 보인다.
- 로그인 필요 기능은 누르는 순간 자연스럽게 로그인으로 안내한다.
- 로그인 후 원래 목적 화면으로 복귀한다.

## FS-03. 오늘의 QT·입체 본문 보기

### 목적

사용자가 오늘의 성경 본문을 절별 한글·영어 형태로 읽고, 한 흐름 안에서 설명·주석·AI·묵상으로 이어가도록 한다.

### 진입 조건

| 항목 | 내용 |
| --- | --- |
| Actor | 비로그인 사용자, 로그인 사용자 |
| 진입점 | 앱 첫 진입의 오늘 QT 화면, 성경 검색 결과 |
| 인증 | 선택 |

### 화면 구성

| 영역 | 표시 내용 |
| --- | --- |
| 본문 헤더 | 본문 범위, 날짜, 주제 |
| 본문 리스트 | 절 번호, 개역한글, KJV |
| 설명 패널 | 한 줄 요약, 쉬운 설명, 어려운 단어 |
| 주석 패널 | Matthew Henry 원문 또는 요약 더미 |
| 액션 | 오늘 QT에서는 AI 질문하기, 묵상 기록하기, 찬양 보기. 일반 성경 검색 결과에서는 조회 전용 |

### API

| 목적 | Method | Path | 인증 |
| --- | --- | --- | --- |
| 오늘의 QT | GET | `/api/v1/qt/today` | 선택 |
| 통합 본문 | GET | `/api/v1/passages/{bookCode}/{chapter}/{verse}` | 선택 |
| 성경 목록 | GET | `/bible/books` | 불필요 |
| 한글 본문 | GET | `/bible/kr/{bookCode}/{ch}/{v}` | 불필요 |
| 영어 본문 | GET | `/bible/en/{bookCode}/{ch}/{v}` | 불필요 |

### 요청 옵션

| 파라미터 | 설명 |
| --- | --- |
| `includeCommentary` | 로그인 사용자가 주석 포함 여부를 선택한다. 기본값은 false |

### 동작 흐름

1. 사용자가 오늘의 QT를 시작한다.
2. BFF가 오늘의 본문과 필요한 본문 데이터를 병렬 조회한다.
3. 화면은 한글·영어 본문을 절 단위로 묶어서 표시한다.
4. 사용자가 스크롤하면 한글·영어 절 위치가 함께 움직인다.
5. 사용자가 설명·주석·AI·묵상 액션을 선택하면 해당 기능으로 이어진다.
6. 일반 성경 보기의 자유 본문에서는 AI·묵상 액션을 제공하지 않는다.

### 데이터 표시 규칙

| 규칙 | 내용 |
| --- | --- |
| 절 정렬 | 한글 본문과 영어 본문은 같은 절 번호 기준으로 묶는다 |
| 본문 누락 | 한쪽 번역이 없으면 해당 절에 빈 상태 메시지를 표시한다 |
| 저작권 | 개역개정, ESV, NIV는 표시하지 않는다 |
| 출처 | 본문 하단에 데이터 출처를 표시한다 |

### 예외

| 상황 | 처리 |
| --- | --- |
| 일부 본문 조회 실패 | 성공한 본문을 먼저 표시하고 실패 영역만 재시도 버튼 제공 |
| 오늘의 QT 데이터 없음 | 기본 샘플 본문 또는 성경 검색으로 유도 |
| 비로그인 주석 열람 | 로그인 유도 |

### 완료 기준

- 절별 한글·영어 본문이 같은 절 기준으로 표시된다.
- 비로그인 사용자도 본문과 쉬운 설명까지 볼 수 있다.
- 주석은 사용자가 선택했을 때만 로딩한다.
- 오늘 QT 본문은 MVP에서 한 절이며, `verseStart`와 `verseEnd`가 같은 값이다.

## FS-04. 쉬운 본문 설명·용어 설명

### 목적

성경 지식이 많지 않은 사용자가 본문을 어렵게 느끼지 않도록 요약, 배경, 단어 설명을 제공한다.

### 진입 조건

| 항목 | 내용 |
| --- | --- |
| Actor | 비로그인 사용자, 로그인 사용자 |
| 진입점 | 오늘의 QT 화면의 설명 패널 |
| 인증 | 불필요 |

### 화면 구성

| 영역 | 표시 내용 |
| --- | --- |
| 한 줄 요약 | 오늘 본문 핵심을 한 문장으로 표시 |
| 쉬운 설명 | 초심자 기준의 배경과 맥락 |
| 어려운 단어 | 본문 안의 낯선 단어와 쉬운 뜻 |
| QT 용어 | QT, 주석, 묵상, 적용, 기도 제목 등 |

### API

| 목적 | Method | Path | 인증 |
| --- | --- | --- | --- |
| 쉬운 설명 | GET | `/api/v1/explanations/{bookCode}/{ch}/{v}` | 불필요 |
| 통합 본문 내 설명 | GET | `/api/v1/passages/{bookCode}/{chapter}/{verse}` | 선택 |

### 데이터

| 필드 | 설명 |
| --- | --- |
| `summary` | 본문 한 줄 요약 |
| `background` | 본문 배경 설명 |
| `terms` | 어려운 단어 목록 |
| `source` | 설명 생성 또는 검수 출처 |

### 예외

| 상황 | 처리 |
| --- | --- |
| 설명 데이터 없음 | `아직 준비 중인 설명입니다` 표시 |
| 용어 데이터 없음 | 앱 내 기본 용어 사전 사용 |

### 완료 기준

- 설명은 기본 본문을 가리지 않고 선택적으로 열린다.
- 용어는 초심자 기준의 쉬운 문장으로 제공된다.
- AI가 매번 즉시 생성한 내용처럼 표시하지 않는다.

## FS-05. 주석 보기

### 목적

본문을 더 깊게 이해하고 싶은 사용자가 Matthew Henry 주석 또는 검수된 더미 주석을 선택적으로 볼 수 있게 한다.

### 진입 조건

| 항목 | 내용 |
| --- | --- |
| Actor | 로그인 사용자 |
| 진입점 | 오늘의 QT 화면의 `주석 보기` |

## FS-06. AI 질문·출처 기반 1회성 Q&A

### 목적

사용자가 본문을 읽다가 생긴 질문을 AI에게 묻고, 성경 본문·주석·RAG 출처에 기반한 답을 스트리밍으로 받는다.

### 진입 조건

| 항목 | 내용 |
| --- | --- |
| Actor | 로그인 사용자 |
| 진입점 | 오늘의 QT 화면의 `AI에게 질문하기` |
| 인증 | 필수 |

### 핵심 정책

- AI는 DeepSeek API를 사용한다.
- AI는 계속 이어지는 자유 챗봇이 아니라 출처 기반 1회성 Q&A로 동작한다.
- AI 질문은 오늘 QT 본문에서만 시작한다. 일반 성경 보기에서 선택한 자유 본문은 MVP에서 AI 대상이 아니다.
- QT A~D는 자동 진행 단계가 아니라 답변 톤과 질문 가이드 구조로 사용한다.
- 사용자가 `묵상 완료`를 눌러야 AI 세션 완료 이벤트가 발행된다.
- AI 답변 하단에는 RAG 출처를 표시한다.

### API

| 목적 | Method | Path | 인증 |
| --- | --- | --- | --- |
| 세션 생성 | POST | `/ai/sessions` | 필수 |
| 질문 전송·SSE 응답 | POST | `/ai/sessions/{id}/turns` | 필수 |
| 세션 조회 | GET | `/ai/sessions/{id}` | 필수 |
| 세션 목록 | GET | `/ai/sessions` | 필수 |
| 묵상 완료 | POST | `/ai/sessions/{id}/complete` | 필수 |

### SSE 이벤트

| 이벤트 | 처리 |
| --- | --- |
| `turn_started` | 응답 영역을 생성하고 로딩 상태를 표시한다 |
| `token` | 토큰을 누적해 답변 본문에 표시한다 |
| `rag_sources` | 답변 하단 출처 영역에 표시한다 |
| `turn_completed` | 입력창을 다시 활성화하고 완료 CTA를 표시한다 |
| `done` | 스트림을 종료한다. 필요 시 data 값으로 `[DONE]` 마커를 함께 보낼 수 있다 |
| `error` | 스트림을 닫고 재시도 버튼을 표시한다 |

### 화면 구성

| 영역 | 표시 내용 |
| --- | --- |
| 본문 맥락 | 질문 대상 본문 범위 |
| 질문 입력 | 사용자의 질문 |
| AI 답변 | 스트리밍 토큰 누적 표시 |
| 출처 | 본문, 주석, RAG 문서 |
| 완료 CTA | `묵상 완료` |

### 동작 흐름

1. 사용자가 AI 질문 화면에 진입한다.
2. AI 세션이 없으면 `POST /ai/sessions`로 세션을 만든다.
3. 사용자가 질문을 입력한다.
4. 앱은 `POST /ai/sessions/{id}/turns`를 호출하고 SSE를 구독한다.
5. AI 답변이 토큰 단위로 표시된다.
6. 출처가 도착하면 답변 하단에 표시한다.
7. 사용자가 `묵상 완료`를 누르면 `POST /ai/sessions/{id}/complete`를 호출한다.
8. AI Service는 `ai.session.completed` Kafka 이벤트를 발행한다.
9. Bible Service는 이벤트를 소비해 오늘 QT DRAFT에 AI 요약과 세션 참조를 연결한다.

### 입력 검증

| 항목 | 처리 |
| --- | --- |
| 오늘 QT 본문과 다른 passage | 422 `AI_PASSAGE_NOT_TODAY_QT` |
| 일반 성경 보기에서 AI 진입 | 클라이언트에서 진입점 미노출, 서버도 422 처리 |

### 예외

| 상황 | 처리 |
| --- | --- |
| 비로그인 | 로그인 유도 |
| DeepSeek 응답 지연 | 5초 이상 첫 토큰이 없으면 안내 메시지 표시 |
| SSE 중단 | 부분 답변 유지, 재시도 제공 |
| RAG 출처 없음 | 출처 없음 안내 후 답변 신뢰도 주의 메시지 표시 |
| 완료 이벤트 실패 | 세션 상태 확인 후 재시도 버튼 제공 |

### 완료 기준

- 질문에 대한 답변이 SSE로 스트리밍된다.
- 답변 하단에 출처가 표시된다.
- `묵상 완료` 후 오늘 QT 묵상 기록에 AI 요약 연결 흐름으로 이어진다.

## FS-07. 묵상 기록 관리

### 목적

사용자가 오늘 본문을 읽고 느낀 점을 4개 필드로 정리해 개인 공간에 저장한다.

### 진입 조건

| 항목 | 내용 |
| --- | --- |
| Actor | 로그인 사용자 |
| 진입점 | 오늘의 QT, AI 세션 완료, 내 묵상 기록 |
| 인증 | 필수 |

### 기록 필드

| 필드 | 화면 라벨 | 필수 여부 | 설명 |
| --- | --- | --- | --- |
| `felt` | 오늘 느낀 점 | 선택 | 본문을 읽고 느낀 점 |
| `memorableVerse` | 기억하고 싶은 구절 | 선택 | 마음에 남은 구절 |
| `application` | 내 삶에 적용할 점 | 선택 | 오늘 실천할 내용 |
| `prayer` | 기도 제목 | 선택 | 기도문 또는 기도 제목 |
| `visibility` | 공개 여부 | 필수 | `PRIVATE`, `ANONYMOUS_SHARED` |

### API

| 목적 | Method | Path | 인증 |
| --- | --- | --- | --- |
| 오늘 QT DRAFT 생성/조회 | POST | `/api/v1/journals/today` | 필수 |
| 목록 조회 | GET | `/api/v1/journals` | 필수 |
| 상세 조회 | GET | `/api/v1/journals/{id}` | 필수 |
| 수정 | PATCH | `/api/v1/journals/{id}` | 필수 |
| 삭제 | DELETE | `/api/v1/journals/{id}` | 필수 |
| 이벤트 로그 | GET | `/api/v1/journals/{id}/events` | 필수 |

### 생성·저장 정책

- 클라이언트는 자유 본문용 `POST /api/v1/journals`를 호출하지 않는다.
- 오늘 QT 화면 진입 시 `POST /api/v1/journals/today`를 호출해 해당 날짜의 DRAFT를 멱등 생성/조회한다.
- 사용자가 입력하는 동안 클라이언트는 `PATCH /api/v1/journals/{id}`를 debounce로 호출해 자동 저장한다.
- AI Service의 `ai.session.completed` 이벤트는 같은 오늘 QT DRAFT에 AI 요약과 세션 참조를 연결한다.
- 사용자 입력 글자 수 제한은 두지 않는다. 서버는 DB 타입 한계, 요청 크기 제한, 악성 요청 방어만 공통 정책으로 처리한다.

### 상태

| 상태 | 설명 |
| --- | --- |
| `DRAFT` | `POST /api/v1/journals/today`로 생성/조회되는 오늘 QT 초안 |
| `PUBLISHED` | 사용자가 내용을 저장 또는 발행한 상태 |
| `DELETED` | 사용자 삭제 또는 소프트 삭제 상태 |

### 동작 흐름

1. 사용자가 오늘 QT 화면에서 내 생각 기록 영역에 진입한다.
2. 앱은 `POST /api/v1/journals/today`로 DRAFT를 생성하거나 기존 DRAFT를 가져온다.
3. 사용자는 개인 기록 화면에서 4개 필드를 작성한다.
4. 입력 변경 시 앱은 `PATCH /api/v1/journals/{id}`를 자동 저장으로 호출한다.
5. 저장 후 기본 공개 상태는 `PRIVATE`를 유지한다.
6. 사용자가 원할 때만 FS-08 흐름으로 공유한다.

### 이벤트

| 이벤트 | 발행 조건 |
| --- | --- |
| `journal.created` | 오늘 QT 묵상 DRAFT 생성 완료 |
| `journal.updated` | 묵상 수정 완료 |
| `journal.deleted` | 묵상 삭제 완료 |
| `journal.creation.failed` | AI 완료 후 오늘 QT 묵상 요약 첨부 실패 |

### 예외

| 상황 | 처리 |
| --- | --- |
| DRAFT 생성 지연 | `묵상 기록을 준비 중입니다` 상태 표시 |
| DRAFT 생성/조회 실패 | 재시도 또는 고객 안내 메시지 표시 |
| 본인 기록 아님 | 403 처리 |
| 저장 중 충돌 | 최신 데이터를 다시 불러오고 사용자에게 재저장 안내 |

### 완료 기준

- 오늘 QT 화면에서 묵상 DRAFT가 멱등 생성/조회된다.
- 묵상 기록은 4개 필드로 수정할 수 있다.
- 별도 저장 버튼 없이 자동 저장된다.
- 모든 기록은 기본 `PRIVATE`다.
- 생성·수정·삭제 이벤트가 Kafka로 발행된다.

## FS-08. 선택적 익명 나눔

| 좋아요 수 | 공개 |
| 댓글 | 공개 |

### 예외

| 상황 | 처리 |
| --- | --- |
| 이미 공개된 기록 공개 요청 | 기존 share 정보를 반환하거나 멱등 처리 |
| 공개 취소 후 접근 | 404 또는 비공개 안내 |
| 본인 기록 아님 | 403 처리 |
| 신고 중복 | 중복 신고 안내 또는 멱등 처리 |

### 완료 기준

- 비공개 기록은 나눔 목록에 노출되지 않는다.
- 공개·취소가 즉시 목록에 반영된다.
- 익명 공유 화면에 작성자 정보가 표시되지 않는다.
- 좋아요, 댓글, 신고 최소 기능이 동작한다.

| `journal.updated` | 필요 시 생략 가능 |
| 신고 접수 | 관리자 확인 대상 |

## FS-11. 교회 인증

### 목적

교회 인증은 사용자 신뢰 표식으로 검토하되, MVP의 필수 진입 조건으로 두지 않는다.

### MVP 정책

| 항목 | 처리 |
| --- | --- |
| 앱 사용 조건 | 교회 인증 없이 사용 가능 |
| 회원가입 UI | 선택 버튼만 둘 수 있음 |
| 인증 API | 조사 후 있으면 구현, 없으면 패스 |
| 커뮤니티 활용 | 인증 시 마크 표시 정도만 후속 검토 |

### 완료 기준

- 교회 인증이 없어도 회원가입과 오늘 QT 사용이 가능하다.
- API가 확인되지 않으면 기능 구현을 막지 않는다.

## 8. MVP 완료 체크리스트

- [ ] 비로그인 사용자가 온보딩과 오늘의 QT 본문을 볼 수 있다.
- [ ] 로그인 사용자가 주석과 AI 질문을 사용할 수 있다.
- [ ] AI 답변은 SSE로 스트리밍되고 출처가 표시된다.
- [ ] 사용자가 오늘 QT 화면에 진입하면 DRAFT 묵상 기록이 생성 또는 조회된다.
- [ ] 묵상 기록은 `felt`, `memorableVerse`, `application`, `prayer` 4필드로 저장된다.
- [ ] 묵상 기록은 별도 저장 버튼 없이 자동 저장된다.
- [ ] 묵상 기록은 기본 `PRIVATE`다.
- [ ] 사용자가 선택한 기록만 `ANONYMOUS_SHARED`로 공개된다.
- [ ] 익명 나눔에서 작성자 식별 정보가 노출되지 않는다.
- [ ] 개역개정, ESV, NIV, 찬양 가사·음원 저장을 하지 않는다.
- [ ] OpenAPI 명세와 구현 필드명이 일치한다.

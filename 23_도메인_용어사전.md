## 1. QT(큐티)란 무엇인가

**QT (Quiet Time, 큐티)**는 성경 말씀 안에서 읽기와 묵상으로 하나님과 교제하는 개인 신앙 훈련이다.

| 항목 | 설명 |
| --- | --- |
| 정의 | 성경 본문을 읽고, 관찰·해석·묵상을 통해 하나님의 뜻을 발견하고 삶에 적용하는 과정 |
| 목적 | 말씀을 통해 하나님을 알아가고, 하나님과 더 깊은 사귐(교제)을 갖는 것 |
| 성경적 근거 | 딤후 3:16-17 / 수 1:8 / 시 119:97 |
| 주기 | 매일 (일상의 영적 훈련) |
| 구조 | 찬양/기도 → 성경 읽기 → 관찰/해석/묵상 → 적용 → 기도 → 나눔 |

> ⚠️ **중요:** QT는 "기분 좋은 영적 감상"이 아니다. 본문이 전달하는 **객관적 사실(관찰)**과 **의미(해석)**를 먼저 파악한 뒤, 주관적 **느낌**과 **적용**으로 이어지는 구조적 과정이다.

## 2. QT 유형 (A~D형)

QT 노트는 포함하는 단계에 따라 4가지 유형으로 나뉜다. **QT-AI 앱의 핵심 목표는 D형 QT를 지원하는 것이다.**

| 유형 | 포함 단계 | 특징 |
| --- | --- | --- |
| **A형** | 느낌만 | QT 입문자. 본문을 읽고 마음에 와닿은 것만 기록 |
| **B형** | 내용관찰 + 느낌 | 본문을 3회 이상 읽고 자신의 말로 풀어 쓴 후 느낌 기록 |
| **C형** | 내용관찰 + 느낌 + 결단·적용 | B형에 구체적 적용(행동 결단)을 추가 |
| **D형** | 내용관찰 + 연구와묵상(해석) + 느낌 + 결단·적용 | C형에 연구와 묵상(해석 단계)을 추가. 가장 심화된 형태. 귀납적 성경공부 수준에 도달 가능 |

> **코드 매핑:** `JOURNALS.qt_type VARCHAR(2)` — 값: `'A'`, `'B'`, `'C'`, `'D'`  
> Java enum: `JournalType { A, B, C, D }`

## 3. D형 QT 4단계 상세

D형 QT는 **관찰 → 해석 → 느낌 → 적용** 4단계로 구성된다.

### 3.1 1단계: 관찰 (Observation)

| 항목 | 설명 |
| --- | --- |
| **정의** | 성경 본문이 전달하고 있는 사실(Fact)을 찾아내는 것 |
| **성격** | 객관적 작업. 내 생각·감정·경험을 배제하고 본문이 말하는 바를 파악 |
| **방법** | 단락 나누기, 재진술, 육하원칙(5W1H) 분석, 주어·동사·명령형 표현 찾기, 반복·대조·비교 표현 탐색 |
| **주의** | 관찰이 목적이 아니라 해석·묵상의 **기초** 역할. 잘 아는 본문이라도 선입견 없이 세밀하게 관찰 |
| **DB 컬럼** | `JOURNALS.observation TEXT` |
| **AI 코칭 방향** | 육하원칙 질문으로 본문 탐색 유도, 관찰 누락 부분 보완 질문 |

### 3.2 2단계: 해석 / 연구와묵상 (Interpretation)

| 항목 | 설명 |
| --- | --- |
| **정의** | 관찰을 기초로 성경 본문이 전달하는 **의미(Meaning)**를 찾아내는 것 |
| **성격** | 객관적 연구 + 주관적 묵상의 결합 |
| **구성** | ① **연구**: 사전·주석 등 외부 자료를 통해 본문 배경·인물·사건·단어 조사 / ② **묵상**: 외부 자료를 근거로 내부에서 이해와 설명 추구. "하나님은 어떤 분이십니까?" 질문 중심 |
| **핵심 질문** | "이 본문에서 하나님(예수님/성령님)은 어떤 분이십니까?" |
| **소재** | 인물, 사건, 장소, 시기, 배경, 어려운 단어 등 |
| **DB 컬럼** | `JOURNALS.interpretation TEXT` |
| **AI 코칭 방향** | RAG로 주석·배경 정보 제공, "하나님은 어떤 분인가?" 깊이 있는 질문 유도 |

> ⚠️ **주의:** 하나님이 직접 등장하지 않는 본문에서도 하나님의 성품과 사역을 찾을 수 있어야 한다.

### 3.3 3단계: 느낌 (Feeling / Response)

| 항목 | 설명 |
| --- | --- |
| **정의** | 관찰·해석을 통해 발견된 진리나 교훈에 대해 **우리의 마음(Mind)속에서 일어나는 모든 것** |
| **성격** | 주관적 작업. 관찰·해석이 성경(말씀)을 다루는 것이라면, 느낌은 **삶**을 다루는 것 |
| **종류** | 도전, 반성, 깨달음, 회개, 감사, 위로, 경고 등 |
| **DB 컬럼** | `JOURNALS.felt TEXT` |
| **AI 코칭 방향** | 공감 응답, 느낌을 더 구체화하도록 유도, 성령의 역사 방향 제시 |

### 3.4 4단계: 적용 (Application)

| 항목 | 설명 |
| --- | --- |
| **정의** | 느낌을 통해 발견한 교훈을 **실제 삶의 행동(Action)**으로 어떻게 적용할지 결단하고 계획하는 것 |
| **성격** | 느낌이 마음(내면)에 관련된 부분이라면, 적용은 **행동**에 관련된 부분 |
| **원칙** | ① 반드시 **나(I)**에게 하는 것 / ② **구체적**이어야 함 (언제, 누구에게, 어떻게) / ③ 느낌과 일관성 있게 연결 |
| **나쁜 예** | "더 열심히 기도해야겠다" → 추상적, 측정 불가 |
| **좋은 예** | "이번 주 금요일 점심에 동료 ○○와 만나서 복음을 나눈다" |
| **DB 컬럼** | `JOURNALS.application TEXT` |
| **AI 코칭 방향** | 적용의 구체성 검증 질문 ("언제?", "누구에게?", "어떻게?") |

## 4. D형 QT 실천 전체 흐름

```
┌──────────────────┬──────────────────────────────────────────────────┐
│  단계            │  내용                                             │
├──────────────────┼──────────────────────────────────────────────────┤
│ 1. 찬양과 기도   │  마음을 열고 고민과 염려를 내려놓음.             │
│                  │  하나님을 알아가고 더 깊은 사귐을 구하는 기도    │
├──────────────────┼──────────────────────────────────────────────────┤
│ 2. 성경 읽기     │  정해진 분량을 2~3회 반복 읽기                   │
│                  │  1회: 전체 내용 파악 / 2회: 육하원칙 분석         │
│                  │  3회: 주어·동사·명령형·반복 표현 탐색             │
├──────────────────┼──────────────────────────────────────────────────┤
│ 3. 관찰(연구)    │  단락 나누기 → 재진술 → 한 문장 요약             │
│   [DB: observation] │  육하원칙, 인물·사건·배경·어려운 단어 분석    │
├──────────────────┼──────────────────────────────────────────────────┤
│ 4. 해석/묵상     │  연구: 주석·사전 등 외부 자료 참고               │
│   [DB: interpretation] │  묵상: 하나님은 어떤 분이십니까?          │
│                  │        내게 주시는 교훈과 도전은 무엇인가?        │
├──────────────────┼──────────────────────────────────────────────────┤
│ 5. 느낌          │  도전·반성·깨달음·회개·감사 등 마음의 반응        │
│   [DB: felt]     │                                                   │
├──────────────────┼──────────────────────────────────────────────────┤
│ 6. 적용          │  나에게, 구체적으로, 실천 가능한 행동 결단        │
│   [DB: application] │                                               │
├──────────────────┼──────────────────────────────────────────────────┤
│ 7. 기도          │  말씀을 통한 도전에 믿음으로 응답하는 기도        │
│   [DB: prayer]   │  하나님 나라·공동체·이웃을 위한 기도              │
├──────────────────┼──────────────────────────────────────────────────┤
│ 8. 나눔          │  묵상 말씀과 은혜를 믿음의 동역자들과 나눔        │
│   [DB: JOURNAL_SHARES] │  → 앱: 익명 나눔 기능                     │
└──────────────────┴──────────────────────────────────────────────────┘
```

## 5. 단계별 핵심 개념 용어

### 5.1 관찰 단계 용어

| 용어 | 정의 |
| --- | --- |
| **단락 나누기** | 본문을 내용의 흐름에 따라 의미 단위로 구분하는 것. 절 번호 범위와 내용 요약으로 표현 |
| **재진술 (Restatement)** | 본문 내용을 자신의 말로 풀어 쓰는 것. 외우기식 복사가 아닌 이해를 통한 재표현 |
| **한 문장 요약** | 단락 나누기와 재진술을 통해 본문 전체를 핵심 한 문장으로 요약 |
| **육하원칙 (5W1H)** | 누가(Who), 무엇을(What), 언제(When), 어디서(Where), 왜(Why), 어떻게(How) — 관찰의 기본 도구 |

### 5.2 해석 단계 용어

| 용어 | 정의 |
| --- | --- |
| **연구 (Research)** | 사전·주석·성경 번역본 등 **외부 자료**를 이용해 본문의 배경·인물·단어를 조사하는 것 |
| **묵상 (Meditation)** | 연구를 통해 찾은 외부 자료를 근거로 **내부에서 이해와 설명을 추구**하는 것. 자료 없는 감상은 묵상이 아님 |
| **하나님은 어떤 분이십니까?** | 해석 단계의 핵심 질문. 본문에서 드러나는 하나님(삼위일체)의 성품과 사역을 찾는 것 |
| **귀납적 성경공부** | 본문 → 관찰 → 해석 → 적용 순서로 성경을 접근하는 방법. D형 QT가 이 수준에 해당 |

### 5.3 공통 용어

| 용어 | 정의 |
| --- | --- |
| **성경 본문 (Passage)** | QT의 대상이 되는 성경 구절 범위. 예: "창세기 1:1~5", "사도행전 16:25~34" |
| **기억할 구절 (Memorable Verse)** | 본문 중 특히 마음에 와닿아 외우거나 간직하고 싶은 특정 구절 |
| **나눔 (Sharing)** | 묵상한 말씀과 은혜를 공동체(믿음의 동역자들)와 함께 나누는 것. 앱에서는 익명 공개 기능 |
| **묵상 노트 (Journal)** | QT 과정을 기록한 노트. 앱에서는 `JOURNALS` 테이블. A~D형 중 선택 |

## 6. 그리스도 중심 묵상 원칙

> **출처:** "말씀과 묵상이란" 교육 자료 — 최현준 목사

성경 전체는 예수 그리스도를 가리킨다. 구약은 오실 그리스도를, 신약은 오신 그리스도와 다시 오실 그리스도를 전한다.

### 6.1 묵상의 4요소 (ABCD 모델)

```
A. 성경 (Scripture)  ──→  B. 연구 (Research)
        │                          │
        ▼                          ▼
D. 적용 (Application) ←──  C. 그리스도 (Christ)
```

| 순서 | 이름 | 설명 |
| --- | --- | --- |
| A → D | 개인의 주관적 묵상 | 성경을 읽고 바로 적용. D형 QT에서 지양해야 할 패턴 |
| A → B → D | 윤리적 적용 | 연구는 하되 그리스도 중심성 누락. 도덕 교훈에 그침 |
| A → B → C → D | 종교적 신앙으로 회귀 | 순서 자체는 맞지만 개선 여지 있음 |
| **A → B → D → C** ✅ | **복음을 이해하고 복음으로 사는 것** | **권장 순서.** 성경 읽기 → 연구 → 적용 결단 → 그리스도 중심 점검 |

> **AI 코칭 프롬프트 설계 시 이 순서를 반영해야 한다.** 사용자가 적용을 먼저 쓰더라도 마지막에 그리스도 중심 관점으로 재확인하도록 안내.

## 7. 앱 도메인 용어 ↔ DB/코드 매핑

| 도메인 용어 | DB 컬럼 | Java 타입 | 설명 |
| --- | --- | --- | --- |
| QT 유형 | `JOURNALS.qt_type VARCHAR(2)` | `JournalType { A, B, C, D }` | 사용자가 선택한 QT 유형 |
| 관찰 내용 | `JOURNALS.observation TEXT` | `String` (nullable) | 1단계: 단락나누기·재진술 포함 |
| 해석/연구와묵상 | `JOURNALS.interpretation TEXT` | `String` (nullable) | 2단계: 연구 결과 + 하나님은 어떤 분인가 |
| 느낀 점 | `JOURNALS.felt TEXT` | `String` (nullable) | 3단계: 마음의 반응 |
| 기억할 구절 | `JOURNALS.memorable_verse TEXT` | `String` (nullable) | 관찰 단계에서 선택하는 인상 깊은 구절 |
| 적용할 점 | `JOURNALS.application TEXT` | `String` (nullable) | 4단계: 구체적 행동 결단 |
| 기도 내용 | `JOURNALS.prayer TEXT` | `String` (nullable) | 마무리 기도 |
| QT 안내 단계 | `AI_TURNS.guide_step VARCHAR(2)` | `String` (nullable) | AI가 현재 안내 중인 QT 유형 단계 |
| 묵상 상태 | `JOURNALS.status VARCHAR(20)` | `JournalStatus { DRAFT, PUBLISHED, DELETED }` | DRAFT: 작성 중 / PUBLISHED: 완료 |

> ⚠️ **혼동 주의:** `AI_TURNS.guide_step`의 `'A'~'D'`는 **QT 유형(Type)**을 의미하고, QT 내부 진행 단계(관찰→해석→느낌→적용)는 AI 세션 컨텍스트로 별도 관리한다.

## 8. 자주 혼동하는 개념 구분

### 8.1 묵상 ≠ 느낌

| 구분 | 묵상 (Meditation) | 느낌 (Feeling) |
| --- | --- | --- |
| 위치 | 2단계 해석 안에 포함 | 3단계 독립 단계 |
| 성격 | 연구 자료를 근거로 한 **이해** | 발견된 진리에 대한 **마음의 반응** |
| 올바른 표현 | "이 본문에서 하나님은 ○○한 분이심을 알 수 있다" | "이 말씀이 내게 도전이 된다. 나는 ○○해야 한다고 느꼈다" |

### 8.2 QT 유형(Type) ≠ QT 진행 단계(Stage)

| 구분 | QT 유형 (Type) | QT 진행 단계 (Stage) |
| --- | --- | --- |
| 값 | A, B, C, D | 관찰 / 해석 / 느낌 / 적용 |
| 의미 | 묵상 노트에 포함하는 섹션의 종류 | 현재 AI 코칭이 안내 중인 단계 |
| DB/코드 | `JOURNALS.qt_type` / `JournalType` | `AI_TURNS.guide_step` 또는 세션 컨텍스트 |

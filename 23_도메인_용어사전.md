## 1. 목적

이 문서는 QT-AI 프로젝트에서 반복해서 사용하는 용어의 의미와 사용 범위를 고정한다.

기능 요구사항의 원본은 `07_요구사항_정의서.md`가 책임진다. 이 문서는 요구사항을 새로 정의하지 않고, 문서와 코드에서 같은 단어를 같은 의미로 쓰기 위한 기준만 관리한다.

## 2. 용어 사용 원칙

| 원칙 | 기준 |
| --- | --- |
| 기준 문서 우선 | 용어 해석이 충돌하면 `07_요구사항_정의서.md` v2.3을 우선한다. |
| 사용자 노출 용어 우선 | 사용자에게 보이는 용어는 쉬운 한국어를 우선한다. |
| 내부 용어와 UI 용어 분리 | 내부 검증·저장 용어와 사용자 화면 용어를 혼용하지 않는다. |
| 금지 표현 유지 | 제외된 기술이나 기능은 허용처럼 쓰지 않고, 제외·보류·금지 문맥으로만 쓴다. |
| 저작권 표현 주의 | "저작권 문제 없음"이라고 쓰지 않고, "저작권 리스크를 낮춘다"라고 쓴다. |

## 3. 제품·사용자 용어

| 용어 | 영문/코드 표현 | 정의 | 사용 기준 |
| --- | --- | --- | --- |
| QT | Quiet Time | 성경 본문을 읽고 묵상하고 기록하는 개인 경건 시간 | 사용자 안내에서는 "QT"와 쉬운 설명을 함께 제공 |
| 오늘 QT | Today QT | 당일 기준으로 제공되는 QT 본문, 해설 진입점, 묵상 진입점, 시뮬레이터 상태 묶음 | 별도 홈 화면 없이 첫 진입의 중심 화면 |

| QT 본문 범위 | QT range | 오늘 QT에 해당하는 성경 책, 장, 시작 절, 끝 절 정보 | 성서 유니온에서 범위 정보만 수집 |
| 공개 시각 | Published time | 성서 유니온이 당일 QT 범위를 공개하는 시각 | 00:00 KST로 통일 |
| 수집 배치 시각 | Collection batch time | 우리 시스템이 QT 본문 범위 정보를 수집하는 시각 | 04:00 KST로 통일 |
| 00:00~04:00 구간 | Pre-collection window | 새 범위 수집과 AI 산출물이 완료되기 전 구간 | 이전에 준비된 캐시를 조회할 수 있음 |
| 본문 | Bible text | 자체 DB에 적재된 한/영 성경 본문 | QT 범위에 맞춰 DB에서 조회 |
| 해설 | Explanation | 사용자가 보는 쉬운 한국어 설명, 요약, 배경, 용어 풀이 | UI/API/사용자 문서에서 사용 |
| 주석 | Commentary | 내부 검증·소스 자료를 가리키는 용어 | 사용자 노출 용어로 사용하지 않음 |
| 출처 | Sources | 해설 생성에 사용된 공개 가능한 근거 메타데이터 | API 필드명은 `sources` |
| 묵상 노트 | Journal | 사용자가 느낀 점, 기억할 구절, 적용할 점, 기도를 기록하는 개인 기록 | 기본값 비공개 |
| 묵상 DRAFT | Journal Draft | 오늘 QT에 대해 생성되거나 불러오는 임시 묵상 기록 | 생성/저장은 로그인 필요 |
| 묵상 달력 | Journal Calendar | 월별로 묵상한 날을 표시하고 날짜별 기록으로 이동하는 마이페이지 기능 | MVP In-Scope |
| 익명 나눔 | Anonymous Sharing | 사용자가 선택한 경우에만 묵상 일부를 익명 형태로 공개하는 기능 | 기본 공개가 아님 |
| 찬양 큐레이션 | Curated Songs | 운영자가 사전에 등록한 찬양 곡 메타데이터 목록 | AI 추천이 아님 |
| 내 찬양 목록 | My Songs | 사용자가 큐레이션 곡을 저장한 개인 목록 | 가사·음원 저장이 아니라 곡 참조 저장 |
| 소프트 로그인 | Soft Login | 첫 진입은 막지 않고, 저장·해설 열람 등 필요한 기능에서 로그인으로 유도하는 정책 | 첫 앱 진입 강제 로그인 없음 |

| 배치 | Batch Job | 정해진 시각 또는 관리자 트리거로 실행되는 백엔드 작업 | 사용자 요청 경로가 아님 |
| 관리자 트리거 | Admin Trigger | 관리자가 특정 본문의 재생성을 요청하는 운영 기능 | 실제 실행은 `SYSTEM` 작업으로 기록 |

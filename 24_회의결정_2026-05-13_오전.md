## 1. MVP 확정 사항

| 항목 | 결정 | 반영 위치 |
| --- | --- | --- |
| AI·묵상 연결 범위 | MVP에서는 AI 질문과 묵상 기록을 오늘의 QT 본문에만 연결한다. 일반 성경 보기는 책·장·절 조회 전용이다. | 요구사항, 기능 명세, AI/Bible OpenAPI, 개인 일정표 |
| 오늘의 QT 본문 단위 | MVP는 한 절 고정으로 제공한다. DB/API는 확장을 위해 `verseStart`, `verseEnd`를 사용하되 MVP에서는 두 값이 항상 같다. | BFF OpenAPI, 관리자 QT 본문 관리, ERD |
| 본문 설명 | Bible Service DB에 미리 저장한다. 구성은 쉬운 요약, 배경 설명, 어려운 단어, 출처다. 적용 질문은 AI가 담당한다. | Bible ERD, Bible OpenAPI, Flutter 화면 |
| 주석 저작권 | 사용 가능한 1개 출처 중심으로 조사한다. 저작권이 불명확한 한국어 상업 주석은 사용하지 않는다. | 데이터 시드, 저작권 표 |
| AI 응답 | 출처 기반 1회성 Q&A다. 긴 대화 맥락은 유지하지 않고, 이상 응답 디버깅과 관리자 모니터링을 위해 응답을 저장한다. | AI OpenAPI, AI ERD, 관리자 기능 |
| 묵상 기록 | `felt`, `memorableVerse`, `application`, `prayer` 4필드로 저장한다. 사용자 입력 글자 수 제한은 두지 않고 자동 저장한다. | Bible OpenAPI, ERD, Flutter |
| 공개·커뮤니티 | 기본 비공개다. 공개/비공개, 좋아요, 댓글, 신고는 MVP 포함이다. 세분화 공개 범위, 팔로우, 랭킹 피드는 후속 버전이다. | Bible OpenAPI, 관리자 신고, Flutter |
| 찬양 | AI가 오늘 QT 주제 기반으로 곡을 추천한다. 사용자는 추천곡을 저장/제거한다. 직접 YouTube URL 입력, 가사·음원 저장, 스트리밍 연동은 제외한다. | 요구사항, 기능 명세, Flutter |
| 교회 인증 | MVP 기본 범위에서는 제외한다. 회원가입 화면에 선택 버튼만 둘 수 있고, API가 있으면 구현하고 없으면 패스한다. 인증이 없어도 앱을 사용할 수 있어야 한다. | Gateway/Auth, Flutter |
| 초기 화면 | 별도 홈 화면 없이 오늘 QT 화면으로 바로 진입한다. 필요해지면 홈을 추가하고 내비게이션을 연결한다. | Flutter 가이드, 기능 명세 |
| 로딩 전략 | 오늘 QT 데이터 먼저 로딩해 화면에 진입한다. 나머지 성경 데이터는 백그라운드에서 로딩한다. | Flutter 가이드, BFF |

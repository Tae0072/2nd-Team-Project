## 1. 문서 목적과 경계

이 문서는 `07_요구사항_정의서.md`의 F-ID를 구현 가능한 기능 단위로 풀어쓴다. 요구사항 포함 여부를 바꾸지 않는다.

| 구분 | 이 문서에서 관리 | 다른 문서에서 관리 |
| --- | --- | --- |
| 상세 동작 | 포함 | 요구사항 원본은 `07` |
| API 진입점 | 요약만 포함 | 상세 요청/응답은 `04` |
| 데이터 참조 | 요약만 포함 | 테이블 기준은 `02` |
| 내부 Port | 요약만 포함 | 도메인 경계와 Port 기준은 `03` |
| 화면 연결 | 요약만 포함 | 화면별 상태는 `06` |

## 2. 기능 공통 규칙

| 항목 | 기준 |
| --- | --- |
| 외부 API | `/api/v1/**` prefix를 사용한다. |
| 내부 호출 | Java Interface로 처리하고, 다른 도메인의 Entity/Service/Repository를 직접 import하지 않는다. |
| AI 호출 | 사용자 요청 경로에서 LLM을 호출하지 않는다. 04:00 KST 배치 또는 관리자 트리거만 허용한다. |
| 소프트 로그인 | 첫 진입, 튜토리얼, 성경 본문 보기, Today QT 미리보기, 찬양 큐레이션 조회는 비로그인 허용이다. |
| 로그인 필요 | 묵상 저장, 해설 본문 열람, 찬양 저장, 묵상 달력 조회는 로그인 필요다. |
| 시뮬레이터 | `READY`만 버튼 활성화, `MISSING`/`FAILED`/`DISABLED`는 비활성화한다. |
| Today QT 캐시 | 응답에는 `cache.cacheStatus`, `servedQtDate`, `staleReason`, `nextRefreshAt`을 포함한다. |
| 익명 나눔 | 사용자가 명시적으로 공유한 묵상만 노출하고 작성자 식별 정보는 응답하지 않는다. |
| 찬양 | 운영자 큐레이션과 사용자 저장만 제공하며 AI 추천, 가사·음원 저장, 직접 YouTube URL 입력은 제외한다. |
| 관리자 | `/api/v1/admin/**`는 `ADMIN` 권한이 필요하고 데이터 변경은 감사 로그 대상이다. |
| 저작권 표현 | “저작권 문제 없음”이 아니라 “저작권 리스크를 낮춘다”로 표현한다. |

## 3. 기능 상세

| F-ID | 기능 | 주요 동작 | 진입점 | 예외·비활성 조건 | 완료 기준 |
| --- | --- | --- | --- | --- | --- |
| F-01 | 성경 묵상 화면 | Today QT 본문, 한/영 본문, 해설 진입점, 묵상 진입점, 시뮬레이터 상태, 캐시 상태를 조립한다. | `GET /api/v1/qt/today`, `GET /api/v1/bible/passages` | 00:00~04:00은 `STALE_FALLBACK`으로 이전 캐시 제공 가능. 캐시가 전혀 없으면 `EMPTY` 준비 중 응답 | Today QT 응답이 본문·해설 진입점·묵상 진입점·시뮬레이터 상태와 `cache.cacheStatus`를 항상 반환 |
| F-02 | AI 콘텐츠 사전 생성 | B 테이블 입력으로 C 해설을 생성하고 편집자 검증 후 저장한다. 시뮬레이터 산출물도 사전 생성한다. | 04:00 KST batch, `POST /api/v1/admin/ai/regenerate` | 사용자 요청 경로 LLM 호출 금지. A 테이블 사용자 노출 금지. 실패는 로그와 재처리 가능 상태로 남김 | C 해설 또는 실패 로그가 기록되고, Today QT 캐시와 AI run 상태가 갱신됨 |
| F-03 | 묵상 노트 | 오늘 QT 기준 DRAFT 생성/조회, 목록·상세 조회, 수정, 삭제, 이벤트 기록, 달력 반영 | `POST /api/v1/journals/today`, `GET /api/v1/journals`, `GET/PATCH/DELETE /api/v1/journals/{id}`, `GET /api/v1/journals/{id}/events`, `GET /api/v1/journals/calendar` | 비로그인 사용자는 저장 불가. 자유 본문 묵상 생성 불가 | 4개 섹션 중 1개 이상 입력 시 저장 가능, 변경 이벤트와 달력 상태가 기록됨 |
| F-04 | 회원 인증 | Google OAuth, JWT 발급·검증, Refresh Token 관리, 소프트 로그인 유지 | `GET /api/v1/auth/google/authorize`, `POST /api/v1/auth/google/token`, `POST /api/v1/auth/refresh`, `POST /api/v1/auth/logout`, `GET /api/v1/me` | 교회 인증 버튼/API 없음. 계정 탈퇴 MVP 제외 | Access Token 30분, Refresh Token 14일 기준 동작 |

| F-06 | 관리자 운영 | QT 범위, 성경 데이터 상태, C 해설, AI 로그, 시뮬레이터 상태, 신고, 찬양 큐레이션, 감사 로그 관리 | `/api/v1/admin/dashboard`, `/api/v1/admin/qt-ranges/**`, `/api/v1/admin/bible/**`, `/api/v1/admin/explanations/**`, `/api/v1/admin/ai/**`, `/api/v1/admin/simulator/**`, `/api/v1/admin/reports/**`, `/api/v1/admin/songs/**`, `/api/v1/admin/audit-logs` | `ADMIN` 외 접근 차단. 관리자 변경은 감사 로그 필수. 사용자·공지 관리는 P1 슬롯 | 관리자 주요 작업이 `ADMIN` 권한과 audit log 기준을 만족 |

| F-12 | 시뮬레이터 보기 | Today QT 본문 기준 사전 생성 클립 상태와 재생 진입점 제공 | `GET /api/v1/simulator/{bookCode}/{ch}/{v}` | `READY` 외 상태는 버튼 비활성화 | 상태값 4종을 정확히 반환하고 `READY`에서만 클립 조회 가능 |

## 4. 도메인별 구현 단위

| 도메인 | 기능 | 공개 Port 또는 API |
| --- | --- | --- |
| `bff` | Today QT 응답 조립, 화면 단위 UseCase, 관리자 API 조립 | 외부 Controller, 응답 DTO |
| `gatewayauth` | JWT, Google OAuth, 권한 검증 | `AuthCommandPort`, `AuthQueryPort` |
| `bible` | 성경 본문, QT 범위, Today QT 캐시, C 해설, journal, anonymous share, songs | `BibleQueryPort`, `ExplanationQueryPort`, `Journal*Port`, `Share*Port`, `Song*Port` |
| `ai` | AI 실행, C 해설 생성, 검증 로그 | `AiRunLogPort`, `AiRegenerateCommandPort` |
| `simulator` | 시뮬레이터 상태·클립 저장/조회 | `SimulatorQueryPort`, `SimulatorClipSavePort` |
| `admin` | 감사 로그, 운영 대시보드 조회 조립 지원 | `AdminAuditCommandPort`, `AdminDashboardQueryPort` |

`journal`과 `songs`는 `bible` 내부 하위 모듈이며 별도 최상위 도메인이나 독립 서비스로 만들지 않는다. 익명 나눔은 `journal` 하위 책임으로 둔다. `admin`은 별도 서비스가 아니라 단일 `qtai-server` 내부 지원 모듈이다.

## 5. 완료 기준 요약

| 기준 | 완료 조건 |
| --- | --- |
| 요구사항 정합성 | 모든 기능이 `07` F-ID와 연결됨 |
| API 정합성 | 외부 API가 `04`의 권한·prefix 기준을 따름 |
| 데이터 정합성 | 주요 저장 데이터가 `02` 테이블 기준과 충돌하지 않음 |
| 화면 정합성 | 사용자 노출 상태가 `06` 화면 기준과 충돌하지 않음 |
| 시퀀스 정합성 | Today QT, 익명 나눔, 관리자 운영 흐름이 `05`와 충돌하지 않음 |
| Port 정합성 | 내부 호출이 `03`의 Java Interface 기준과 충돌하지 않음 |
| 캐시 정합성 | Today QT 완료 기준이 `cacheStatus`와 00:00~04:00 정책을 포함함 |
| 품질 정합성 | 금지 기술·금지 API·금지 데이터가 없음 |

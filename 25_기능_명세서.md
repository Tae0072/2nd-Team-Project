## 1. 문서 목적과 경계

이 문서는 `07_요구사항_정의서.md`의 F-ID를 구현 가능한 기능 단위로 풀어쓴다. 요구사항 포함 여부를 바꾸지 않는다.

| 구분 | 이 문서에서 관리 | 다른 문서에서 관리 |
| --- | --- | --- |
| 상세 동작 | 포함 | 요구사항 원본은 `07` |
| API 진입점 | 요약만 포함 | 상세 요청/응답은 `04` |
| 데이터 참조 | 요약만 포함 | 테이블 기준은 `02` |
| 화면 연결 | 요약만 포함 | 화면별 상태는 `06` |

## 2. 기능 공통 규칙

| 항목 | 기준 |
| --- | --- |
| 외부 API | `/api/v1/**` prefix를 사용한다. |
| 내부 호출 | Java Interface로 처리하고, 다른 도메인의 Entity/Service/Repository를 직접 import하지 않는다. |
| AI 호출 | 사용자 요청 경로에서 LLM을 호출하지 않는다. 04:00 KST 배치 또는 관리자 트리거만 허용한다. |
| 소프트 로그인 | 첫 진입, 튜토리얼, 성경 본문 보기, Today QT 미리보기, 찬양 큐레이션 조회는 비로그인 허용이다. |
| 로그인 필요 | 묵상 저장, 해설 본문 열람, 찬양 저장, 묵상 달력 조회는 로그인 필요다. |
| 시뮬레이터 | `READY`만 버튼 활성화, `MISSING`/`FAILED`/`DISABLED`는 비활성화한다. |

## 3. 기능 상세

| F-ID | 기능 | 주요 동작 | 진입점 | 예외·비활성 조건 | 완료 기준 |
| --- | --- | --- | --- | --- | --- |
| F-01 | 성경 묵상 화면 | Today QT 본문, 한/영 본문, 해설 진입점, 묵상 진입점, 시뮬레이터 상태를 조립한다. | `GET /api/v1/qt/today`, `GET /api/v1/bible/passages` | 00:00~04:00은 이전 캐시 제공. 본문 범위 수집 실패 시 마지막 정상 캐시 유지 | Today QT 응답이 본문·해설 진입점·묵상 진입점·시뮬레이터 상태를 항상 반환 |
| F-02 | AI 콘텐츠 사전 생성 | B 테이블 입력으로 C 해설을 생성하고 편집자 검증 후 저장한다. 시뮬레이터 산출물도 사전 생성한다. | 04:00 KST batch, `POST /api/v1/admin/ai/regenerate` | 사용자 요청 경로 LLM 호출 금지. A 테이블 사용자 노출 금지 | C 해설 또는 실패 로그가 기록되고, Today QT 상태가 갱신됨 |
| F-03 | 묵상 노트 | 오늘 QT 기준 DRAFT 생성/조회, 수정, 삭제, 이벤트 기록, 달력 반영 | `POST /api/v1/journals/today`, `PATCH /api/v1/journals/{id}` | 비로그인 사용자는 저장 불가. 자유 본문 묵상 생성 불가 | 4개 섹션 중 1개 이상 입력 시 저장 가능, 변경 이벤트 기록 |
| F-04 | 회원 인증 | Google OAuth, JWT 발급·검증, Refresh Token 관리, 소프트 로그인 유지 | OAuth callback, token refresh | 교회 인증 버튼/API 없음. 계정 탈퇴 MVP 제외 | Access Token 30분, Refresh Token 14일 기준 동작 |
| F-05 | 사용자 알림 | 이벤트 기반 알림 생성 후 인앱 알림 목록에서 조회 | `GET /api/v1/me/notifications` | WebSocket/STOMP는 v1 제외 | 로그인 사용자가 알림 목록과 읽음 상태를 확인 가능 |
| F-06 | 관리자 운영 | QT 범위, 성경 데이터 상태, C 해설, AI 로그, 신고, 찬양 큐레이션, 감사 로그 관리 | `/api/v1/admin/**` | `ADMIN` 외 접근 차단. 관리자 변경은 감사 로그 필수 | 관리자 주요 작업이 `ADMIN` 권한과 audit log 기준을 만족 |
| F-07 | 첫 실행 튜토리얼 | QT 의미와 앱 사용 순서를 안내하고 Today QT로 진입 | 앱 첫 실행 | 로그인 wall로 막지 않음 | 튜토리얼 종료 후 Today QT 화면으로 이동 |
| F-08 | 쉬운 본문 요약·해설·용어 풀이 | C 테이블의 요약, 배경, 단어, 해설을 선택 열람 형태로 제공 | `GET /api/v1/explanations/{bookCode}/{ch}/{v}` | 비로그인 사용자는 실제 해설 본문 열람 전 로그인 유도 | A 테이블 노출 없이 C 해설과 `sources`만 표시 |
| F-09 | 찬양 큐레이션·내 찬양 목록 | 운영자 큐레이션 곡 조회, 사용자 저장/제거 | `GET /api/v1/songs/curated`, `/api/v1/me/songs` | AI 추천 금지, 가사·음원 저장 금지, 직접 URL 입력 MVP 제외 | 큐레이션 조회는 PUBLIC, 저장/제거는 USER로 동작 |
| F-10 | 선택적 익명 나눔 | 사용자가 선택한 묵상만 익명 공유하고 좋아요·댓글·신고를 제공 | 공유 관련 API는 구현 시 `04`에 추가 | 기본 비공개. 팔로우·랭킹·실시간 피드 제외 | 작성자 정보 없이 공유 글이 노출되고 신고가 관리자 대상으로 전달 |
| F-11 | 교회 인증 | 기능 없음 | 없음 | 회원가입 화면 버튼, API, DB 테이블 모두 금지 | 관련 UI/API/테이블이 존재하지 않음 |
| F-12 | 시뮬레이터 보기 | Today QT 본문 기준 사전 생성 클립 상태와 재생 진입점 제공 | `GET /api/v1/simulator/{bookCode}/{ch}/{v}` | `READY` 외 상태는 버튼 비활성화 | 상태값 4종을 정확히 반환하고 `READY`에서만 클립 조회 가능 |
| F-13 | 묵상 달력 | 월별 묵상 여부 표시, 날짜 클릭 시 해당 기록 조회 | `GET /api/v1/journals/calendar`, `GET /api/v1/journals?date=` | 로그인 필수. 기록 없는 날은 자동 생성하지 않음 | 묵상한 날짜 표시와 상세 이동이 정확히 동작 |

## 4. 도메인별 구현 단위

| 도메인 | 기능 | 공개 Port 또는 API |
| --- | --- | --- |
| `bff` | Today QT 응답 조립, 화면 단위 UseCase | 외부 Controller, 응답 DTO |
| `gatewayauth` | JWT, Google OAuth, 권한 검증 | 인증 검증 Port |
| `bible` | 성경 본문, QT 범위, C 해설, journal, songs | `BibleQueryPort`, `ExplanationQueryPort`, `Journal*Port`, `Song*Port` |
| `ai` | AI 실행, C 해설 생성, 검증 로그 | `AiRunLogPort`, 관리자 트리거 |
| `simulator` | 시뮬레이터 상태·클립 저장/조회 | `SimulatorQueryPort`, `SimulatorClipSavePort` |

`journal`과 `songs`는 `bible` 내부 하위 모듈이며 별도 최상위 도메인이나 독립 서비스로 만들지 않는다.

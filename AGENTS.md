## 1. Project Summary

QT-AI is a Quiet Time app for today's Bible passage, easy Korean explanations, personal notes, curated worship songs, nickname sharing, fact-based Q&A AI, simulator status, and admin operations.

- **Users:** `ANONYMOUS`, `USER`, `ADMIN`, `OPERATOR`, `REVIEWER`, `CONTENT_CREATOR`, `SUPER_ADMIN`, `SYSTEM_BATCH`
- **Source of truth:** `07_요구사항_정의서.md` v3.1
- **Repository role:** documentation/specification baseline only
- **Implementation target:** separate implementation GitHub with one `qtai-server` Modular Monolith + Flutter app/admin surface
- **Hard boundary:** do not create Spring Boot, Flutter, OpenAPI, DB, infra, data seed, or CI implementation skeletons in this repository.

## 3. Target Tech Stack

- **Backend:** Java 21, Spring Boot 3.3, Gradle, Spring Modulith, ArchUnit
- **Backend shape:** single `qtai-server`; no independent services
- **DB/cache:** MySQL 8.0, H2 for tests, Caffeine app cache, Redis for token/rate/idempotency after review
- **AI:** DeepSeek OpenAI-compatible client; batch/admin generation plus F-15 single-turn fact Q&A with validation
- **Frontend:** Flutter app; admin UI by agreed Flutter Web/admin surface
- **CI/deploy:** GitHub Actions, Spectral, JaCoCo, gitleaks, Docker Compose

## 4. Architecture Rules

Flutter App/Admin Web calls `/api/v1/**` on single `qtai-server`.
Domains: `member`, `bible`, `qt`, `study`, `note`, `sharing`, `report`, `notification`, `praise`, `mission`, `ai`, `admin`, `audit`.

- `note`, `sharing`, and `praise` are top-level domains, not `bible` submodules.
- External APIs are HTTP paths under `/api/v1/**`.
- Internal domain calls use Java Interfaces and DTOs, not HTTP paths.
- Other domains may import only `api/` UseCase/DTO packages.
- Do not import another domain's Entity, Service, Repository, or infrastructure type.
- Controllers must not call repositories directly.
- Explanation and simulator request paths must not generate AI content on demand; F-15 Q&A may call LLM only through the single-turn validation flow.
- Keep these as documentation constraints here. Actual packages, workflow files, API contracts, data review tables, and test classes are created in the separate implementation repository.

## 5. Fixed Product Decisions

- QT range is published at **00:00 KST**; our collection batch runs at **04:00 KST**.
- From 00:00 to before 04:00, serve the previously prepared QT cache.
- "Today QT 100%" means passage, explanation entry point, note entry point, and simulator status are returned.
- It does not mean every passage has an actual simulator clip.
- Simulator status is `READY`, `MISSING`, `FAILED`, or `DISABLED`; enable the button only for `READY`.
- AI explanations and simulator content are pre-generated by batch or admin trigger.
- Fact-based Q&A AI is single-turn, limited to words, historical background, and period context, and must pass inline validation.
- Explanation source model: protect Korean validation references, use approved source materials for generation, and expose only verified `verse_explanations`.
- Songs are operator-curated metadata. Store user song references only.
- Anonymous users may start Kakao login only; app content APIs require authenticated roles per `04_API_명세서.md`.

## 6. Forbidden Work

Do not add or scaffold these, even temporarily:

- AI free chatbot, multi-turn conversation, SSE, or `/ai/sessions/**`
- RAG, ChromaDB, vector DB, Elasticsearch
- Kafka, Kubernetes, Helm for v1
- Church verification screens, buttons, APIs, or DB fields
- AI worship-song recommendations
- Lyrics, audio files, or direct YouTube URL input/storage
- Korean Revised Version, ESV, NIV seed/test/fixture/response data
- Scripture Union or Duranno Bible text storage; collect QT range information only
- Plain secrets in code, YAML, examples, logs, or docs

Use cautious copyright wording: "저작권 리스크를 낮춘다".
For events, describe handler failure logs and reprocessing ability; do not promise perfect delivery.

## 7. Code Rules

- Use `jakarta.*`, not `javax.*`.
- Use `@Transactional` on write use cases and command services; use `readOnly = true` for read paths where appropriate.
- Keep domain models inside their domain. Return DTOs across boundaries.
- Record AI work in `ai_generation_jobs`, `ai_generated_assets`, and `ai_validation_logs` with prompt/model/hash/status/error details.
- Admin APIs require the proper admin role; batch/AI internal work is recorded as `SYSTEM_BATCH`.
- Event handlers must log `eventId`, event type, handler name, and error message on failure.
- Logs must not include passwords, tokens, private keys, or sensitive personal data.
- Prefer Korean for user-facing messages, PR explanations, and domain comments.

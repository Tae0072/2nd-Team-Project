# QT-AI 문서·명세 기준 저장소

QT-AI는 오늘의 QT 본문, 쉬운 해설, 묵상 기록, 찬양 큐레이션, 익명 나눔, 시뮬레이터 상태, 관리자 운영을 다루는 큐티 앱 프로젝트입니다.

이 저장소는 **실제 구현 코드 저장소가 아니라 문서·명세 기준 틀**입니다. Spring Boot, Flutter, OpenAPI, DB, infra, CI workflow 같은 실제 구현 골격은 별도 구현 GitHub에서 생성합니다.

## 고정 기준

- 요구사항 충돌 시 [07_요구사항_정의서.md](./07_요구사항_정의서.md) v2.3을 우선합니다.
- 백엔드는 v1에서 단일 `qtai-server` Modular Monolith 기준입니다.
- 외부 공개 API는 `/api/v1/**`, 내부 도메인 호출은 Java Interface로 분리합니다.
- QT 공개 시각은 00:00 KST, 수집 배치 시각은 04:00 KST입니다.
- Today QT 100%는 본문, 해설 진입점, 묵상 진입점, 시뮬레이터 상태값이 정상 응답한다는 뜻입니다.
- 시뮬레이터는 `READY`일 때만 보기 버튼을 활성화하고, `MISSING`, `FAILED`, `DISABLED`는 비활성화합니다.
- 저작권 표현은 "저작권 리스크를 낮춘다"로 통일합니다.
- 이벤트는 "핸들러 실패 로그 + 재처리 가능" 기준으로 현실화합니다.

# 성경 JSON 출처 검토표 — QT-AI v2.3

> **문서 버전:** v0.1
> **작성일:** 2026-05-15
> **기준 문서:** `07_요구사항_정의서.md` v2.3
> **문서 역할:** 구현 저장소에 적재할 성경 JSON 데이터의 출처, 라이선스, 출처 표기, 재배포 가능 여부를 검토한다.

---

## 1. 목적

성경 본문 데이터는 GitHub 공개 JSON이라도 라이선스가 불명확하면 적재하지 않는다.

이 문서는 실제 DB seed 또는 데이터 적재 PR 전에 번역본별 사용 가능 여부를 검토하기 위한 기준 표다. 본문 텍스트 원문은 이 문서에 붙여 넣지 않는다.

---

## 2. 승인 기준

| 항목 | 기준 |
| --- | --- |
| Repo URL | 공개적으로 접근 가능한 원본 저장소 URL이 있어야 한다. |
| 라이선스 | 라이선스명 또는 Public Domain 여부가 명확해야 한다. |
| 번역본명 | 앱과 관리자 화면에 표시할 번역본명을 확정해야 한다. |
| 출처 표기 방식 | 앱 설정/출처 화면과 관리자 데이터 화면에 표시할 문구를 정해야 한다. |
| DB 적재 가능 여부 | 로컬 DB에 절 단위로 적재 가능한지 확인해야 한다. |
| 앱 내 노출 가능 여부 | 사용자 화면/API 응답에 노출 가능한지 확인해야 한다. |
| 재배포 가능 여부 | GitHub, 배포 산출물, seed 파일로 재배포 가능한지 확인해야 한다. |
| 검토자/검토일 | 누가 언제 검토했는지 남겨야 한다. |

## 5. 적재 전 체크리스트

- [ ] Repo URL이 실제 원본 저장소를 가리킨다.
- [ ] 라이선스 파일 또는 명시 문구를 확인했다.
- [ ] 번역본명과 약어를 확정했다.
- [ ] 출처 표기 문구를 확정했다.
- [ ] DB 적재 가능 여부를 확인했다.
- [ ] 앱 내 노출 가능 여부를 확인했다.
- [ ] 재배포 가능 여부를 확인했다.
- [ ] 금지 번역본이 아님을 확인했다.
- [ ] seed, fixture, 테스트 데이터에 금지 번역본이 섞이지 않았다.
- [ ] 본문 텍스트 원문을 문서에 붙여 넣지 않았다.

## 6. 구현 반영 기준

| 산출물 | 기준 |
| --- | --- |
| DB seed | 승인된 번역본만 포함 |
| 테스트 fixture | 승인된 번역본의 짧은 샘플 또는 mock 사용 |
| 관리자 화면 | 번역본명, 라이선스, 출처, 검토 상태 표시 |
| 사용자 화면 | 필요한 경우 설정/출처 화면에 attribution 표시 |
| CI Guard | 금지 번역본 문자열이 seed/config/fixture에 있으면 차단 |

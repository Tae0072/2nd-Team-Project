# 팀 개인 작업 공간 (workspaces/)

이 디렉터리는 **각 팀원의 개인 작업 공간**입니다. 워크플로우·리포트·개인 문서·메모를 보관합니다.

## 🚨 핵심 규칙 — 폴더 격리

| 규칙 | 설명 |
|------|------|
| **본인 폴더만** | 각 개발자(에이전트)는 자신의 폴더(`workspaces/{본인명}/`)에서만 작성·수정·삭제 가능 |
| **타인 폴더 금지** | 다른 팀원의 폴더는 **읽지도, 수정하지도, 삭제하지도** 않음 |
| **프로젝트 영향 금지** | `workspaces/`의 어떤 파일도 빌드·런타임·테스트에 영향을 주면 안 됨 |
| **에이전트 격리** | Claude/AI 에이전트도 동일 규칙 적용 — 본인 담당이 아닌 폴더 접근 금지 |

## 📁 구조

```
workspaces/
├── README.md                    (이 파일)
├── Lead_강태오/                 ← Lead만 접근
├── DevA_이지윤/                 ← DevA만 접근
├── DevB_김태혁/                 ← DevB만 접근
├── DevC_강상민/                 ← DevC만 접근
├── DevD_이승욱/                 ← DevD만 접근
└── DevE_김지민/                 ← DevE만 접근
```

각 개인 폴더 내부:

```
{개발자명}/
├── README.md         (개인 폴더 사용법)
├── workflows/        (작업 시작 전 워크플로우)
│   └── _template.md
├── reports/          (작업 완료 후 리포트)
│   └── _template.md
├── docs/             (개인이 작성하는 문서·명세)
└── notes/            (개인 메모·로그·스크린샷)
```

## 📋 표준 작업 프로세스 (모든 팀원 동일)

```
1. workflows/YYYYMMDD-작업명.md  작성  (작업 시작 전)
        ↓
2. 자기 검토 (체크리스트 ✓)
        ↓
3. 작업 진행
        ↓
4. reports/YYYYMMDD-작업명.md  작성  (작업 종료 후)
```

- 워크플로우 양식: `{개발자명}/workflows/_template.md`
- 리포트 양식: `{개발자명}/reports/_template.md`

## ⚠️ Git 정책

이 디렉터리는 **개인 작업 공간이지만 Git 추적 대상**입니다 (팀 가시성 확보).
단, 각자 본인 폴더만 commit 합니다. 다른 사람 폴더의 변경 사항이 PR에 포함되면 자동 reject.

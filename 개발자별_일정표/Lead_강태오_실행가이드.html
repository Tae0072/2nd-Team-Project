1. 역할 선언

이 파일은 강태오 Lead 실행가이드이면서 팀원별 일정표의 공통 기준이다. 개인별 작업은 달라질 수 있지만 W1~W5 게이트, 금지 기준, Today QT 기준은 바꾸지 않는다.

주 역할

Lead / gatewayauth / bff / DevOps / 전체 조율. 단일 파트 전담이 아니라 도메인 경계와 PR 품질을 함께 잡는다.

구현 구조

백엔드는 단일 qtai-server Modular Monolith다. 도메인은 패키지와 내부 Interface로 분리한다.

협업 방식

gatewayauth와 bff는 Bible·Flutter 담당자와 API 계약을 맞춰 구현한다. 외부 API와 내부 도메인 Interface를 섞지 않는다.

산출물

PR, 검증 로그, Foundation 5/5 판정표, 회고 메모, 문서 정합성 수정 내역을 남긴다.

2. 고정 기준

구분기준
일정 기준W1은 2026-05-18 ~ 2026-05-22 Foundation Lock-in, W3는 Feature Freeze, W4는 시연 후보 빌드, W5는 리허설과 발표다.
Today QT성서 유니온 공개 시각은 00:00 KST, 우리 시스템 수집 배치 시각은 04:00 KST로 통일한다.
Today QT 100%본문·해설·묵상 진입점·시뮬레이터 상태값이 정상 응답하면 100%다. 모든 클립이 존재해야 한다는 뜻이 아니다.
AI사용자 AI Q&A, 챗봇, SSE, /ai/sessions/** 사용자 경로는 만들지 않는다. AI는 배치와 관리자 트리거 전용이다.
이벤트v1은 Spring ApplicationEventPublisher를 사용한다. Kafka는 MVP 작업 목표에 넣지 않는다.
배포v1은 Docker Compose 기준이다. Kubernetes와 Helm은 MVP 작업 목표에 넣지 않는다.
성경 데이터Repo URL, 라이선스, 번역본명, 출처 표기, 재배포 가능 여부가 확인된 GitHub 공개 JSON만 적재한다. 개역개정, ESV, NIV는 금지한다.

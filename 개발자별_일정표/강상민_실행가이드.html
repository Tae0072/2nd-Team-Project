2. 공통 고정 기준

구분기준
문서 저장소 경계이 저장소는 문서·명세 기준 틀이다. 실제 Spring Boot/Flutter 골격, OpenAPI 파일, DB/데이터 디렉터리, infra, CI workflow는 별도 구현 GitHub에서 만든다.
기준 문서요구사항 충돌 시 07_요구사항_정의서.md v2.3을 우선한다. 개인 작업은 00_개발_일정_총괄표.md의 W1~W5 게이트를 벗어나지 않는다.
백엔드 구조v1은 단일 qtai-server Modular Monolith다. gatewayauth, bff, bible, ai, simulator는 패키지와 내부 Interface로 분리한다.
Today QT공개 시각은 00:00 KST, 수집 배치 시각은 04:00 KST다. 00:00~04:00은 이전 준비 캐시를 사용한다.
Today QT 100%본문, 해설 진입점, 묵상 진입점, 시뮬레이터 상태값이 정상 응답하면 100%다. 모든 시뮬레이터 클립이 있다는 뜻이 아니다.
AI/금지 기능사용자 AI Q&A, 챗봇, SSE, /ai/sessions/** 사용자 경로, AI 찬양 추천, 교회 인증은 MVP 작업 목표에 넣지 않는다.
성경 데이터Repo URL, 라이선스, 번역본명, 출처 표기 방식, 재배포 가능 여부를 확인한다. 개역개정, ESV, NIV는 저장·응답·테스트 데이터로 쓰지 않는다.
인프라Kafka, Kubernetes, Helm은 v1 MVP 목표에 넣지 않는다. 이벤트는 ApplicationEventPublisher, 배포는 Docker Compose 기준이다.

2. 고정 기준

구분 기준
AI 호출 04:00 KST 배치 또는 관리자 트리거로만 실행한다. 사용자 요청 경로에서는 LLM을 호출하지 않는다.
사용자 AI 사용자 AI Q&A, 챗봇, SSE, `/ai/sessions/**` 사용자 경로는 만들지 않는다.
해설 구조 B 테이블 영어 원문 주석을 AI가 재해석해 C 테이블 사용자 노출본을 만든다.
A 테이블 최신 주석 A 테이블은 검증용 비공개 데이터이며 사용자에게 노출하지 않는다.
검증 편집자 에이전트가 사실 정합성과 말투를 검증하고 통과한 결과만 C 테이블에 적재한다.
실패 처리 실행 실패는 `ai_runs`에 `FAILED` 또는 `REJECTED`로 기록하고 재처리 가능하게 둔다.

## 2. 고정 기준

| 구분 | 기준 |
| --- | --- |
| Today QT 100% | 시뮬레이터 상태값이 정상 응답하면 100% 기준을 만족한다. 모든 클립이 필수라는 뜻이 아니다. |
| 버튼 처리 | `READY`만 보기 버튼 활성화, `MISSING`/`FAILED`/`DISABLED`는 비활성화 |
| 생성 방식 | 본문 범위별로 클립을 한 번 생성한 뒤 재사용한다. 매 요청 시 재생성하지 않는다. |
| 일반 성경 보기 | 일반 성경 보기에서는 시뮬레이터 진입점을 노출하지 않는다. |
| 도메인 경계 | simulator 도메인은 bible/ai 도메인의 내부 클래스를 직접 import하지 않는다. |
| 실패 처리 | 생성 실패는 `FAILED` 상태와 로그를 남기고 재처리 가능하게 둔다. |
